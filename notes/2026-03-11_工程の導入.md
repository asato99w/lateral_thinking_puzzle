# 工程の導入

## 背景

v3 統合アルゴリズムの Phase 1-8 を一本の直線として並べる構造に問題がある。Phase 6-8（偽側構築〜検証）を機械的に適用した結果、モデル分析を欠いた浅い出力が生成された。これは適用の問題ではなくアルゴリズムの構造自体の問題。

Phase 1-8 の上位に「工程」という区切りを導入し、段階的に構築する。

## 工程の構成

### 工程1: 真側の構築

- Phase 1: 緊張構造分析 + モデル列挙
- Phase 2: モデル具象命題 + 分割構造
- Phase 3: S 細分化 + 戦術連鎖
- Phase 4: 質問生成
- 検証（真側）

真側だけで完結する。偽側は扱わない。

### 工程2: ブラッシュアップ

- 真側の想起条件の跳躍レビュー
- 戦術連鎖の修正
- 再検証
- 反復

工程1の真側を磨き上げる。

### 工程3: 偽側の構築（要設計）

- 棄却分析（現 Phase 5 に相当）
- 偽側戦術連鎖構築
- 偽側質問生成と分岐統合
- 全体検証

## 設計上のポイント

- **棄却分析の位置**: 現行では Phase 5 として Phase 4 の直後にあるが、偽側の構築と一体であるため工程3の冒頭に移動する
- **検証の位置**: 工程1の末尾と工程3の末尾の2箇所。現行は Phase 8 として最後尾にしかない
- **工程3の内部構成**: 現行アルゴリズムのままでは不十分。0から設計が必要
