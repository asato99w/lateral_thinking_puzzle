# H の役割分析

## 結論

H は特定パラダイム内での質問オープンの微調整を**理論的には果たせる**。現在は果たしていない（データ/設計の問題）。

## H の位置づけ

- H の唯一の役割: `open(H) = { q | ∃(d,v) ∈ effect(q) : |H(d)−v| < ε }`
- tension, alignment, パラダイムシフトは O のみに依存し、H は不使用
- 同じ O なら同じパラダイムに属する → H はパラダイム間の差異を生まない
- H の機能は「パラダイム内の段階的開示」に限定される

## 同化による段階的開示

更新式: `H(d') ← H(d') + w × (P(d') − H(d'))`

- P(d')=1 の場合: H(d') = 0.5 + 0.5w
- ε=0.2 でオープン条件を満たすには **w > 0.6**

R(P) のトポロジーに沿った波及:

1. d_a が O に入る → 同化で H(d_b) が移動 → d_b 関連質問がオープン
2. d_b に回答 → 同化で H(d_c), H(d_d) が移動 → 次の質問がオープン
3. R(P) の構造が開示の順序を自然に制御する

複数ソースの合流: 単独 w < 0.6 でも累積的にオープン可能。「複数の裏付けが揃って初めて見える質問」が表現される。

## 現在機能しない原因

1. **init_questions が H を迂回**: P の予測と effect(q) の一致だけで一括オープンしており、H の段階的制御が入る余地がない
2. **Ps が D(P1) に含まれていない**: 初期同化が起きず、H は非 Ps 記述素について 0.5 のまま
3. **R(P) の根と Ps/O の接続が不在**: 同化の起点がなく波及が始まらない

## 設計方針

- init_questions を廃止し open(H) に統一する
- Ps を適切な D(P) に含め、Ps→Fs の R(P) 関係を設計する
- w と ε の関係を整合的に設計する

## H を廃止する場合

open(O) = { q | ∃(d,v) ∈ effect(q) : d∈O ∧ O(d)=v } で代替可能。より単純だが「推測による先行オープン」を失い、段階的開示の粒度が粗くなる。
