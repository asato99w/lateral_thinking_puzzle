# 分割構造の再検討

## 発端

工程3（偽側の構築）を検討する中で、以下の認識に至った。

### 偽側連鎖の概念的誤り

分割命題は全て T において真。ゆえに P1=偽の方向にプレイヤーが「進む」ことは原理的に起こらない。ゲームマスターは T に基づいて回答するため、プレイヤーは常に真側の回答を受け取る。「偽側の連鎖を設計する」という発想は、存在しないパスを作ろうとしていた。

### 別の分割軸の存在

工程3で考えるべきは「偽側の連鎖」ではなく「別の分割軸による別のパス」の設計かもしれない。現行の分割（P1→P2→P3→P4）はありうる分割の一つであり、異なる抽象命題で切れば異なる階層・異なる到達経路が生まれる。

## 現行 Phase 2 の問題

### 規則の矛盾

現行規則に2つの基準が混在していた:

1. Step 3: 「最も多くのモデルを分割する共通命題を上位に」
2. 規則: 「分割の階層は上位（棄却力大）から下位（棄却力小）へ」

抽象度と棄却力は本質的にトレードオフ（広い共通性 = 大きい true_set = 小さい false_set = 低い棄却力）。基準2の「棄却力大→小」は「棄却力小→大」の記述誤り。修正すれば矛盾は解消する。

### 抽象命題列挙の欠如

現行の手順は「意味的にグルーピングし、共通する上位命題を構成する」とだけ記載されており、網羅的な列挙の手続きがない。結果として、最大の分割（true_set=8）が見落とされ、次点の分割（P1, true_set=7）が選ばれていた。

## 改善案: 抽象命題列挙の手続き化

抽象化と意味の確定を一度にやると、抽象を歪めて具体化したり、意味が曖昧なまま進めてしまう。ステップを分け、各ステップの出力をデータとして記録する。

### Step 2b-1: 軸ごとのモデルマッピング

モデル具象命題を因果の役割ごとの軸で分類し、マトリクスとして記録する。ここでは抽象化しない。モデル定義そのままの事実を並べる。

軸の例（ウミガメのスープの場合）:
- 過去の出来事の状況
- スープの正体
- 欺瞞の構造
- 発見のメカニズム
- 自殺の直接動機
- 人物の属性

軸の設定自体にはまだ判断が残る。

### Step 2b-2: グルーピング

マトリクスの各軸で、同じ値や類似の値を持つモデル群を特定する。グループとそのメンバーを列挙する。まだ命題にしない。

### Step 2b-3: 抽象命題の生成

各グループの共通性を命題として言語化する。この段階では意味の確定性は問わない。抽象としての正しさだけ見る。

### Step 2b-4: 意味の確定性チェック

各抽象命題について、全モデルで真偽が一意に定まるかを検証する。不確定なモデルがあれば、モデル定義の不備か命題の曖昧さかを判別する。命題を修正する場合は修正前後を両方記録する。

## ウミガメのスープでの実例（抜粋）

### 同一分割の発見

3つの異なる抽象命題が同一の分割（true_set=8）を生んだ:
- 「味の比較が発見のきっかけ」
- 「スープに関連する過去体験がある」
- 「欺瞞が存在した（方向問わず）」

いずれも true_set = {T,M1,M2,M3,M4,M5,M8,M12}。異なる軸の抽象が同一分割に合流する現象は、この8モデルが構造的に一つのクラスタであることを示唆する。

### 現行P1との差

上記の最大分割（size=8）と現行P1（size=7）の差は M5（料理人）のみ。M5は偽スープを「飲んだ」のではなく「作った」側。「飲んだ経験」という具体的な命題では M5 が外れるが、「味の比較」「過去体験」「欺瞞の存在」といったより抽象的な命題では M5 を含む。

## 工程3の方向性: 間接棄却と直接棄却

### 二種類の棄却

現行の構造では、モデルの棄却はすべて**間接棄却**で行われている。

- **間接棄却（抽象による棄却）**: 分割命題（抽象命題）を確認することで、その命題が偽であるモデル群がまとめて棄却される。プレイヤーは個々のモデルを意識せず、抽象レベルで絞り込む。
  - 例: P1 を確認 → M5, M6, M7, M9, M10, M11 がまとめて棄却される

- **直接棄却（コア命題による棄却）**: 各モデル固有のコア命題を直接否定することで、そのモデルを個別に棄却する。プレイヤーが具体的な仮説を能動的に探索・排除する体験。
  - 例: 「男は料理人ですか？」→「いいえ」→ M5 が棄却される

間接棄却のみではプレイヤーが個々の仮説を検討する機会がない。直接棄却を加えることで探索の具体性と深さが増す。

### コア命題の定義

各モデルの「コア命題」を恣意的に選ぶのではなく、以下の基準で特定する:

1. そのモデルの具象命題のうち、**T において偽であるもの**（T と区別する命題）を列挙する
2. その中から、**除去するとモデル自体が成立しなくなるもの**を特定する — モデルの因果構造における根幹の前提

「最も直接的」「最も重要」といった曖昧な基準ではなく、「その命題がなければモデルの因果連鎖が崩壊する」という構造的な基準。

### 分割ノードに紐づく戦術連鎖

分割階層の各ノードに対して、二種類の戦術連鎖を設計する:

- **真側の戦術連鎖**（工程1で構築済み）: S 命題または上位分割命題から、当該ノードの分割命題への到達パス
- **棄却側の戦術連鎖**（工程3で構築）: 当該ノードで棄却されるモデル群のコア命題を直接テストするパス

棄却側の連鎖は分割構造のノードに紐づくため、どのノードにどの棄却連鎖があるかが構造的に明確になる。

#### ウミガメのスープでの例

```
P1 ノード（false_set: M5, M6, M7, M9, M10, M11）
├─ 真側連鎖: S → ... → P1 確認（間接棄却: M5,M6,M7,M9,M10,M11）
└─ 棄却側連鎖:
    ├─ M5 のコア命題へ: 「男は料理人ですか？」→ いいえ
    ├─ M6 のコア命題へ: 「宗教的な理由ですか？」→ いいえ
    ├─ M7 のコア命題へ: 「男は病気ですか？」→ いいえ
    ├─ M9 のコア命題へ: 「男は保護活動家ですか？」→ いいえ
    ├─ M10 のコア命題へ: 「男は調査員ですか？」→ いいえ
    └─ M11 のコア命題へ: 「自殺は事前に決めていましたか？」→ いいえ

P2 ノード（false_set: M2, M3, M8, M12）
├─ 真側連鎖: P1 → ... → P2 確認（間接棄却: M2,M3,M8,M12）
└─ 棄却側連鎖:
    ├─ M2 のコア命題へ
    ├─ M3 のコア命題へ
    ├─ M8 のコア命題へ
    └─ M12 のコア命題へ

P3 ノード（false_set: M4）
├─ 真側連鎖: P2 → ... → P3 確認（間接棄却: M4）
└─ 棄却側連鎖:
    └─ M4 のコア命題へ

P4 ノード（false_set: M1）
├─ 真側連鎖: P3 → ... → P4 確認（間接棄却: M1）
└─ 棄却側連鎖:
    └─ M1 のコア命題へ
```

プレイヤーの体験としては:
- 抽象的な絞り込み（真側連鎖）と具体的な仮説検証（棄却側連鎖）の両方が可能
- 「こういう可能性もあるのでは？」→「いいえ」という能動的な探索パスが生まれる

### 連鎖内分岐による棄却の分析

棄却側を分割ノード単位で考えるのではなく、真側連鎖の各ステップにおける並列命題（反対側）として捉え直す。分岐の設計は質問レベルではなく戦術連鎖（命題と戦術）のレベルで行う。質問は後の工程で連鎖から導出される。

#### 連鎖レベルでの分岐の検討

各連鎖の各ステップについて、真側命題に対する並列命題と、それに対応するモデルを整理する。

**C1: S → N1a → I1 → P1**

| ステップ | 真側命題 | 並列命題 | 対応モデル |
|---|---|---|---|
| N1a | 味の不味さが理由ではない | 味の不味さが理由 | M7? |
| I1 | 以前にも飲んだことがある | 飲んだことがない | M6, M7, M9, M10, M11 |
| P1 | 過去に偽のスープを飲んだ | 偽のスープを飲んでいない | M5, M6, M7, M9, M10, M11 |

**C2: P1 → N2a → N2b → P2**

| ステップ | 真側命題 | 並列命題 | 対応モデル |
|---|---|---|---|
| N2a | 日常的ではなかった | 日常的だった | M2 |
| N2b | 極限状態だった | 非日常だが極限ではない | M8 |
| P2 | 海での遭難 | 極限だが海ではない | M3, M12 |

**C3: P2 → N3a → N3b → P3 → N4a → P4**

| ステップ | 真側命題 | 並列命題 | 対応モデル |
|---|---|---|---|
| N3a | 仲間がいた | 仲間がいなかった | M4 |
| N3b | 仲間が死んだ | 仲間は死んでいない | — |
| P3 | 他者が意図的に偽った | 意図的な欺瞞ではない | M4 |
| N4a | 一般的な食用動物ではない | 食用動物だった | M1 |
| P4 | 人肉だった | 人肉ではない | M1 |

#### カバレッジ

- C2, C3: 各ステップの分岐で全対象モデル（M1, M2, M3, M4, M8, M12）がカバーされている
- C1: M5 の位置が不明確（I1 と P1 の間に落ちる — 以前に飲んだことはあるが偽のスープではない）。M7 の位置も曖昧

## 工程3の流れ

1. **連鎖内分岐の分析**: 真側連鎖の各ステップの並列命題を列挙し、モデルとの対応を確認する
2. **カバレッジ確認**: 全モデルが連鎖内分岐でカバーされているかを確認する
3. **質問生成**: 分岐から質問を導出する

まずこの3ステップで工程を回し、カバーされないモデルへの対応（コア命題による直接棄却、別の分割軸等）は結果を見てから検討する。

### 保留事項

以下は工程3の基本フローの結果を見てから判断する:

- コア命題による直接棄却の必要性と手続き
- 別の分割軸による代替パスの設計
- 連鎖内分岐の並列命題を体系的に列挙する手続きの詳細化

## 未解決の課題（工程1・Phase 2 関連）

- 軸の設定（Step 2b-1）をより機械的にする方法
- 抽象命題の生成（Step 2b-3）と意味の確定（Step 2b-4）の間で抽象が歪まないことの保証
- 規則ファイルへの反映（棄却力の順序の記述誤り修正を含む）
