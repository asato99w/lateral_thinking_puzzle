# 目的先行型統合アルゴリズムの検討

## 背景

現行の統合アルゴリズムは、記述素空間を発散的に展開し、その空間からパラダイムを発見し、質問を設計するボトムアップ型のプロセスである。これを、S と T から先に真相へ至る質問の流れを導出し、それに必要な構造を逆算的に構築するトップダウン型（目的先行型）のプロセスに再構成することを検討する。

入力は現行と同じ S と T のみ。設計者が体験を具体的に指定するのではなく、アルゴリズム自体が目的先行的にプロセスを進める。

## 提案するプロセス（暫定）

### Step 1: ギャップ分析 — 認知的断層の特定

S と T を突き合わせ、プレイヤーが超えなければならない認知的断層を特定する。

### Step 2: パス候補の探索

Step 1 で特定した断層の組に対して、横断するパスの候補を複数探索する。各パスは S の異なる要素を起点とし、断層を異なる順序で横断する。

- 起点の選択: S のどの要素から出発するか（例: 確認行為、自殺、スープの注文）
- 横断順序: 断層をどの順で通過するか（逆行型、前進型など）
- 完成条件: 各パスが全断層をカバーしていること

### Step 3: 質問シーケンスの導出

Step 2 で探索したパス候補から最も論理的に自然なものを選択し、具体的な質問連鎖を構成する。S から T への論理的な妥当性を基準とし、各質問は「S の要素、または前の質問の回答」から導出される。

### ステップ間の関係

- Step 1 が「何を超えるべきか」（断層の特定）を担う
- Step 2 が「どの順路で超えるか」（パスの探索）を担う
- Step 3 が「具体的にどう超えるか」（質問連鎖の構成）を担う
- Step 3 の連鎖が Step 1 の全断層をカバーしていることが完成条件

## ウミガメのスープへの適用

### Step 1: 認知的断層

**断層 α: 確認行為の意味の転換**
- S の読み: 味がおかしいから確認した（品質への不満）
- T の実態: 過去に飲んだものと味が違うから確認した（記憶との比較）

**断層 β: 過去の体験の存在**
- S: 過去についての情報はゼロ
- T: 遭難・食料枯渇・仲間の死・人肉食という極限体験がある

**断層 γ: 「本物である」ことの反転**
- S の読み: 本物だと確認され安心する（通常の帰結）
- T の実態: 本物だと確認されたことが「過去のスープは偽物だった」という証拠に反転する

**断層 δ: 自殺に至る動機の深度**
- S の読み: 食事に関する出来事で自殺する理由が不明
- T の実態: 仲間の人肉を食べていたという悟りによる絶望

**断層間の因果関係**:
```
α（確認は過去に起因）→ β（過去に極限体験がある）→ γ（本物＝過去は偽物という反転）→ δ（偽物の正体を悟り自殺）
```

### Step 2: S → T 質問連鎖

起点: Ps-5 + Ps-6 のパラドックス（本物だと確認された → 自殺した）

**Q1: 本物だと確認されたことが、自殺の引き金になったか？** → はい
- 根拠: Ps-5 + Ps-6 の直接的な結合。S が提示するパラドックスそのもの

**Q2: 本物だとわかったことで、過去のある事実が明らかになったか？** → はい
- 根拠: Q1。現在の真実が絶望を生むなら、それが過去の何かを暴いた

**Q3: 男が味に疑問を持ったのは、過去の味の記憶と違ったからか？** → はい
- 根拠: Q2 + Ps-3 + Ps-4。過去の事実 + 一口飲んでから確認した → 味の比較

**Q4: 過去に味わったものは「ウミガメのスープ」だと言われていたか？** → はい
- 根拠: Q3 + Ps-4。味の記憶の対象がウミガメのスープに関わる → 確認行為の動機

**（論理的帰結: 今回が本物で味が違う → 過去のものは本物ではなかった）**
- Q3 + Q4 + Ps-5 からの演繹。質問不要

**Q5: その正体を悟ったことが自殺の動機か？** → はい
- 根拠: 帰結 + Ps-6

**Q6: それは人肉だったか？** → はい
- 根拠: Q5。自殺に至るほどの正体

### 断層と質問連鎖の対応

| 断層 | 内容 | 対応する質問 |
|------|------|-------------|
| γ | 本物の確認 → 反転 | Q1（パラドックスの起点） |
| α | 確認行為の意味の転換 | Q3, Q4（味の記憶、過去の呼称） |
| β | 過去の体験の存在 | Q2, Q3, Q4（過去の事実の浮上） |
| δ | 自殺動機の深度 | Q5, Q6（正体の悟り） |

横断の順序は断層の列挙順（α→β→γ→δ）とは異なり、S のパラドックス（γ）から出発して論理的に自然な順序で他の断層を通過する。

## 設計判断

### 断層の複数分解はオプション

Step 1 の断層特定において、同じ S-T ギャップを別の観点（例: 確認行為パラドックス起点 vs 欺瞞起点）で分解し、異なる断層の組を導出できる可能性がある。複数の分解が得られれば、異なる質問連鎖が導出され、パズルの構造を多角的に捉えられる。

ただし、S の情報量が少ないパズルでは自然な複数分解が困難。強引に導出するのではなく、自然に見つかる場合に活用するオプションとする。

## 骨格記述素の導出

質問連鎖の各ステップが確定させる事実を記述素として構成する。Ps/Pt からの直接抽出ではなく、S → T の論理的橋渡しとして生成される最小構成。

| ID | 記述素 | 由来 |
|----|--------|------|
| d1 | 本物だと確認されたことが自殺の引き金になった | Ps-5 + Ps-6 の結合 |
| d2 | 本物だとわかったことで過去のある事実が明らかになった | d1 からの推論 |
| d3 | 男は過去の味の記憶と今回の味が違うと感じた | Ps-3 + Ps-4 + d2 |
| d4 | 過去に味わったものは「ウミガメのスープ」だと言われていた | d3 + Ps-4 |
| d5 | 過去に飲まされたものは本物のウミガメのスープではなかった | d3 + d4 + Ps-5 からの演繹 |
| d6 | その正体を悟ったことが自殺の動機である | d5 + Ps-6 |
| d7 | それは人肉だった | d6 の具体化 |

## 骨格・戦略・パラダイムの関係の再考

### 骨格と実際のゲームの乖離

骨格の論理連鎖は「はい」の回答のみで T に到達する最短経路である。しかし実際のゲームでは、プレイヤーは T を知らず、汎用的な探索戦略を用いて状況を絞り込んでいく。

たとえばウミガメのスープでは、骨格に「船乗り」「遭難」という要素は現れないが、実際のゲームでは「状況の絞り込み」戦略による「船乗り → 遭難 → 食料枯渇 → 仲間の死」の発見がコア体験になることが多い。これは骨格の跳躍（d6 → d7）を埋める文脈的探索である。

### 汎用戦略の例

パズルに依存しない、プレイヤーが用いる探索戦略：
- 状況の絞り込み（場所、時代、人数など）
- 登場人物の探索（他の関係者の存在）
- 時間軸の確認（過去の関与）
- 常識的仮説の排除（毒、病気、事故など）
- 動機の探索（意図、理由）
- 異常性の特定（なぜ通常と異なるか）

### 骨格はピースの一覧、パラダイムは解釈の枠組み

骨格をパラダイムの段階に直接対応させる（P1 → P2 → T を骨格の区間に写す）のは適切でない。実際のゲーム体験は：

- プレイヤーは汎用戦略で探索する
- 探索の過程で骨格のピースが**順不同に**拾われていく
- 「船乗り」「遭難」などの文脈的ピースも同時に拾う
- ある時点で十分なピースが揃い、真相が**閃く**

骨格はジグソーパズルのピースのように散在しており、パラダイムは骨格の上に直接乗るのではなく、**拾ったピースの組み合わせが生む解釈の枠組み**として現れる。パラダイムシフトは「ピースが揃って絵が見えた瞬間」に対応する。

この捉え方は、骨格 → パラダイムという直接導出ではなく、骨格と戦略の二層からパラダイム構造が浮かび上がるという構図を示唆する。
