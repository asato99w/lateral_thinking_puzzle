# 工程3: 仕掛け（ひっかけ）アルゴリズム

## 概要

工程1-2 で構築された CORE データ（記述素・ピース・戦術連鎖・質問）に対して、ひっかけ・ミスリードの仕掛けを追記する。前工程の出力は変更しない（追記のみ）。

工程1-2 が T → S 方向（逆算連鎖）で構築するのに対し、工程3 は S → プレイヤーの解釈 方向の現象を扱う。プレイヤーが自然に持ちうる誤った仮説を体系的に設計し、その仮説の形成・棄却をゲーム進行に組み込む。

## 入力

- S: 問題文
- T: 真理論（正解のストーリー）
- CORE データ: 工程1-2 の出力（data_src.json）
  - descriptors: 記述素群
  - initial_confirmed: S から確認済みの記述素
  - pieces: ピース群
  - triggers: ピースのトリガー条件
  - clear_conditions: クリア条件
  - _phase4: 戦術連鎖（chains_to_terminals, internal_chains）
  - questions: CORE 質問群

## 出力

CORE データへの追記:
- 対立仮説用の記述素（否定形を含む）
- 対立仮説の連鎖（_phase4 への追記または別セクション）
- 追加質問（rejection_conditions 付き）
- CORE 連鎖の分岐ノードに対する質問

## 手順

### Step 1: 認知的断層の特定

#### 目的

S と T の間に存在する構造的ギャップ（プレイヤーが越えなければならない認知的断層）を列挙する。

#### 入力

S, T

#### 手順

1. S のテキストが誘導する自然な解釈フレームを記述する
2. T の実際の因果構造を記述する
3. 両者の乖離箇所を認知的断層として列挙する
4. 各断層に ID（α, β, γ, ...）とラベルを付与する

#### 規則

- 断層は S と T の客観的な乖離から導く。主観的な「難しさ」ではない
- 1つの断層は1つの認知的ギャップに対応する。複数のギャップが含まれる場合は分割する
- 断層の粒度は後続ステップ（仮説集合の定義）で操作可能なレベルに保つ

#### 出力例

```
断層 α: 確認行為の意味の転換
  S側: 味に不満 → 品質確認
  T側: 過去の記憶との比較

断層 β: 過去の体験の存在
  S側: 情報なし（S の時間軸内で完結するはず）
  T側: 極限体験が存在する
```

---

### Step 2: 断層の主要仮説集合 H の定義

#### 目的

各断層を記述素の集合として形式化し、操作可能にする。

#### 入力

Step 1 の断層リスト、CORE の descriptors

#### 手順

1. 各断層に対して、その断層を構成する CORE 記述素を特定する
2. 記述素の重複を許可する（1つの記述素が複数の断層に属してよい）
3. 各断層の主要仮説集合 H を定義する: H_x = {D-i, D-j, ...}

#### 規則

- H の記述素は全て CORE（工程1-2 で定義済み）の記述素であること
- T の真の因果構造を反映すること

#### 出力例

```
H_β = {D-16(危機的体験), D-15(船), D-6(遭難), D-7(漂着)}
H_ε = {D-8(食糧尽きた), D-9(仲間死んだ), D-10(肉でスープ)}
```

---

### Step 3: 各断層の「問い」の定式化

#### 目的

各断層が核心的に問うている質問を自然言語で定式化する。Step 4 の対立仮説列挙の指針となる。

#### 入力

Step 1 の断層リスト、Step 2 の H

#### 手順

1. 各断層について「この断層をプレイヤーが越えるとき、本質的に何が問われているか」を一文で表現する
2. 「問い」に対する真の解決（H の記述素による回答）を確認する

#### 規則

- 「問い」は断層の本質を捉えた端的な表現であること
- 個別の記述素ではなく、断層全体の認知的ギャップを表現すること

#### 出力例

```
β の問い: 自殺の原因はどこにあるのか？
  真の解決: S の時間軸の外（過去の極限体験）にある

δ の問い: なぜ自殺するほどの絶望に至ったのか？
  真の解決: 自分が人肉を食べていたと悟った
```

---

### Step 4: 対立仮説集合族 {H'} の列挙

#### 目的

各断層の「問い」に対して、真とは異なる解決を提供する仮説群を列挙する。

#### 入力

Step 3 の「問い」と真の解決、S のテキスト

#### 手順

1. 各断層の「問い」に対して、S のテキストと一般常識から自然に浮かぶ別の解決を列挙する
2. 各対立解決を記述素の集合として表現する（新規記述素の生成を含む）
3. 対立仮説集合族 {H'_x} = {H'_x_1, H'_x_2, ...} を構成する

#### 規則

- 対立仮説は**解決**を持たなければならない。単なる方向・連想ではなく、断層の「問い」に対する整合的な回答であること
- S のテキストと矛盾しないこと（S の記述と直接矛盾する対立仮説は除外）
- T を知っている状態で評価しないこと（出題者バイアスの排除）。プレイヤーが T を知らない状態で自然に思いつくかどうかで判断する
- S に近い断層では S のテキスト表現が対立仮説を直接誘導する。S から遠い断層では「S の時間軸内で完結するはず」というデフォルト前提が対立仮説の源泉となる

#### 出力例

```
{H'_δ} = {
  H'_δ_1: {D-neg-δ1: スープに毒が入っていた, D-neg-δ1a: 毒で死んだ}
    解決: レストランで毒を盛られた
  H'_δ_2: {D-neg-δ2: 絶滅危惧種を食べた, D-neg-δ2a: 罪悪感で自殺}
    解決: 食べてはいけないものを食べた罪悪感
  H'_δ_3: {D-neg-δ3: 宗教的禁忌を犯した, D-neg-δ3a: 戒律違反の絶望}
    解決: 宗教的タブーを犯した
}
```

#### 射程の評価（参考）

対立解決が複数の断層にまたがって整合する場合、その対立仮説は射程が長い。射程の長い対立仮説はプレイヤーを長く引きつけるため、より強いひっかけとなる。列挙時に射程を参考情報として記録するが、独立したステップとしては不要。

---

### Step 5: 対立仮説の連鎖構成

#### 目的

各対立仮説集合について、CORE と同じ戦術体系を用いて戦術連鎖を構成する。プレイヤーがその仮説に至る推論経路を明示化する。

#### 入力

Step 4 の {H'_x}、CORE の戦術連鎖（_phase4）、S のテキスト、initial_confirmed

#### 手順

1. 各対立仮説集合 H'_x_i について、その記述素に至る連鎖を構成する
2. 連鎖の入力（起点）は以下のいずれか:
   - **S のテキスト（initial_confirmed の記述素）から直接**: S に近い断層の対立仮説に多い
   - **CORE 連鎖の各ノード（既に confirmed/derived された記述素）からの分岐**: CORE の途中で方向が分かれるケース
3. CORE と同じ戦術（因果の逆転、段階的絞り込み、動機の探索、etc.）を適用して連鎖を構成する
4. 各連鎖ステップを `{input, tactic, output}` の形式で記述する

#### 規則

- 連鎖は CORE の戦術体系と同一の戦術を使用すること
- 連鎖の各ステップの妥当性をプレイヤー視点（T を知らない状態）で評価すること
- 「一本道」かどうかの判断に出題者バイアスを持ち込まないこと。例: 「船に乗っていた → 遭難した」は T を知っていれば一本道に見えるが、プレイヤーにとっては多くの候補の一つに過ぎない
- S から直接連鎖する対立仮説も存在することを忘れないこと

#### 出力例

```
H'_δ_2（絶滅危惧種の罪悪感）の連鎖:
  起点: D-2(ウミガメのスープを注文) [initial_confirmed]
  Step 1: {input: [D-2], tactic: "観測事実の深掘り", output: [D-neg-δ2: ウミガメは絶滅危惧種]}
  Step 2: {input: [D-neg-δ2, D-5], tactic: "因果の逆転", output: [D-neg-δ2a: 絶滅危惧種を食べた罪悪感で自殺]}

H'_β_1（レストラン内完結）の連鎖:
  起点: D-3(本当にウミガメのスープかと尋ねた) [initial_confirmed]
  Step 1: {input: [D-3], tactic: "動機の探索", output: [D-neg-β1: スープに何か問題があった]}
  Step 2: {input: [D-neg-β1, D-5], tactic: "因果の逆転", output: [D-neg-β1a: スープの問題が自殺の原因]}
```

### Step 6: 質問の設計

2段階で質問を設計する。

#### Step 6a: 対立仮説の連鎖に対する質問生成

##### 目的

対立仮説の連鎖を棄却するための質問を生成する。主に「No」回答の質問。

##### 入力

Step 5 の対立仮説連鎖、Step 4 の対立仮説集合族

##### 手順

1. 各対立仮説の連鎖に対して、その仮説を棄却する質問を設計する
2. 質問の回答は「No」（対立仮説が誤りであることを示す）
3. 質問に rejection_conditions を付与する: 対立仮説の記述素が棄却される条件
4. 質問の recall_conditions を設定する: プレイヤーがその対立仮説を持っている状態（＝対立仮説の連鎖の入力記述素が confirmed されている状態）

##### 規則

- 質問は自然な問いかけであること（パズルとして不自然な質問は避ける）
- rejection_conditions は confirmed のみを参照すること（導出記述素は参照しない）
- 棄却が連鎖的に波及しうる（A 棄却 → A に依存する B も導出不能）ことを考慮する

##### 出力例

```
{
  "id": "Q-R1",
  "text": "スープに毒が入っていましたか？",
  "answer": "No",
  "recall_conditions": [["D-neg-δ1"]],
  "reveals": ["D-neg-δ1-rejected"],
  "rejection_conditions": {"D-neg-δ1": [["D-neg-δ1-rejected"]]},
  "mechanism": "rejection"
}
```

#### Step 6b: CORE 連鎖の分岐ノードへの質問配置

##### 目的

CORE 連鎖の途中で、プレイヤーが対立仮説の方向に逸れうるノードに、分岐を扱う質問を配置する。

##### 入力

Step 5 で特定された分岐点（CORE ノードから対立仮説連鎖が分岐する箇所）、CORE 質問群

##### 手順

1. CORE 連鎖の各ノードについて、対立仮説の連鎖が分岐するノードを特定する
2. 各分岐ノードに対して:
   a. 対立方向に進む質問（「No」回答で棄却、または「Yes」回答で CORE 方向に誘導）を設計する
   b. 既存の CORE 質問で分岐を処理できる場合は、追加不要
3. 質問の配置が CORE 質問群と整合することを確認する（recall_conditions の競合がないこと）

##### 規則

- CORE 連鎖の既存の質問・構造を変更しないこと
- 分岐ノードの質問は、プレイヤーが自然にその分岐点で抱く疑問であること
- 全ての分岐に質問を用意する必要はない。プレイヤーが自然に CORE 方向に進むステップは対処不要

##### 出力例

```
分岐ノード: D-15(船に乗っていた)
  CORE方向: D-6(遭難) [時間軸の拡張]
  対立方向: 漁師だった（日常的な船の経験）/ 海軍にいた / 船旅を楽しんだ

  → 追加質問: 「男の船で何か重大なことが起きましたか？」(Yes → D-6 への誘導)
  → この質問は CORE 連鎖の Q-6 と統合可能か検討する
```

## 検証

### 整合性検証

- 追加された記述素・質問が CORE のデータ構造と矛盾しないこと
- rejection_conditions が confirmed のみを参照していること
- 対立仮説の連鎖の各ステップが戦術体系に準拠していること

### ゲームプレイ検証

- 対立仮説に誘導された状態から、質問によって棄却され、CORE に復帰できること
- 棄却後に行き詰まり（オープン質問が尽きる）が生じないこと
- 対立仮説の存在がゲームの面白さ（探索の深み）を増していること

## 前提基盤

### rejection_conditions（未実装）

本アルゴリズムの Step 6a は rejection_conditions の仕組みを前提とする。データモデル・エンジン・eval への実装が必要:

- formation_conditions（肯定条件）に加え、rejection_conditions（否定条件）を導入
- 導出判定: formation_conditions 充足 AND rejection_conditions 非充足
- confirmed のみ参照（導出記述素は参照しない）
- 棄却は不可逆（振動しない）

### 戦術体系

CORE 連鎖で使用される戦術と同一:

| ID | 名称 | 説明 |
|---|---|---|
| 2 | 観測事実の深掘り | 既知の事実の詳細を探る |
| 3 | 時間軸の拡張 | 過去・未来に視野を広げる |
| 4 | 登場人物の探索 | 関係する人物を探す |
| 5 | 因果の逆転 | 原因と結果の関係を探る |
| 6 | 矛盾の指摘 | 既知の事実間の矛盾を見出す |
| 7 | ラベルと実体の乖離 | 名称・見かけと実態の違いを探る |
| 8 | 動機の探索 | 行動の動機を探る |
| 10 | 段階的絞り込み | 抽象から具体へ絞り込む |

## 備考

- 本アルゴリズムは初版であり、turtle_soup 03 への試行適用を通じて検証・改訂する
- 工程1-2 の出力を変更しない原則（追記のみ）を厳守する
- 出題者バイアスの排除は本アルゴリズムの最重要規律である
- 対立仮説の射程（複数断層にまたがる整合性）は強いひっかけの指標だが、射程の評価を独立したステップとはしない
- 「地形」（断層間の距離・重なりの形式化）は未解決の課題であり、将来の改訂で取り組む可能性がある
