# 統合アルゴリズム v6 検討

## 背景

v5サンプル生成と並行して進めた「パズルの面白さ」分析シリーズ（分析1〜16）から、パズルの構造に関する主要概念が得られた。これをv6としてアルゴリズムに反映することを検討する。

## 反映する主要概念

### 2要素特性

2緊張パターンを一般化したもの。Sの中の2つの独立した要素が、1つの鍵で繋がる構造。各要素が個別に緊張を持つことは前提としない（緊張は特殊ケース）。

- 例: バーの男 — 要素A（銃を向けた）、要素B（ありがとうと帰った）、鍵（しゃっくり）
- 排他的な分類ではなく、パズルが持ちうる**特性**として扱う

参照: 分析3（2緊張構造）、分析4（複数緊張の同時緩和）、分析7（2つのパターンとアノマリー）

### 隠れアノマリー特性

登場人物の前提群（パラダイム）→ 演繹 → 予測 → 観測（反証）→ どの前提が誤りか → 真相、という構造。

- 例: ウミガメのスープ — 味の不一致がアノマリー
- 2要素特性とは独立。両方を持つパズルもある（ウミガメのスープ）

参照: 分析7、分析8（不一致の設計）、分析14（前提群と反証の構造）

### 両特性の関係

- 排他的でなく重なりうる
- 登場人物の推論があること自体は区別基準ではない（バーの男は推論があるが2要素特性のみ）
- 隠れアノマリーの特徴はアノマリー観測を起点とする連鎖構造

## v5からの構造的変更の方向性

### 全体構成

```
FP（鍵を含む主要命題）= クリア条件
  FP.fc = [[..要素A側.., ..要素B側..]]
         ↑                ↑
    要素Aの経路        要素Bの経路
    （中間命題群）      （中間命題群）
         ↑                ↑
        S の観測          S の観測
```

- FPの想起条件が要素A・Bの両方をバランスよく含む
- プレイヤーが両方の経路を進めることでFPに到達する
- 設計課題: 各要素について、Sの観測からFPのfc要素に至る中間命題をどう構成するか

### 競合モデルの再定義

v5: パズル全体の代替ストーリーを列挙
v6方向: 各要素に対する**独立した説明モデル**を検討

各要素について「なぜこれが起きたか」の説明候補を独立に分析する。Tの鍵は両方の要素の説明を同時に提供するもの。

### 導出構造の設計（Phase 3相当）

v5: モデル比較で分割命題を作り → 命題間の導出関係を分析
v6方向: 鍵と要素の間を埋める中間命題を導出の中で直接設計する

導出構造を先に設計し、モデル弁別の機能は後から付与する方向。

## 中間命題導入の手法: バーの男での試行

### Step 1: T の原子的事実への分解 + 暗黙的事実の明示

T の明示的事実:
1. 男はしゃっくりが止まらなかった
2. 男は困っていた
3. 水を飲めば治ると思った
4. バーに立ち寄った
5. バーテンダーがしゃっくりに気づいた
6. 驚かせて止めようとした
7. 銃を向けた
8. しゃっくりが止まった
9. 水が不要になった
10. 「ありがとう」と言って帰った

暗黙的事実:
11. しゃっくりは外見・音で他者に認識可能な症状である
12. 驚かせることはしゃっくりの一般的な対処法である
13. バーテンダーの行為は男を助ける意図だった
14. 男は銃を向けられて驚いた
15. 男はバーテンダーの意図を理解した

### Step 2: S に含まれるものを除く

4, 7, 10 は S に含まれる。残り:

1, 2, 3, 5, 6, 8, 9, 11, 12, 13, 14, 15

### Step 3: 各要素に振り分け、鍵を含まない形にする

**要素A（銃を向けた）側:**

| T\S の事実 | 鍵を含まない形 |
|---|---|
| 5. バーテンダーがしゃっくりに気づいた | 男の状態に気づいた |
| 6. 驚かせて止めようとした | 対処しようとした |
| 11. しゃっくりは認識可能 | 男の状態は外見上認識可能だった |
| 12. 驚かせることは対処法 | 銃を向けることが対処として有効だった |
| 13. 助ける意図 | そのまま |

**要素B（ありがとうと帰った）側:**

| T\S の事実 | 鍵を含まない形 |
|---|---|
| 1. しゃっくりが止まらなかった | 男は何らかの問題を抱えていた |
| 2. 困っていた | そのまま |
| 3. 水を飲めば治ると思った | 水は問題への対処として求めた |
| 8. しゃっくりが止まった | 問題が解消した |
| 9. 水が不要になった | そのまま |
| 14. 男は驚いた | そのまま |
| 15. 意図を理解した | そのまま |

### 観察

- 鍵を抜くと各事実は自然に抽象化される（「しゃっくりに気づいた」→「男の状態に気づいた」）。この抽象化された形が中間命題の候補になる
- 要素A・Bへの振り分けは、鍵から各要素に至る因果上の位置で自然に決まる
- 「困っていた」（要素B側）と「男の状態に気づいた」（要素A側）の「状態」が両側で共通しており、合流点の候補

## 競合モデルからの抽象命題導出手順

### 手順

1. **要素の観測を特定する** — Sから、説明を要する観測を取り出す
2. **競合具象モデルの列挙** — その観測に対する具体的な説明ストーリーを複数列挙する（Tを含む）
3. **各モデルの具象原子命題への分解** — 各モデルを、それ以上分解できない事実の列に分解する
4. **Tの原子命題と各競合モデルの原子命題の間で、共通する抽象を見出す** — Tの特定の命題と、競合モデルの特定の命題を1対1で対にし、両者を包含する抽象命題を導出する。これが導出の自然なステップの候補になる

### バーの男・要素A（銃を向けた）での試行

観測: バーテンダーが男に銃を向けた

競合具象モデル:
- 強盗: バーテンダーが男から金を奪おうとした
- 自衛: 男が脅威的な行動をし、バーテンダーが身を守った
- 排除: 迷惑な男を店から追い出そうとした
- 対処（T）: 男の問題に対処しようとした
- 遊戯: 冗談や悪ふざけとして

Tと競合モデルの間で見出された抽象:

| Tの原子命題 | 競合モデルの原子命題 | 共通する抽象 |
|---|---|---|
| バーテンダーがしゃっくりに気づいた | 自衛: バーテンダーは身の危険を感じた | バーテンダーは男について何かを認識した |
| しゃっくりは外見・音で認識可能な症状である | 排除: 男は迷惑行為をしていた | 男の状態は外から認識可能だった |
| 驚かせて止めようとした | 排除: 銃で強制的に追い出そうとした | 銃を向けることで何かを達成しようとした |
| 驚かせることはしゃっくりの一般的な対処法である | 強盗: 銃での脅しは強盗の手段である | 銃を向けることがその目的の手段として成立する |
| バーテンダーの意図は男を助けることだった | 遊戯: 場を盛り上げようとした | バーテンダーに悪意はなかった |
