# v2 アルゴリズム初期フェーズの方向性

## 核心的アイデア

汎用的な探索戦略を S と T に体系的に適用し、T に至る認知的に自然な経路を可能な限り列挙する。この経路の集合が後続の全ての基盤となる。

## v1 との対比

- **v1**: 記述素空間を展開 → パラダイム発見 → 質問設計（発散→収束）
- **v2 案**: 汎用戦略 × (S, T) → 経路の列挙 → そこから構造を導出（目的先行）

## 汎用戦略（暫定リスト）

- 状況の絞り込み（場所、時代、人数など）
- 登場人物の探索（他の関係者の存在、関係性）
- 時間軸の確認（過去の関与、時系列）
- 常識的仮説の排除（毒、病気、事故など）
- 動機の探索（意図、理由）
- 異常性の特定（なぜ通常と異なるか）

戦略の体系はパズル横断的な理論の一部であり、個別パズルに依存しない。

## ウミガメのスープでの適用例

### 経路 A: 状況の絞り込み
男の職業は？ → 船乗り → 海で何かあった？ → 遭難した → 食料は？ → 尽きた → 仲間は？ → 死んだ → 何を食べた？ → 人肉を「ウミガメのスープ」と偽った → 本物を飲んで気づいた

### 経路 B: 登場人物の探索
他に関係する人物がいるか？ → いる、過去の仲間 → 仲間に何があった？ → 一人が死んだ → なぜ仲間の話がスープに関係する？ → 仲間の肉でスープを作った → 「ウミガメのスープ」と言われた → 本物との味の違いで気づいた

### 経路 C: 時間軸の確認
過去が関係するか？ → はい → いつの出来事？ → 遭難中 → 遭難中に何があった？ → 食料が尽き仲間が死んだ → スープとどう関係する？ → 人肉で作ったスープを飲まされた → 今回本物を飲んで悟った

### 観察

- 各経路は異なる戦略を起点とするため、経由する中間地点が異なる
- 骨格（論理的最短経路）の並べ替えではなく、戦略が異なることで質的に異なる経路が生まれる
- 経路の集合が、プレイヤーがゲーム中にたどりうる探索空間を構成する

## 骨格との関係

先に検討した論理的骨格（S → T の最小論理連鎖）と、戦略駆動の経路は異なるものである。

- **骨格**: 「はい」の回答のみで T に至る最短の論理連鎖。到達すべきピースの一覧
- **戦略駆動経路**: 汎用戦略による探索の結果としての認知的経路。骨格のピースだけでなく文脈的要素（船乗り、遭難など）を含む

骨格のピースは戦略駆動経路の中に散在しており、複数の経路を通じて順不同に拾われていく。パラダイムは骨格に直接対応するのではなく、拾ったピースの組み合わせが生む解釈の枠組みとして現れる。

## 今後の検討事項

- 汎用戦略の体系化（分類の精緻化、網羅性の確認）
- 戦略適用の手順の形式化（アルゴリズムとしての記述）
- 経路の集合からパラダイム・質問群をどう導出するか
- 他のパズルへの適用による検証
