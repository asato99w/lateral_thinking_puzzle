# 最終命題とクリア条件の再設計

## 経緯

バーの男の v3 サンプル生成時に、導出グラフの頂点が「T」（非形式的なストーリー全体）になっている問題が判明した。T は命題として形式化されておらず、導出グラフのノードとして不適切。

## 問題

- 導出グラフは全ノードが形式的な命題であるべきだが、頂点が「T」のまま
- T から直接演繹されるノード（P5, NT1, P1, P6）は互いに無関係にぶら下がり、導出構造が断片的になる
- クリア条件（clear_conditions）は T 事実（TF*）で定義されているが、これも導出グラフの命題体系とは別の枠組みになっている

## 解決策: 最終命題の導入

### 最終命題とは

T の情報を形式的な命題として表現したもの。導出グラフの頂点に位置し、そこから全分割命題が演繹される。T そのものではなく、T から必要十分な情報を抽出した命題。

### 設計方針

- 最終命題は**1つの命題**として定義する
- この命題が導出グラフの唯一の頂点となる
- この命題から全分割命題への演繹経路が存在すること
- true_models の非自明分割は要求しない（{G1} のみ = T 近傍でも可）

### クリア条件との統合

clear_conditions を最終命題で置き換える:

```json
"clear_conditions": [["FP"]]
```

- 集合族の要素が1つ、その集合の要素も1つ
- 最終命題の確認 = 全分割命題が導出済み = T に到達

### 導出グラフの構造

変更前（T が非形式的な頂点）:
```
      T
    / | \ \
  P5 NT1 P1 P6
   \  |\
    P4 P2 P3
```

変更後（FP が形式的な頂点）:
```
      FP
     / | \  ...
   P5 NT1 P1 P6
    \  |\
     P4 P2 P3
```

FP の具体的な内容と演繹経路はパズルごとに設計する。

## 対応方針

- **導出構造**: Phase 2 の導出関係分析で最終命題を定義し、頂点に据える（今回対応）
- **クリア条件**: clear_conditions を `[["FP"]]` に変更する（今回対応）
- **アルゴリズムの構造改訂**: Phase 2 に「最終命題の定義」ステップを正式に追加する（別途対応）

## バーの男での適用

最終命題の候補:

「男はしゃっくりが止まらなくて困っており、水を飲めば治ると思いバーに立ち寄ったが、バーテンダーはしゃっくりに気づき驚かせて止めてやろうと銃を向けた」

- → P5（身体的反射症状）: 「しゃっくり」を「身体的反射症状」に抽象化
- → NT1（症状に気づき解消しようとして銃）: 「しゃっくりに気づき驚かせて止めてやろうと銃を向けた」から「しゃっくり」の具体名を落とす
- → P1（水を飲みたくてバーに来た）: バーテンダーの行為を落とし、男の目的だけ残す
- → P6（初対面）: 文脈から（この点は FP に明示するか、別途検討が必要）

true_models = {G1}（P5 と同一の分割。最終命題として許容）
