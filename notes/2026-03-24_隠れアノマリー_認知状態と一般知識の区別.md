# 隠れアノマリー: 認知状態と一般知識の区別

手順案（`2026-03-24_隠れアノマリー分解_手順案.md`）の適用過程で発見された根本的な問題の整理。

## 発見された問題

毒キノコへの手順適用において、登場人物の認知的推論ステップを「一般知識で完結」と括り、モデル化の対象から外してしまった。これにより毒キノコとウミガメのスープが構造的に異なるものに見えていたが、実際には同じ構造を持つ。

## 一般知識と認知状態の推移の区別

### 一般知識（推論の道具）

- 自然の森では食用と毒が共存する
- キノコを採るには人がその場にいる必要がある
- 同じ料理は同じ味がする

これらは推論の道具であり、パズルの構造に関わらず存在する。

### 登場人物の認知状態の推移（具象的・T由来）

一般知識を用いて登場人物が実際に辿った推論は、Tの中で暗黙的に語られる物語固有の具象的事実である。

**毒キノコ:**
- 男は辺りに毒キノコしかないのを見た
- 男は食用キノコがないことに気づいた
- 男はそこから誰かが採ったと推論した
- 男はそこから近くに人がいると結論づけた

**ウミガメのスープ:**
- 男はスープを飲んで味の違いに気づいた
- 男は過去の食事の正体を悟った

いずれもTが暗黙的に語る具象的な認知の推移であり、ゲーム中にモデル化すべき対象。

### 誤り: 毒キノコの認知的推論を「一般知識で完結」と判断した

毒キノコの推論ステップ（食用がない → 誰かが採った → 人がいる）を「一般知識で完結するのでモデル化不要」としたのは誤り。推論の道具が一般知識であっても、推論が行われたこと自体はT由来の具象的事実であり、モデル化の対象。

## プレイヤーの認知との関係

登場人物の認知状態の推移がプレイヤーにとって意味を持つには、プレイヤー側に前提が必要。

**例: 毒キノコ**

「男は食用キノコがないことに気づいた」がプレイヤーにとって意味を持つためには:
- プレイヤーが「男は森にいる」を知っている必要がある
- これがなければ「食用キノコがない」は無意味な情報

### 二重の前提関係

背景事実は:
1. **登場人物の認知の前提**: 登場人物がその推論を行うために必要な状況
2. **プレイヤーの認知の前提**: プレイヤーにとってその推論が意味を持つために必要な情報

この二重の前提関係がfc設計の根拠となる。

「男は食用キノコがないことに気づいた」のfc には「男は森にいる」が含まれるべきであり、これはプレイヤーがその情報の意味を理解するための前提。

## 構造の統一

この区別を正しく適用すると、ウミガメのスープと毒キノコは同じ構造を持つ:

- **背景事実**がある（T由来）
- **登場人物の認知状態の推移**がある（T由来、一般知識を道具として使用）
- **背景事実はプレイヤーにとっての認知的前提**でもある

両者の違いは構造の違いではなく、認知的推論の道具が物語固有の知識か一般知識かの違いに過ぎない。

## アノマリー解消の認知的過程

### 現行の問題

現行のアルゴリズムでは、背景事実の観測後にアノマリー解消が起きるとしていたが、アノマリーから解消結果まで一気に飛んでいた。この間を埋める認知的過程を明示的にモデル化する必要がある。

### 具体例

**毒キノコ:**
```
食用キノコがない（アノマリー）
→ 自然の森なのに食用がないのはおかしい（異常の認識）
→ 食用キノコだけが選択的にない（精緻化）
→ 誰かが食用キノコを採った（推論）
→ 近くに人が来ていた（解消）
```

**ウミガメのスープ:**
```
以前に飲んだ時と味が違うと感じた（アノマリー）
→ 今のは本物だと確認された
→ 過去に飲んだものが本物ではなかった可能性
→ 過去に飲んだものはウミガメのスープではなかった（解消）
```

### 語用論的含意の検証はここにも適用

認知的過程の各ステップの表現にも語用論的含意の検証が必要。

- ○ 「以前に飲んだ時と味が違うと感じた」: 過去に何を飲んだかを特定しない。含意なし
- ✗ 「ウミガメのスープだと言われて飲んだものと味が違った」: 「と言われて」が棄却前提の否定を含意

### 粒度の注意

この過程は**細かすぎるとくどくなる**。ゲームとして適切な粒度を選ぶ必要があり、定量的に微妙な判断が求められる。機械的に全ステップを命題化するのではなく、ゲーム体験として自然な粒度を判断する必要がある。

## 手順案への影響

### 新規ステップの追加

背景事実の収集の後に「アノマリー解消の認知的過程の列挙」を追加する。各ステップについて:
- 具象的事実として記述
- 使用される一般知識を明示
- 認知的前提を特定（fc根拠）
- 語用論的含意を検証
- ゲームとして適切な粒度を判断

### 既存ステップへの影響

- 「T由来の前提命題なし、一般知識で完結」という判断を避ける
- 背景事実間のfcだけでなく、認知的過程の各ステップの前提関係もfcとして考慮する

## 実行者の課題

Claude（実行者）は一般知識と具象的認知状態の推移を混同しやすい。手順の記述において「一般知識で完結」という判断を安易に行わないよう、以下の問いを挟む必要がある:

- この推論を登場人物が実際に行ったか → 行ったならT由来の具象的事実
- この情報がプレイヤーにとって意味を持つための前提は何か → fcの根拠
- この粒度はゲーム体験として適切か → 細かすぎないか、飛びすぎていないか
