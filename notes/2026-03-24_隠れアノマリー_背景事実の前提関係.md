# 隠れアノマリー: 背景事実の前提関係

## 前提関係の基準

前提関係は時系列ではなく、**プレイヤーにとってその命題が意味を持つかどうか**で判断する。

時系列的に後であっても前提関係がない場合がある。逆に時系列的に前でも前提関係がない場合がある。

## 重要な判例: 「仲間が死んだ」の前提

「仲間が死んだ」の前提を「遭難した」にすべきではない。

理由: 仲間が死んだという事実は、遭難の文脈がなくてもプレイヤーにとって意味を持つ。「仲間が死んだ」は独立に理解可能な事実であり、遭難は死の原因の一つに過ぎない。前提関係として遭難を要求すると、「仲間が死んだ」という情報へのアクセスが不必要に制限される。

「仲間がいた」は前提として必要（仲間がいなければ仲間は死ねない）。

## Tの原子命題（ウミガメのスープ）

### 命題の記述規則

- 各命題は単独で意味が確定する自己完結的な形式とする
- 文脈依存の表現（「それ」「その」等）は使わない
- 主語・対象・相手を明示する
- 「スープを〜と言った」のように、誰に対してかが不明な表現は不可

### 原子命題と前提関係

1. 男は船乗りだった ← {}
2. 男は仲間たちと船で航海していた ← {}
3. 男の乗った船が航海中に遭難した ← {2}
4. 男たちの食料が尽きた ← {}
5. 男の仲間の一人が死んだ ← {2}
6. 男の仲間がスープを作った ← {2}
7. スープは死んだ仲間の肉で作られていた **（鍵 — 分離）**
8. 男の仲間は、作ったスープを男にウミガメのスープだと言って出した ← {6}
9. 男は仲間が作ったスープを飲んだ ← {6}
10. 男はレストランのスープを飲んだ時、以前に飲んだスープと味が違うと感じた **（認知的体験）** ← {9} + S

### 前提関係の判断根拠

- 3 ← {2}: 航海していなければ遭難は意味を持たない
- 4 ← {}: 食料が尽きたこと自体は独立に意味を持つ
- 5 ← {2}: 仲間がいなければ仲間の死は意味を持たない。遭難は前提としない（重要判例）
- 6 ← {2}: 仲間がいなければ仲間がスープを作ったは意味を持たない
- 8 ← {6}: スープが作られていなければ、ウミガメのスープだと言って出すことは意味を持たない
- 9 ← {6}: スープが作られていなければ飲むことは意味を持たない
- 10 ← {9} + S: スープを飲んだ事実 + 現在のスープ（S）がなければ味の比較は意味を持たない

### 前提関係の原則

- 時系列を前提関係と混同しない
- ある命題がプレイヤーにとって意味を持つために、別の命題を知っている必要があるかで判断する
- 因果関係（AがBの原因）と前提関係（AがなければBが意味を持たない）を混同しない

## 構造の整理

### 三つの関係

1. **導出元集合**: 主動因命題がそこから導出される命題群。主動因命題の直接の導出元であり、「前提」ではない
2. **前提**: 各命題が意味を持つために必要な命題。意味の成立条件
3. **導出元部分集合**: 導出の十分条件ではないが、導出の足しになるもの

### ウミガメのスープでの適用

**主動因命題の導出元集合**: {4, 5, 9, アノマリー解消}

**各命題の前提と導出元部分集合**:

| 命題 | 前提 | 導出元部分集合 |
|---|---|---|

| 5: 男の仲間の一人が死んだ | {2} | 3（遭難を知れば死に至りやすい） |
| 9: 男は仲間が作ったスープを飲んだ | {6} | |

### 戦術連鎖への接続

#### 導出元部分集合の定義の明確化

導出元部分集合は独立した関係ではなく、導出元集合族の要素の集合の部分集合である。導出元集合族の構造:

- 導出元 = 集合族 `{{set1}, {set2}, ...}`
- 各要素の集合（例: `{アノマリー解消, 4, 5, 9}`）の中の全てが揃う必要がある（必要条件）
- 集合族としては、いずれかの集合が充足すれば導出される（OR関係）
- 導出元部分集合 = ある要素集合の部分集合（例: `{4, 5}`）。全体が揃わなくても途中まで寄与する

#### 戦術連鎖での活かし方

戦術連鎖で中間命題を導入する際、導出元部分集合と**協力関係**にあるものとして設計する。

- 導出元部分集合だけでは導出に足りない
- 中間命題が加わることで導出に近づく
- 中間命題 + 導出元部分集合 → 導出元集合の充足に向かう

設計指針:
- Phase 1 で導出元部分集合を把握しておく
- Phase 2（戦術連鎖）で、導出元部分集合と協力して導出を進める中間命題を設計する

現行の戦術連鎖規則（`rules/戦術連鎖_v6.md`）がこの構造をどう受け取るかは未検討。戦術連鎖規則の更新が必要になる可能性がある。
