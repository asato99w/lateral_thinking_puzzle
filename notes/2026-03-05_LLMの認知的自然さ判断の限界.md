# LLM の認知的自然さ判断の限界

## 日付
2026-03-05

## 発見の経緯

turtle_soup 03 の Q-2（「男は過去に危機的な体験をしましたか？」）の recall_conditions 再設計において、LLM が認知的自然さを自力で判断・設計することの限界が明確になった。

## 具体的な問題

### 1. 認知的妥当性の判断ができない

Q-2 の recall_conditions が [["D-5"]]（男は自殺した）のみの状態で、これが不自然であることを LLM は判断できなかった。ユーザーに指摘されるまで「自然」と評価していた。

### 2. 段階的な思考の流れを設計できない

ユーザーが示した船の例:
- 「職業が関係あるか」→「乗り物に乗る職業か」→「船乗りか」

この段階的絞り込みの自然な流れを、自殺の原因について同様に構成することを求められたが、できなかった。

LLM が出力したもの:
- 「自殺の原因は何か」→「レストランが関係」→「スープが関係」→「味が引き金」→「過去の記憶」→「過去が原因」

これは機械的なステップの列挙であり、プレイヤーが実際にどう考えるかという認知的なシミュレーションになっていない。

### 3. 「言われたことだけ」になる傾向

ユーザーが recall_conditions の具体例（X-1: 船、X-2: 過去が原因）を示した後、LLM はそれをそのまま使用するだけで、同等の思考を他の要素に対して自力で適用できなかった。

## 影響範囲

### Phase 4: 戦術連鎖
- 戦術連鎖の設計において、各ステップの「認知的自然さ」を LLM が自力で担保できない
- 戦術の形式的な適用はできるが、プレイヤーの実際の思考過程に沿った連鎖設計ができない

### Phase 7: 検査
- 認知的自然さの検査（レイヤー 1〜3）を LLM が正確に実行できない
- 特にレイヤー 3（想起の認知的妥当性）は構造的チェックでは補えない認知的判断を要する
- 対策として Step 3（方向性漏洩の構造的チェック）を追加したが、これは認知的判断の代替ではなく応急処置

## 根本的な課題

LLM は「形式的に正しい戦術連鎖」は構成できるが、「プレイヤーが自然にたどる思考の流れ」を再現できない。これは以下の違いに起因する:

- **形式的正しさ**: 入力記述素 → 戦術 → 出力記述素の整合性
- **認知的自然さ**: その入力を持つプレイヤーがなぜその方向に思考を向けるか

後者は人間の認知プロセスのシミュレーションであり、LLM の現在の能力では信頼性が低い。

## 対処の方向性（未決）

- 工程2（検査・調整）での認知的妥当性チェックに構造的手法をどこまで適用できるか
- 人間によるレビューをどの工程に組み込むか
- 戦術連鎖の設計フェーズ（Phase 4）で認知的自然さを担保する仕組み
