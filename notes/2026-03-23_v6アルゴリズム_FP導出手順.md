# v6アルゴリズム：FP導出手順

v6検討ノート（2026-03-22）の続き。バーの男・要素Aでの試行を踏まえた手順の整理。

## 前提

- Sに2つの要素（要素A、要素B）が含まれる（2要素特性）
- Tが既知
- 鍵が特定されている

## 手順

### Step 1：要素を導出するTの原子命題の特定

各要素について、それを導出するのに必要十分なTの原子命題を求める。

規則：
- 一般論的事実（常識として解き手が持つ知識）は含めない
- 物語固有の事実のみを原子命題とする

**バーの男・要素Aでの結果：**

要素A：バーテンダーが男に銃を向けた
鍵：しゃっくり

原子命題：
1. 男はしゃっくりが止まらなかった
2. バーテンダーはしゃっくりに気づいた
3. バーテンダーは驚かせて止めようとした

### Step 2：原子命題から鍵を省いてFPを構成

原子命題群から鍵に関する具体的情報を省き、
1つの命題（FP）にまとめる。

規則：
- FPから元の原子命題群（鍵なし版）が導出可能であること
- 可能な限り具象的で意味のある命題とすること

**バーの男・要素Aでの結果：**

原子命題（鍵なし）：
1. 男はある問題を抱えていた
2. バーテンダーはその問題に気づいた
3. 銃を向けることがその問題への対処だった

FP1：「バーテンダーは男の異変に気づき、その対処として銃を向けた」

導出の検証：
- → 男に異変があった（1を導出）
- → バーテンダーがそれに気づいた（2を導出）
- → 銃を向けることが対処だった（3を導出）

### Step 3：各要素の競合具象モデルの原子命題を特定

各要素について競合する説明モデルを列挙し、
それぞれを同様に原子命題に分解する。

規則：
- 一般論的事実は含めない
- 要素を導出するのに必要十分な命題のみ

**バーの男・要素Aでの結果：**

| モデル | 原子命題 |
|---|---|
| T（対処） | 1. しゃっくりが止まらなかった 2. 気づいた 3. 驚かせて止めようとした |
| 強盗 | 1. 男の金を奪おうとした |
| 自衛 | 1. 男が脅威的行動をした 2. 身を守ろうとした |
| 排除 | 1. 男が迷惑行為をしていた 2. 追い出そうとした |
| 遊戯 | 1. 冗談として銃を向けた |

### Step 4：T原子命題と競合モデル原子命題の共通抽象を求める

T原子命題と各競合モデルの対応する原子命題の間で、
共通する抽象命題を求める。

規則：
- 可能な限り具象的で意味のある命題を求める（生成ステップ）
- 事後に以下の基準で評価し間引く（評価ステップ）：
  - 客観性：主観や解釈に依存しないか
  - 意味の確定性：「何か」「何らかの」を含んでいないか
  - 非トートロジー性：情報量があるか
  - 弁別力：競合モデル間を区別する力があるか

意味のある共通が見出せなかった場合、
そのT命題はTに固有であると記録する。

## 全体の流れ

```
S → 要素A, 要素Bを特定
T → 各要素を導出する原子命題を特定（Step 1）
   → 鍵を省いてFPを構成（Step 2）
   → 競合モデルの原子命題を特定（Step 3）
   → 共通抽象を求め評価（Step 4）
```
