# 工程3 適用フィードバック: ウミガメのスープ

対象: `app/poc_v3/samples/turtle_soup/02_2026-03-08/`

## Phase 10: 認知的断層分析

### フィードバック 1: 断層γ「言葉の二重性」は不適切

初回の断層分析で γ を「言葉の二重性」（「ウミガメのスープ」が2つの意味を持つ）としたが、これは不適切。

**問題**: 「言葉の二重性」は言語的トリックとしてフレーミングしている。実際のパズルの構造は:
- 仲間が意図的に「ウミガメのスープだ」と嘘をついた（欺瞞行為）
- 本物の味との違いで欺瞞が発覚した（発覚のメカニズム）
- 食べたものの正体（人肉）を悟った（真相の理解）

本質は「ラベルの偽装」「欺瞞とその発覚」であり、言葉遊びではない。

**影響の仮説**: この誤フレーミングが工程1の質問生成にも影響した可能性がある。「言葉の二重性」視点だと「本物のウミガメのスープを飲んだことがあるか」のような確認的質問に偏る。「欺瞞と発覚」視点なら「仲間に騙されたのか」「スープの正体は何だったのか」のような核心的質問になるはず。

### 検討事項: 入力への補足情報の追加

S, T に加えて補足（annotation）を入力とすることで、断層分析の精度を上げられる可能性。

補足の候補:
- T のどの部分が核心か（パズル設計者の意図）
- S が誘導する主要な誤解釈の方向
- このパズルの「驚き」の構造

→ 具体的な形式は工程3の適用を通じて検討する。

## 工程3 の設計方針

### フィードバック 2: T' の導出は「木の構造」に沿うべき

中間目標: T に対する代替理論 T', T'', ... を提示すること。

T を木の葉として、T に近い葉から遠い葉へと探索する方針は正しい。しかし「近い」の意味に注意が必要。

**問題**: T の命題を1つ変えた「隣接する理論」を構造なしに列挙するやり方は、木の上の葉から隣の空中に葉を探すようなもの。存在しうる葉を想像で生成しているだけで、木の構造（枝分かれ）に根拠を持たない。

**あるべき姿**: 木の枝分かれ（分岐点）に沿って隣の葉に到達すること。つまり:
- T に至る推論パスのどこで分岐が起きるかを特定する
- 分岐点から異なる方向に進んだ先に自然に存在する葉が T'
- T' は「空中に想像した理論」ではなく「推論の木の上に実在する葉」

これは、プレイヤーが実際にたどる途中で自然に分岐しうるポイントから導出される。

### フィードバック 3: 推論の木ではなく、観測の木

ゲームの進行は「推論」と「観測（質問による事実の確認）」の積み重ね。大きな流れを捉えるべきは推論ではなく**観測**のほう。

- プレイヤーは質問により事実を一つずつ確認していく（観測）
- 観測の蓄積が世界像を形成する
- 推論はその世界像の上で起きる二次的な活動

したがって木の構造は「推論の分岐」ではなく「観測の蓄積による分岐」として捉えるべき。ある事実が確認された時点で、その事実と整合する複数の世界像（T, T', T'', ...）が存在しうる。次の観測がどれかの世界像を排除する——この過程の構造が木。

### フィードバック 4: 構造の粒度——質問ではなく戦術連鎖・ピース

観測系列の分岐を見る際、個々の質問よりも上位の構造要素に着目すべき。

- **戦術連鎖**: 質問の背後にある「推論の方向性」。個々の質問は戦術連鎖のステップであり、分岐を見るには戦術連鎖の粒度が適切
- **ピース**: さらに上位の構造。ピースは情報の意味的まとまりであり、「どのピースまで開示された段階で、どの世界像が成り立つか」という捉え方ができる

具体的には:
- P-PAST（遭難経験）が開示された段階 → この事実群と整合する世界像は？
- P-PAST + P-SOUP の一部が開示された段階 → 世界像はどう絞られるか？
- P-TRUTH の開示で T に収束

ピース間の遷移が「大きな分岐点」、戦術連鎖のステップが「中間的な分岐点」、個々の質問が「最小の分岐点」。T' の探索はまず上位構造（ピース）から始め、必要に応じて戦術連鎖・質問レベルに降りていく。

### フィードバック 5: ピース分解の問題——P-SOUP と P-TRUTH の重複

T' のピースレベル分析を行うと、工程1のピース分解の問題が浮かび上がる。

**問題**: P-SOUP（欺瞞の経験側）と P-TRUTH（正体の真相側）が、同一の出来事（欺瞞とその発覚）を不自然に分断している。

- P-SOUP: D-8（味が違った）、D-11（本物を飲んだことがない）、D-12（信じていた）
- P-TRUTH: D-17（ウミガメではなかった）、D-5（人肉だった）

「ウミガメではなかった」と「では何だったのか」は一つの発覚の表裏。ピース境界をまたいでいるため:
- T' 分析で P-SOUP 分岐と P-TRUTH 分岐が曖昧になる
- 情報隔離が意味的に機能しにくい（P-SOUP の情報だけで「何か別の肉」という推測に十分到達する）

**示唆**: ピース分解は T' 分析の自然な単位であるべき。逆に言えば、T' 分析がピース分解の妥当性を検証する手段になりうる。工程3 は工程1 のピース設計へのフィードバックループとしても機能する可能性。

### フィードバック 6: 独立/依存の設計原則——S との連結による階層化

ピースの独立/依存の分類は、S との連結の有無に基づくべき。

**原則**:
- S と連結するピース → 独立ピース
- S と連結しないピース → 依存ピース
- 一つのピース内に S 連結部分と非連結部分が混在する → 分割すべき

これは構造の**階層化**と捉えられる:
- 第1層: S（プレイヤーの出発点）
- 第2層: 独立ピース群（S から直接到達可能な情報のまとまり）
- 第3層: 依存ピース群（独立ピースを経由してのみ到達可能）
- ...（さらに深い依存があれば続く）

各層が T' 分析の自然な境界になる:
- 第3層で分岐 → T に最も近い T'（最深部のみ異なる）
- 第2層で分岐 → 中距離の T'
- 第1層（S の解釈）で分岐 → T から最も遠い T'

**現状の問題への適用**: P-SOUP は S と連結しうる要素（D-8←D-S2, D-6←D-S1）を含むにもかかわらず依存ピースとして扱われている。S 連結を認めるなら分割が必要。認めないなら、その設計判断の根拠を明示すべき。

**ピース分解への影響**: 独立/依存の判断がピースの境界を決め、ピースの境界が T' 分析の粒度を決める。この連鎖により、工程1 のピース設計と工程3 の T' 分析が整合的になる。

## ピース設計の再検討

### フィードバック 7: ピースは T の主要概念の抽象

ピースは真理論（T）を構成する主要な概念を抽象化したもの。依存ピースは複数の主要概念を結びつける（統合する）もの。

この定義により、ピース設計の方向性は「T をどのような主要概念に分割できるか」という分析になる。

### フィードバック 8: 因果を外した独立分割

T の主要概念を抽出する際、因果関係に囚われず独立したものとして分割すべき。

ウミガメのスープの T から抽出される独立概念:
1. **遭難** — 船で遭難した
2. **人肉食** — 人肉が食料として使われた
3. **欺瞞** — 「ウミガメのスープだ」と偽られた

これらは因果的には連鎖するが、概念としては独立に成立する（遭難なしの人肉食、人肉食なしの食品欺瞞はあり得る）。「発覚」はこれら3つと S を結びつけた結果であり、独立概念ではなく統合（依存）の役割。

### フィードバック 9: 抽象連鎖と T' 生成テンプレート

各具体概念は複数方向に抽象化できる（抽象連鎖）:
- 遭難 → 危機的状況 → 非日常 / 遭難 → 海での出来事 → ...
- 人肉食 → 禁忌の食 → ... / 人肉食 → 極限状態での行為 → ...
- 欺瞞 → 偽りのラベル → ... / 欺瞞 → 仲間の行為 → ...

低レベルの抽象を適切にまとめると、T のインスタンスを生成するテンプレートが得られる:

**「[危機的状況] で [禁忌の食] が [偽りのラベル] で提供された」**

各スロットを入れ替えると T' が導出される:
- [戦場] で [毒物] が [薬と偽って] → T'
- [監禁] で [禁忌の肉] が [通常の食事と偽って] → T''
- [遭難] で [人肉] が [ウミガメのスープと偽って] → T そのもの

→ 低レベル抽象のスロットが T' の生成器として機能する。ピースの独立概念がスロットに対応し、依存ピース（統合）がスロットの結合を担う。

## フェーズ2 の再設計: 抽象連鎖に基づく骨格構造

### フィードバック 10: 3層構造

T の骨格構造を3層で定義する:

```
上層: 抽象パターン群（T から生成される多数の抽象）
中間層: 抽象グルーピングの交差（骨格の構成単位）
下層: T（具象）
```

- **上層**: T の具体概念から複数方向に抽象化して得られるパターン群
- **中間層**: 上層の抽象をグルーピングし、その交差として現れる中間的な概念。現行の「ピース」に相当するが、因果分析ではなく抽象の親和性から導出される
- **下層**: T そのもの。中間層ノードの統合として得られる

中間層の分解の良し悪しは、**中間層ノード間の重複の大きさ**で評価できる。重複が大きい分解は概念の境界が不適切であることを示す（例: 旧 P-SOUP / P-TRUTH の「欺瞞の発覚」分断）。

### フィードバック 11: 集合族としての候補提示

中間層のグルーピングは一意に定まるとは限らない。複数の分け方を**集合族**として提示し、比較評価すべき。

上層の親和性クラスタ（T に依存しない自然なまとまり）:
- α: {危機的状況, サバイバル, 極限行為}
- β: {禁忌の食, 食人, 死者の利用}
- γ: {偽りのラベル, 情報の非対称, ラベルと実体の乖離}
- δ: {仲間の体験, 仲間の犠牲, 仲間の嘘}（横断的）

グルーピング候補:
- パターン I（α+β / γ）: 「極限状況での禁忌行為」と「認知的偽装」
- パターン II（α / β+γ）: 「極限状況」と「隠された禁忌」
- パターン III（α / β / γ）: 3分割（β-γ間に親和性あり）

各パターンの独立性と重複を比較し、最適な分解を選択する。

### フィードバック 12: 抽象レベルの調整——独立性を制約とした抽象度の最小化

同じ概念でも抽象レベルにより独立性が変わる:
- 低抽象（遭難 ↔ 人肉食）: 強く結合
- 中抽象（危機的状況 ↔ 禁忌の食）: まだ結合
- 高抽象（非日常体験 ↔ 隠された事実）: かなり独立

**指針: 独立性を失わない範囲で、抽象度をできるだけ下げる。**

高すぎる抽象はパズル固有性を失い（どのパズルにも当てはまる汎用概念になる）、低すぎると概念間の結合が強まり独立性が損なわれる。最適点はその間——**このパズル固有でありつつ独立性が高い**抽象レベルの組み合わせ。

この最適点の探索がフェーズ2の核心作業となる。各概念の抽象レベルは独立に調整可能なパラメータであり、組み合わせ空間の中から独立性最大・抽象度最小の解を見つける。

### フィードバック 13: パターンの定義——抽象レベル × グルーピング

パターンとは、**各構成概念の抽象レベル選択とグルーピングの組み合わせ**として定義される。

3つの構成概念がそれぞれ異なる抽象レベルを取りうる:

| | L1（低） | L2 | L3 | L4（高） |
|---|---------|----|----|---------|
| 遭難 | 海での遭難 | 遭難 | 危機的状況 | 非日常体験 |
| 人肉食 | 食人 | 禁忌の食 | 禁忌 | 隠された事実 |
| 欺瞞 | 偽りのラベル | 情報の非対称 | 認知の歪み | 見かけと実体の不一致 |

パターン = (各概念の抽象レベル) × (グルーピング)

例:
- (遭難L2, 禁忌L3, 偽りのラベルL1) × {遭難} / {禁忌, 偽りのラベル}
  → Node A「遭難」/ Node B「偽装された禁忌」
- (危機的状況L3, 禁忌の食L2, 偽りのラベルL1) × {危機的状況, 禁忌の食} / {偽りのラベル}
  → Node A「危機的状況下の禁忌の食」/ Node B「偽りのラベル」

結合が強い概念ペアへの対処は2つ:
1. **一方の抽象度を上げる** → 結合を切り、別グループに置く
2. **同じグループに入れる** → 結合をグループ内に封じ込める

集合族として複数パターンを提示し、独立性と抽象度のバランスを比較評価する。

### フィードバック 14: 抽象グルーピングの目的——交差による具象の導出

グルーピングの目的は、高レベルの抽象の交差として具象を自然に導出すること。

- 抽象を個別に見ると範囲が広い（「禁忌」は食に限らない、「偽りのラベル」は食品に限らない）
- 複数の抽象をグルーピングすると、交差により範囲が絞られ具象が現れる
- {禁忌, 偽りのラベル} → 交差: 「偽装された禁忌」→ このパズルでは人肉をウミガメのスープと偽った行為

これを直接 T から具象を定めるのでなく、3層構造で階層的に行う:
- 上層（抽象パターン群）→ グルーピング → 中間層（交差による半具象）→ 統合 → 下層（T）

各階層で交差が具象を絞り込み、最終的に T に到達する構成。

### フィードバック 15: 独立性の評価——T' 生成による操作的定義

独立性の判断を T' の生成と一体化することで、操作的かつ定量的な評価基準を得る。

**操作的定義**: グループ G1 と G2 が独立であるとは、以下の両方が成り立つこと:
1. G1 を固定し G2 を変動させた T' 群が自然に生成できる
2. G2 を固定し G1 を変動させた T' 群が自然に生成できる

**定量化の指標**:
- **変動容易性**: 各方向で生成できる T' の数
- **自然さ**: T' に無理矢理感がないか（生成の容易さ）
- **非対称性の検出**: G1→G2 方向と G2→G1 方向で容易さが異なれば、潜在的な依存関係を示唆

**適用例（パターン4: {遭難, 禁忌の食} / {情報の非対称}）**:
- G1 固定・G2 変動: 「遭難中に人肉を食べたが、**本人が記憶喪失で忘れていた**」「**日記に暗号で記録されていた**」→ 自然に生成可能
- G2 固定・G1 変動: 「情報の非対称があったが、**戦時中の食料配給で**」「**医療現場で**」→ 自然に生成可能
- → 両方向とも容易 = 高い独立性

**利点**:
- T' 導出とセットで扱うことで、独立性評価と T' 生成が相互に補強する
- パターン比較に使える（T' 生成の容易さでパターン間の優劣を判断）
- 「独立性が高い分解」＝「T' が豊かに生成できる分解」という直観的な対応が得られる
- **工程間の接続**: Phase 2 で独立性評価のために生成した T' 群が、そのまま後続のT探索フェーズの設計材料になる。プレイヤーが中間層パターンにマッチした段階で、T' 群が探索空間の分岐選択肢として機能し、そこから T への絞り込みを導く。独立性評価と T 探索設計が同一の T' 群を共有することで、工程間の一貫性が自然に保たれる

## パターン探索と戦術連鎖の関係

### フィードバック 16: 円錐モデル——幾何的イメージ

3つの主要概念をそれぞれ円錐として捉える幾何的モデル:

- **3つの円錐**: 各主要概念（遭難・人肉食・欺瞞）に対応
- **頂点が重なり T をなす**: 具象化するほど収束し、最終的に T で合流
- **パターン**: 各円錐のどの高さで断面を切るか（抽象レベルの選択）
- **底面**: S に近い側（広い、多くの T' が成立）
- **頂点**: T（狭い、収束）

正確には3次元では表現しきれない（各概念が独自の抽象軸を持つ）が、直観的なメンタルモデルとして有用。

### フィードバック 17: 戦術連鎖の位置づけ——S-T 間の橋渡し

パターン探索の進み方を2軸で捉える:
- **縦**: 抽象レベルの階層（上層→中間層→T）——先の考察で扱ったもの
- **横**: S と T の隔たり——戦術連鎖が橋渡しするもの

戦術連鎖は抽象度を上下する操作ではなく、S の世界像から T の世界像へ向かって隔たりを埋めていく操作。

### フィードバック 18: 戦術連鎖の定義と質問への具体化

**戦術連鎖**は以下で定義される:
- **起点**: 命題群（出発点として与えられる）
- **終点**: 命題群（到達目標として与えられる）
- **得られるもの**: その間を埋める戦術の列と中間生成命題群の集合

戦術連鎖は抽象的な経路設計であり、これを具体化（インスタンス化）すると質問列になる。

プレイヤーの体験としての具体化:
1. 起点の命題群が観測済みの状態
2. 質問により新たな命題を観測
3. 観測済み命題群から仮説導出が起きる
4. 導出された仮説が次の質問を想起させる
5. 繰り返しにより終点の命題群に到達

**パターンとの接続**: 戦術連鎖の起点と終点をどう定めるかが、パターン（抽象レベル×グルーピング）の設計と直結する。パターンが中間層の構造を決め、その構造が戦術連鎖の起点・終点を規定する。

### フィードバック 19: パターンと戦術連鎖の役割分担

**パターンの役割**:
- 各主要概念周辺の自由な探索空間を提供する
- それらの探索が一定レベルまで進んだとき、収束の構造（トリガー条件）を与える
- パターン＝収束トリガーは固定的なものとして扱える

**戦術連鎖の役割**:
- 各概念の探索空間内での局所的な動き方を記述する
- 固定経路ではなく、探索のレパートリー（使える動きの種類）を定める

これにより「発見（問題A）と降下（問題B）の分離問題」が解消する: 発見も降下も同じ「自由な探索」であり、パターンがチェックポイントとして「十分に探索が進んだか」を判定する。複数パターンを集合族として持つことで、異なる探索経路からでも収束が保証される。

### フィードバック 20: 分析アプローチ——縦と横の分離

探索の具体化にあたり、分析レベルでは縦と横を分離して扱う（方針B）:

1. **横の分析**: S → 各概念の最上位抽象への接続（Sのどの要素がどの概念への入口になるか）
2. **縦の分析**: 各概念内の抽象レベル間の接続（最上位から具象への降下構造）

プレイヤー体験への変換は後の工程に委ねる。分析と体験の分離は、現行アルゴリズムでも戦術連鎖（分析的構造）→ 質問生成（体験）という形で存在しており、方向として整合する。

### フィードバック 21: 抽象ツリー——抽象連鎖の改称と構造

「抽象連鎖」を「**抽象ツリー**」に改称する。一つの具象概念から複数の抽象方向に枝分かれするため、一本道の「連鎖」より分岐構造を表す「ツリー」が適切。

各主要概念の抽象ツリー:
- T（具象）を根として、上方に枝が広がる逆さの樹
- 上に行くほど枝が増え、各枝の先端（最上位抽象）は広い領域をカバーする
- 例: 遭難 → {危機的状況方向, 海での出来事方向, 仲間との体験方向, ...}

最上位抽象の「発見しやすさ」は2つの要因に支えられる:
1. **インスタンスの多さ**: 一つの抽象が多くの具象を含む（領域の広さ）
2. **枝の多さ**: 複数の抽象方向が存在する（入口の多さ）

Sからの横の探索は、この樹のどの枝にも当たりうる。枝が多く各枝の領域が広いほど、Sとの接点が見つかりやすい。

### フィードバック 22: 横の探索の本質——戦術による中間命題生成

#### 水平思考パズルの特性と直接接続の不可能性

Sには意図的に矛盾・緊張が仕込まれるが、それは直接的には解決できないように設計されている。したがって、Sの矛盾を分析して概念ツリーに直接接続しようとするアプローチは原理的に行き詰まる。

**誤ったアプローチ**: S の矛盾 → 概念への直接接続（Sの内容分析に固執）
**正しいアプローチ**: S の要素 → 戦術適用 → 中間命題生成 → 概念ツリーへの到達

#### 戦術の本質的役割: Sにない情報の引き出し

戦術はSと概念を橋渡しするものではなく、**Sに存在しない新たな命題（中間命題）を生成する**ための操作。生成された中間命題が概念ツリーへの入口を開く。接続はSの分析からではなく、戦術の適用結果から事後的に生まれる。

具体例:
- S の中の平凡な要素:「男がいる」
- 戦術: 登場人物の深掘り
- 中間命題:「男は船乗りだ」（Sには存在しない新情報）
- → 遭難 → 危機的状況 → 非日常体験（概念ツリーへの入口が開く）

注目すべき点: 入口はSの矛盾（スープ→自殺）からではなく、Sの何でもない要素（「男がいる」）から戦術を通じて開かれる。

#### 横の探索 = 中間命題生成の連なり

戦術連鎖は中間命題の連鎖的な生成として捉えられる:
1. Sの要素（または既観測命題）に戦術を適用
2. 中間命題が生成される
3. 中間命題が次の戦術の入力になる
4. さらなる中間命題が生成される
5. この積み重ねで概念ツリーへの到達が実現する

横の探索とは、この中間命題生成の連なりそのもの。

#### 設計上のニュアンス: 戦術が主体、Sは材料

形式的にはプレイヤーの認知状態は S＋観測済み命題で表現されるが、設計の視点としては:
- 「Sから何に繋がるか」ではなく「どの戦術が適用可能で、何を生むか」から考える
- 戦術が探索の主体であり、Sや観測済み命題はその入力材料
- Sは利用可能な要素の一つであり、出発点として特権的な位置を占めない

このニュアンスはアルゴリズムの構成順序に影響する: 行き先（概念ツリー上部）と戦術の組み合わせから接続を見出していくアプローチが適切。

### フィードバック 23: Sの細分化と戦術連鎖による横の探索

#### Sの原子的分割

Sを可能な限り細かい命題に分割することで、各要素が戦術の入力材料として独立に使えるようになる。

S:「ある男がレストランで「ウミガメのスープ」を注文した。スープを一口飲んだ男は、それ以上スープを飲むことなく店を出た。そして男は自殺した。なぜか？」

分割:
- 男がいる
- レストランがある
- 男はレストランにいる
- 「ウミガメのスープ」というメニューが存在する
- 男はウミガメのスープを注文した（自発的行為）
- スープが提供された
- 男はスープを飲んだ
- 一口だけ飲んだ
- それ以上飲まなかった（飲むのをやめた）
- 男は店を出た
- 男は自殺した

各命題はSの矛盾を持たない単純な事実であり、戦術の素材として機能する。

#### 分割→戦術連鎖による概念ツリーへの到達

Sの原子的命題から戦術を適用し、中間命題の連鎖として概念ツリーの上位抽象に到達する。

具体例:
- 「男がいる」→ 登場人物の深掘り →「男は船乗りだ」→ 遭難ツリー
- 「ウミガメのスープを注文した」→ 動機の深掘り →「過去にウミガメのスープを飲んだことがある」→ 過去の体験 → 遭難ツリー
- 「スープが提供された」→ 対象の深掘り →「スープの材料は何か」→ 禁忌の食ツリー

方針: Sの分割から戦術連鎖として横の探索を構成する。各連鎖は原子的S命題を起点に、中間命題を経て概念ツリーの枝に到達する経路を示す。
