# 戦術連鎖の目的再定義

## 日付
2026-03-07

## 経緯

バーの男を v2 アルゴリズムで生成した際、Phase 3（戦術連鎖）が中間記述素を 1 つも導入しなかった。既存の記述素同士を戦術ラベルで繋いだだけで、フェーズとしてほぼ何も成していなかった。

## 問題の分析

### 砂漠の男との対比

砂漠の男の P-BAL 戦術連鎖（ノート 2026-03-06）では、連鎖構築過程で 7 つの中間記述素（D-15, D-16, D-17, D-32, D-33, D-34, D-35）が導入されている。これらの記述素こそが「連鎖」を連鎖たらしめている。

バーの男では中間記述素が 0 個。例:
- `{D-S3, D-S4} →[矛盾の指摘]→ 「銃を向けたことが有益だった」→ D-7`
- 「銃を向けたことが有益だった」は非形式的なテキストで、正式な記述素として導入されていない
- 記述素の連鎖ではなく、テキストラベルの連鎖になっている

### 規則の問題

`戦術連鎖v2.md` の Step 2:
> "末端記述素に対して、起点記述素集合からその記述素に至るまでの汎用戦術の列を定義する"

焦点が「戦術の列を定義する」に向いている。しかし戦術連鎖の意義は「適切な中間記述素の導入」に尽きる。戦術は「どこに記述素が必要か」を発見するための分析手法であり、出力は戦術ラベルではなく記述素群である。

### 工程3ノートとの一致

2026-03-05 ノート point 6:
> 対立戦術連鎖は「対立仮説の記述素に戦術ラベルを貼る」ような機械的構成物ではない

正解連鎖でもまったく同じ。

## 改善

### 戦術連鎖v2.md の改訂

1. フェーズの目的を「中間記述素の導入」に再定義
2. 出力に「Phase 3 で導入された中間記述素群」を追加
3. 「中間記述素の導入原則」セクションを追加
4. 各ステップに記述素の形式化を義務化
5. 規則に「中間記述素を導入しない出力は不完全」を追加

### 統合アルゴリズムv2.md の改訂

Phase 3 の説明を更新し、中間記述素の導入が主目的であることを明記。

### Phase 2 との関係

Phase 2（構造設計）はトリガーレベルの抽象記述素を導入する。Phase 3 は連鎖内部の中間記述素を導入する。両フェーズとも記述素の導入が主出力であり、相補的。

- Phase 2 の記述素: ピースへの入口（トリガー条件）
- Phase 3 の記述素: 連鎖内の通過点（プレイヤーの推論ステップ）
