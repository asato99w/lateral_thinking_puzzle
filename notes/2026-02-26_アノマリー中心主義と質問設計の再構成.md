# アノマリー中心主義と質問設計の再構成

## 背景

P_pred の全域関数化、固有記述素・共有記述素・偶然の開き・anomaly到達性充足の廃止を経て、質問設計の検証観点を再整理する。

統合アルゴリズム自体の構造（4フェーズ＋反復ループ）は維持し、個別ルールの修正で対応する。

## 以前との大きな違い

特定のパラダイムにいるとき、そのパラダイム外のことを考える必要があまりなくなった。固有・共有などの記述素の分類は不要。シフトの関係性はシフトの検証として別フェーズで扱えばよい。

## 質問設計の2段構成

### 第1段: パラダイム内のアノマリー発掘の流れ

各パラダイム内で、質問を通じてアノマリーが適切に発掘されるかを検証する。

検証観点:

1. **完全性**: 全質問がオープン・回答された時点で、そのパラダイムの全アノマリーが O に入ること（必須条件）
2. **段階性**: 浅い層で一気にアノマリーが集中しないこと（品質条件）
3. **横方向探索**: 初期オープンから同化連鎖を通じて、質問空間が適切に広がること

横方向探索は同化メカニズムによる。回答 → 同化で H が伝播 → 新たな質問がオープン、という連鎖がパラダイム内の探索を構成する。初期オープン質問はこの探索の起点であり、アノマリーへの横方向の到達を可能にする観点から設定される。

### 第2段: アノマリー発掘による遷移

蓄積されたアノマリーが適切なパラダイムシフトを引き起こすかを検証する。

第1段が保証されれば、十分な質問回答によりシフト条件（tension strict `<` + resolve >= N + 近傍）が満たされる。段階性がなければ浅い層で一気にシフトが起き、深い層の質問が活かされない。

## アノマリー中心主義

質問設計の全てがアノマリーの発掘を中心に組み立てられる。

- 質問の存在意義 → アノマリーを O に入れること
- 層構造 → アノマリーの発掘順序を制御すること
- 同化・横方向探索 → アノマリーへの到達経路を開くこと
- 初期オープン → アノマリー探索の起点を設定すること
- シフト → 蓄積されたアノマリーの帰結

以前は質問自体の分類（固有/共有/偶然の開き等）が多かったが、それらを廃止した結果、アノマリーという一点から全てが導出される構造になった。

## 統合アルゴリズムへの反映方針

統合アルゴリズムの4フェーズ構造は維持し、以下の個別ルールを修正する:

- **質問構造設計**: アノマリー中心の3観点（完全性・段階性・横方向探索）で再構成
- **補完検証**: 同上の観点に基づく検証に改定
- **不足ルーティング**: 新しい検証観点に合わせた分類に改定

シフト方向検証は第2段として独立性を保つ。
