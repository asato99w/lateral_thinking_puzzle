# 導出構造の再整理

## 経緯

砂漠の男（desert_man 02）を v3 アルゴリズムで生成したところ、導出グラフの構造が機能していないことが判明した。可視化で確認した結果、問題は可視化側の形式的問題ではなく、アルゴリズムから導かれた構造自体の問題であると切り分けた。

## 生成された分割命題

| ID | ラベル | true_models |
|----|--------|-------------|
| P1 | 男は乗り物に乗っていた | T, M3, M7 |
| P2 | 男は空中にいた | T, M7 |
| P3 | マッチ棒は物事を決めるために使われた | T, M1, M5, M7, M8, M9 |
| P4 | 男は複数人のグループの中にいた | T, M1, G1, M5, M6, M7, M8, M9 |
| P5 | グループの中で誰かが命を落とす決定がなされた | T, M5 |
| P6 | 乗り物の重量を減らす必要があった | T |

## 生成された導出グラフ（問題のある構造）

```
P2 → P1    P3 → P4
P6 → P1    P5 → P4
```

- 2層4辺のみで導出関係が薄すぎる
- P1/P4 系の2ツリーが完全に分離しており、間の関係がない
- T の中ではこれらは無関係ではないのに、構造に反映されていない

## 問題の原因分析

### 1. 生成時のミス

アルゴリズムの Step 3e「導出ギャップの補足」をスキップした。

### 2. アルゴリズムの構造的問題

- Step 3a〜3d は「Step 2 で生成された分割命題群に対して、概念的な導出関係を分析する」— 既存命題間の関係を見つけるだけで、新しい命題は生まれない
- T からの演繹分析は Step 3e で「補助的に用いる」とされているが、導出グラフの深さを作る本質的な工程である
- 補足ではなく中心に据えるべき

### 3. 導出の方向の不明確さ

アルゴリズムは「A の確認が B を不要にする」（具象→抽象）と定義しているが、「T から出発して演繹構造を構築する」手順が明示されていない。結果として、モデル横断比較で独立に抽出した命題を後から繋ぐだけの浅い構造になった。

## 演繹構造の再整理

### 演繹の基本原則

- 演繹は具象→抽象の方向。情報を落とす（= 抽象化する）ことで複数の命題が得られる
- 全分割命題は T において真であるため、T から直接演繹できる
- 一本道（T → X → P*）の中間は価値が低い。見出すべきは、**一つの中間命題から複数の分割命題が演繹される分岐構造**

### 誤った方向性の例

当初の分析では以下の誤りがあった:

- **因果と演繹の混同**: P2 → P6 → P5 → P3 という因果の連鎖を導出関係として扱おうとした。因果は演繹ではない
- **情報の欠落**: 「くじ引きで犠牲者を決めた」から P3（マッチで決め事をした）を導出しようとすると「マッチ」という具象性が落ちる。演繹は情報を失わない方向でなければならない

### 共通具象元の例

P3（マッチで決め事）と P5（命の決定）の共通具象元:

```
「マッチのくじ引きで犠牲者を決めた」
  → P3（「犠牲者」を落とす → マッチで決め事をした）
  → P5（「マッチ」を落とす → 命を落とす決定がなされた）
```

中間命題は P3 と P5 より具象的（両方の情報を含む）であり、異なる情報を落とすことで P3 と P5 がそれぞれ得られる。

### 中間命題の要件

中間命題は以下の二つを同時に満たす必要がある:

1. **演繹的に複数の分割命題を導出する**: T からの演繹構造に位置し、情報を落とすことで複数の分割命題が得られる
2. **モデル集合を非自明に分割する**: true_models が {T} だけにならない。分割命題として機能する

どちらか一方だけでは不十分。T からの演繹分析とモデル横断比較を突き合わせて初めて得られる。

## 導出関係分析のアルゴリズム（改訂案）

### Step 1: 分割命題を列挙する

Phase 2 Step 2 の出力（モデル横断比較から抽出された分割命題群）を入力とする。

### Step 2: T から全分割命題への演繹構造を整理する

T から各分割命題への演繹関係を描く。この過程で命題間の演繹関係（P2→P1 等）も自然に現れる。T → P1 のような直接辺が上位経路（T → P2 → P1）に包含される場合、冗長として消える。

```
      T
   / | | \
  P2 P3 P5 P6
   \  |  |  /
    P1  P4
```

### Step 3: 共通具象元を見出す

演繹構造の中に、複数の命題を同時に演繹する共通具象元を見出す。T と分割命題の間に限らず、分割命題同士の間にも存在しうる。

見出した共通具象元は中間命題の要件（演繹的分岐 + モデル分割）を満たすか検証する。

### Step 4: 再帰的に適用する

見出した共通具象元に対して Step 2〜3 を再帰的に適用する。さらにその上に共通具象元がないか探し、これ以上分解できなくなるまで続ける。結果として T から分割命題までの演繹 DAG が構築される。

### Step 5: 抽象化による発見（可能であれば）

既存の分割命題に抽象化を適用することで、一見無関係に見える命題間の共通構造が浮かび上がり、新たな共通具象元の発見を導くことがある。

例: P6（重量削減）と P5（命の決定）を抽象化すると、どちらも「困難への段階的対処」という構造を共有しており、「軽量化では足りず犠牲者選択に至った」という共通具象元の存在を示唆する。

この操作は機械的ではなく発見的であるため、必須ステップとはしない。判例で具体パターンを示し、適用者の発見をサポートする。
