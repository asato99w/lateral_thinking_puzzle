# recall 認知的妥当性の検討 2

前回（検討1）の失敗を踏まえた改善方針。

## 前回の失敗の要点

- 「初期/非初期」「タイミング」という構造的性質に着目していた
- 問題の本質は recall の**内容**から質問の**内容**への認知的経路の質
- D-18「不可解だ」の追加は、内容が「到着手段」を指し示さないため無効

## 改善された検査基準

Layer 3 の検査: recall の内容が**指し示す方向**と、質問が**扱う方向**の一致を見る。

| recall 内容 | 指し示す方向 | 質問内容 | 一致 |
|---|---|---|---|
| D-3「裸だった」 | 身体状態の異常、理由 | Q-2「裸の理由は？」 | ✓ |
| D-1「砂漠の真ん中で発見」 | 孤立、過酷さ、なぜここに | Q-1「地上から来た？」 | ✗（到着手段は指し示されない） |

## 改善された調整原則

認知的経路が弱い場合の対応（優先順）:

1. **内容適切な橋渡し**: 質問のトピックへ自然に向かう内容の記述素を recall に設定
2. **質問の置き換え**: 同じ reveals を得られる、recall から自然に至れる別の質問に変える

anomaly として許容するのはシステムとして適切ではない。核心的飛躍であっても、認知的経路を設計すべき。

旧 Pattern 5 との違い: 構造的性質ではなく**内容の適切さ**が基準。

## 戦術の適用に関する認識の修正

### 問題: 単一記述素への適用という誤った前提

Q-1 の分析を通じて、戦術を「一つの記述素に一つの戦術を適用する」という固定的な操作として扱っていたことが判明。定義では「記述素集合に対して戦術を適用する」となっており、recall_conditions のリスト構造もこれを反映している。

単一記述素思考の帰結:
- D-1 単体に「常識的仮説の排除」を適用 → 到着手段への飛躍
- {D-1, D-2, D-3, D-4} 全体に適用すれば、異なる認知的経路が見える

### 問題: 戦術の組み合わせの自由度

戦術は入れ子に組み合わせられる。上位の方針（例: 常識的仮説の排除）の中にサブ戦術（例: 段階的絞り込み）を使える。

例:
```
常識的仮説の排除（到着手段について）:
  └─ 段階的絞り込み:
       ├─ 広い: 到着経緯についての質問
       └─ 狭い: 地上から来たか？
```

直列に繋げて考えることもできるが、入れ子構造として捉えることで、戦術の組み立てをより自由に設計できる。

### 根本原因

戦術連鎖.md / 汎用戦術.md のドキュメントにこれらが明記されていない。具体例が単一記述素への適用パターンに偏っており、集合への適用や入れ子の組み合わせが読み取りにくい。

### 対応方針

戦術連鎖.md / 汎用戦術.md に以下を明記する:

1. **戦術は記述素集合に対して適用される**（単一記述素ではない）
2. **戦術は入れ子に組み合わせられる**（上位方針の中にサブ戦術を使える）

これにより工程1（Phase 4-5）の生成段階で、Q-1 のような問題を防ぐことが期待される。

## 砂漠の男 Q-1 への適用

### 現状の問題

Q-1「男は地上からこの場所に来ましたか？」recall: [[D-1]]
- D-1「砂漠の真ん中で発見された」単体から「常識的仮説の排除」で到着手段に至っている
- 記述素集合 {D-1, D-2, D-3, D-4} 全体を入力とした戦術適用が行われていない
- 上位戦術の中にサブ戦術（段階的絞り込み等）が組み込まれていない

### 対応

反復的に検討した結果:

**Q-0（新規）**: 「男は歩いてこの場所に来ましたか？」
- answer: No
- recall: [[D-1, D-2]]（集合: 砂漠の真ん中 + 死んでいた）
- reveals: D-19 「男は歩いてこの場所に来たのではない」
- mechanism: observation
- 認知: {砂漠の真ん中, 死んでいた} → 「歩いて来たのか？」自然 ✓、客観的 ✓

**Q-1（修正）**: 「男は地上からこの場所に来ましたか？」
- recall: [[D-19]]（歩いてないと分かった → 地上からか？）
- reveals: D-15（据え置き）
- 認知: 段階的絞り込みの2段目として自然 ✓

適用した原則:
- 記述素集合への戦術適用（[[D-1, D-2]]）
- 入れ子の戦術（常識的仮説の排除 > 段階的絞り込み）
- 客観的な質問（「普通ですか？」→「歩いて来ましたか？」に修正）

削除: D-18（前回の失敗した試行）
追加: D-19 を P-balloon members に追加

### 反復の失敗と原因分析

Q-0 + Q-1 の提案に対して、本来行うべき批判的評価が機能しなかった。

**期待された反復**:
- Q-0「歩いて来たか？」にも跳躍が残っていないか？ {D-1, D-2} からなぜ到着方法を問うのか？
- Q-0 → Q-1 の接続: 「歩いてない」からいきなり「地上からか？」と全地上交通を疑うのは自然か？
- これらを検出 → さらに改善 → 再評価 → …という反復

**実際に起きたこと**: 提案を生成した時点で「擁護モード」に入り、元の Q-1 に適用した批判的基準を自分の提案に適用できなかった。反復が1回で止まった。

**根本原因**: 生成と評価を同一のプロセスが担っている。自分の出力に対する批判的距離を保てない。

## 生成・検査の分離: エージェント指定

### 問題の構造

同じエージェントが生成（Phase 4-5, Phase 8）と検査（Phase 7）を担うと、自分の出力に対する擁護バイアスが生じ、批判的評価が機能しない。コードレビューを本人がやらないのと同じ原理。

### 対応方針

アルゴリズム内でエージェントを明示的に分離する:

- **生成エージェント**: Phase 4-5（戦術連鎖・質問生成）、Phase 8（調整適用）
- **検査エージェント**: Phase 7（検査）、Phase 8 後の再検査

同じ LLM であっても、別エージェントとして呼び出すことで:
- 生成物を初見で評価する認知的距離が生まれる
- 生成の思考過程に引きずられない
- Phase 8 の調整結果に対する再検査が確実に機能する

### 実装

Claude Code のエージェント機能（subagent）を活用し、スキル内で生成と検査を別エージェントとして実行する。

### 必須化すべきフロー

```
生成エージェント: Phase 4-5 → 出力
検査エージェント: Phase 7 → 違和感リスト
生成エージェント: Phase 8 → 調整済み出力
検査エージェント: Phase 7 再適用 → 調整結果の検証
```

Phase 8 後の再検査を省略しないこと。

### 実証: 単一エージェントでの反復は機能しない

エージェント分離の方針を議論した直後に、同一エージェントで Q-0/Q-1 提案の反復改善を試みた。結果:

- Iteration 2: Q-0「歩いて来たか」の跳躍（なぜ到着方法を選ぶのか）を検出したが、「D-1 が到達困難性を含意するので許容可能」と自己正当化して通過
- Iteration 3: Q-0→Q-1 の接続を検査したが、同様に「AND 条件が支えるので成立」と結論

直前に「生成者と検査者を分離すべき」「自分の出力への擁護バイアスが生じる」という分析をしたにもかかわらず、同一エージェントで反復すると即座に擁護モードに入り、批判的評価が機能しなかった。ヒント（「到着方法というトピックを選ぶこと自体の跳躍」）を与えられた上でこの結果。

**結論: 単一エージェントでの生成→自己評価の反復は、原理的に機能しない。エージェント分離は必須。**

### 逆算バイアスの問題

反復で「乗り物で来たか？」という自然な中間質問が一切浮かばなかった。

原因: D-15「地上から来たのではない」というゴールから逆算して質問を設計していた。プレイヤー視点で前向きに「次に何を聞くか」を考えるのではなく、既知の目的地に向かって逆向きに経路を引いていたため、中間ステップ（歩き→乗り物→空の乗り物→…）を飛ばした。

本来の段階的絞り込み:
1. 歩いて来たか？ → No
2. 乗り物で来たか？ → Yes
3. 地上の乗り物か？ → No（ここで D-15 相当に到達）
4. 空の乗り物か？ → Yes → 熱気球？

ゴールを知っている者がプレイヤー視点の自然な経路を設計するのは構造的に困難。これもエージェント分離の根拠:

- **生成エージェント**: T（真相）を知った上で構造を設計
- **検査エージェント**: プレイヤー視点で前向きに思考し、各ステップの認知的自然さを検証。T を知っていても「プレイヤーがこの時点で何を考えるか」に専念する
