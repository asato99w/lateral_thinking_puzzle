# 工程3: ひっかけ — 検討メモ

## ひっかけの構造分析

### S-T ギャップとしてのひっかけ

ウミガメのスープにおいて、S 自体がすでにひっかけの構造を持っている。

- **S が提示するもの**: レストラン → スープを注文 → 確認 → 自殺。プレイヤーはこの時系列を読み、「スープの確認」と「自殺」を直接的な因果で結ぶ。スープに何か問題があった、と
- **T の実際の因果**: 「スープの確認」は因果の起点ではなく、過去の記憶を呼び覚ますトリガーに過ぎない。実際の因果は S の時間軸の外（過去の遭難体験）にある

ここから 3 つの構造が抽出できる:

1. **因果の錯誤**: S の出来事の間にプレイヤーが自然に想定する因果関係が、T の実際の因果と異なる
2. **時間軸の隠蔽**: T の真の因果が S の提示する時間軸の外にある
3. **言葉の二重性**: 「ウミガメのスープ」が本物と偽物の二重の意味を持つ

一般化すると: **ひっかけ = S がプレイヤーに誘導する解釈フレームと、T の実際のフレームの乖離**

工程1-2 は T → S 方向（逆算連鎖）で構築する。ひっかけは S → プレイヤーの解釈 方向の現象。工程3 は逆方向の分析を扱う工程になる。

### 認知的断層との関係

認知的断層（`notes/2026-03-02_目的先行型統合アルゴリズムの検討.md`）は S と T を突き合わせて特定される、プレイヤーが超えなければならないギャップ:

- 断層 α: 確認行為の意味の転換（品質不満 vs 記憶との比較）
- 断層 β: 過去の体験の存在（情報ゼロ vs 極限体験）
- 断層 γ: 「本物である」ことの反転（安心 vs 過去は偽物だった証拠）
- 断層 δ: 自殺動機の深度（食事で自殺？ vs 人肉を食べていた悟り）

ひっかけと認知的断層の関係:

- **認知的断層**: S と T の間に客観的に存在するギャップ。S と T が決まれば自動的に特定される
- **ひっかけ**: そのギャップを超えにくくする仕組み

断層は存在するだけでは「ひっかけ」にはならない。S の表現がプレイヤーを特定の解釈に強く誘導するとき、断層が深くなる。例えば断層 γ で、S が「間違いなくウミガメのスープです」と強調しているのは、プレイヤーに「本物 = 問題解決」と読ませる誘導であり、反転の衝撃を大きくしている。

### 工程3 の位置づけ

工程3 は「認知的断層を活用・深化させる仕掛けの設計」。ただし S の改変は入力なので範囲外であり、ゲーム進行中の仕掛け（質問やデータ構造の側）で行う。

## 仮説の棄却メカニズム

### 現状の問題

現在のデータモデルでは confirmed（確認済み記述素）と導出記述素（formation_conditions による）が増加する一方で、仮説が排除されるメカニズムがない。実際のプレイヤー体験では「誤った仮説が崩される」ことがパズルの重要な要素。

### 提案: rejection_conditions

formation_conditions（肯定条件）に加え、rejection_conditions（否定条件）を導入する。

```
formation_conditions: [["D-3"]]        → D-3 確認で導出
rejection_conditions: [["D-neg-1"]]    → D-neg-1 確認で棄却
```

導出判定: formation_conditions が満たされる AND rejection_conditions が満たされない

### 「No」回答の質問との連携

1. 認知的断層の S 側の読み（誤った解釈）に対応する質問を設計する
2. 「No」回答で否定形の記述素が reveals される
3. rejection_conditions により誤った仮説が棄却される
4. その仮説に依存していた recall_conditions の質問が閉じる

### rejection_conditions の参照範囲

rejection_conditions は **confirmed のみ**を参照する。導出記述素は参照しない。

理由: 導出記述素（仮説）同士の排他関係を定義すると、仮説空間のモデリングに踏み込むことになる。「仮説 A を思いついたから仮説 B を捨てる」というのは一般的に成り立たない — プレイヤーは複数の仮説を並行して保持する。confirmed のみを参照すれば「事実として否定された → 仮説が棄却される」というシンプルな関係に収まる。

### 単調性への影響

導出の単調増加が崩れる（棄却により導出記述素が減少しうる）。ただし:

- confirmed は増加のみ（不可逆）
- rejection_conditions の充足も不可逆（confirmed が増えるのみなので）
- 棄却された仮説が復活することはない
- 連鎖的棄却（A 棄却 → A に依存する B も導出不能）は起きうるが、振動はしない

不動点計算のロジックと eval スクリプト・エンジンの改修が必要。

## 断層の仮説集合と対立仮説集合族

### 次元の整理

断層そのものは仮説ではなく、S と T の間の構造的ギャップである。これを操作可能にするために:

1. **主要仮説集合 H**: 断層を構成する記述素の集合（T 側の真の仮説群）
2. **対立仮説集合族 {H'}**: プレイヤーが持ちうる誤った仮説の集合群（S 側から誘導される読み）

両者を同じ形式（記述素の集合）で表現することで、同列に扱える。H の記述素が confirmed されていくと H' が棄却される、という関係を rejection_conditions で形式的に記述できる。

**断層 β: 過去の体験の存在**
- H_β = {D-16(危機的体験), D-15(船), D-6(遭難), D-7(漂着)}

### S からの距離と対立仮説の性質

断層の S からの位置によって、対立仮説の供給源が異なる:

- **S に近い断層**（α, γ）: S のテキスト表現が対立仮説を直接誘導する
- **S から遠い断層**（β, ε）: 対立仮説は「S の時間軸内で完結するはず」というデフォルトの前提、または情報の不在から生じる

## 工程3 の設計方針: 戦術連鎖への分岐の付加

### 基本方針

工程1-2 の戦術連鎖（`_phase4`）の各ステップは「入力記述素 → 戦術 → 出力記述素」の構造を持つ。工程3 では、各ステップに対して「同じ入力から別の方向に進む可能性」を設計する。つまり**戦術連鎖の各ステップに対する分岐の付加**が工程3 の仕事。

### ウミガメのスープでの試行

**D-5(自殺した) → D-16(危機的体験) [因果の逆転]**
他の方向:
- レストランで何か起きた（S の時間軸内に留まる）
- 精神的に不安定だった（男自身の属性へ）
- 誰かに追い詰められていた（外部の人間関係へ）

**D-16(危機的体験) → D-15(船に乗っていた) [段階的絞り込み]**
他の方向:
- 戦争体験
- 事故に遭った
- 犯罪に巻き込まれた

**D-15(船) → D-6(遭難) [時間軸の拡張]**
他の方向:
- 漁師だった・船旅を楽しんだ（日常的な船の経験）
- 海軍にいた
- 船上で事件があった

**D-8(食糧尽きた) → D-9(仲間が死んだ) [因果の逆転]**
他の方向:
- 救助が来た（極限からの脱出方向）
- 食べられるものを探した（サバイバル方向）
- 仲間と争いになった（人間関係方向）

**D-10+D-2(肉でスープ+注文) → D-12(味が違った) [矛盾の指摘]**
→ 二つの事実が揃えば論理的に導ける。分岐は少ない。

### 出題者バイアスへの注意

分岐の豊かさ（一本道かどうか）を判断する際、**T の答えを知っている状態で評価すると誤る**。例えば「船に乗っていた → 遭難した」を「自然に出る」と判断するのは、T を知っている出題者の視点。実際のプレイヤーにとって「船に乗った経験がある人」の大多数は遭難していないため、「遭難」は多くの候補の一つに過ぎない。

一本道かどうかの判断基準: **T を知らない状態で、入力の記述素だけを見て、出力が他の候補に比べて圧倒的に優勢かどうか**。この基準で真に一本道と言えるステップはごく少数（D-10+D-2→D-12 程度）。

このバイアスは工程3 の設計全体に影響するため、常に「プレイヤーの視点」で評価する規律が必要。

## 対立仮説の再定義: 解決を持つ仮説

### 対立仮説の要件

先の分析で対立仮説を「別の方向」として列挙したが、多くは解決（Sの出来事に対する整合的な説明）を持たない単なる方向に過ぎなかった。対立仮説と呼べるのは **Sの要素を整合的に説明する別のストーリー** のみ。

例: 「味に不満でクレーム」は α の対立方向だが、自殺を説明できないため対立仮説としては不完全。一方「絶滅危惧種を食べた罪悪感」は確認行為・自殺動機を含む複数の断層にまたがる整合的な説明を持つ。

### 断層レベルでの解決

S 全体ではなく、各断層の「問い」に対する解決として考える:

**α の問い**: なぜ「本当にウミガメのスープか」と確認したのか？
- 真の解決: 過去の記憶との味の比較
- 対立解決: 味がまずかった / 見た目が違った / アレルギー確認

**β の問い**: 自殺の原因はどこにあるのか？
- 真の解決: S の時間軸の外（過去の極限体験）にある
- 対立解決: S の時間軸内で完結する / そもそも原因は別にある

**ε の問い**: 過去に何が起きたのか？
- 真の解決: 食糧が尽き、仲間が死に、禁忌に踏み込んだ
- 対立解決: 通常の範囲で救助された / 仲間との関係が壊れた

**γ の問い**: 「本物」であることは何を意味するのか？
- 真の解決: 本物であることが過去の偽装を証明する
- 対立解決: レストランのスープが偽物だった / 確認は話の通過点

**δ の問い**: なぜ自殺するほどの絶望に至ったのか？
- 真の解決: 自分が人肉を食べていたと悟った
- 対立解決: 絶滅危惧種を食べた罪悪感 / 宗教的禁忌を犯した

### 対立解決の射程

断層レベルで考えると、対立解決の **射程**（他の断層とも整合するかどうか）が分析できる:

- α の対立解決（味の不満）は射程が短い — δ（自殺動機）と整合しない
- δ の対立解決（絶滅危惧種の罪悪感）は射程が長い — α（確認行為）、γ（本物の意味）とも整合し、プレイヤーを長く引きつける

射程の長い対立解決ほど強いひっかけとなる。工程3 では射程を意識した対立解決の設計が重要。

### 戦術連鎖との統合

対立解決を戦術連鎖と同じ形式で構成できる。真の連鎖と同じ入力ノードから、同じ戦術体系を使って対立仮説への連鎖を構築する。対立解決の連鎖が深いほど（多くのステップを持つほど）、プレイヤーがそちらに引き込まれやすく、ひっかけとして強い。

### 残課題

- Q-11（善意の欺瞞）は γ 内に置いたが、独立した断層の可能性がある（断層 ζ 候補）。現時点では保留
- 陸地を埋めるには「誤った方向に進んだことを教える」仕組みが必要 → 棄却メカニズム（rejection_conditions）が前提基盤
- 地形の形式化（断層間の距離・重なりの定義）は未解決。記述素の重複パターンが手がかりになりうるが、適切な形式が見つかるまで保留
- 対立仮説集合族の H' を記述素として形式化し、戦術連鎖と同形式で構成する
- 分岐の豊かさの評価方法を確立する必要がある（出題者バイアスの排除）
- 対立解決の射程を評価する方法の確立

## 工程3 アルゴリズムの全体フロー

検討を経て以下の 6 ステップに整理された。詳細は `notes/2026-03-05_工程3アルゴリズム.md` に記述。

### Step 1: 認知的断層の特定

入力: S, T。S と T を突き合わせ、プレイヤーが越えなければならない構造的ギャップ（認知的断層）を列挙する。

### Step 2: 断層の主要仮説集合 H の定義

各断層を構成する記述素の集合として形式化する。これにより断層が操作可能になる。

### Step 3: 各断層の「問い」の定式化

各断層が問うている核心的な質問を自然言語で定式化する。対立仮説の列挙の指針となる。

### Step 4: 対立仮説集合族 {H'} の列挙

各断層の「問い」に対する、真とは異なる解決を持つ仮説群を列挙する。記述素の集合として H と同形式で表現。

### Step 5: 対立仮説の連鎖構成

各対立仮説集合について、CORE と同じ戦術体系を使って連鎖を構成する。連鎖の入力は:
- S のテキストから直接（S に近い断層の場合）
- CORE 連鎖の各ノード（既に confirmed/derived された記述素からの分岐）

### Step 6: 質問の設計（2段階）

- **Step 6a**: 対立仮説の連鎖に対する質問を生成する。主に rejection_conditions で棄却される「No」回答の質問
- **Step 6b**: CORE 連鎖の内部で分岐が生じるノードに、分岐を誘導・解消する質問を配置する

### 注意: 出題者バイアスの排除

全ステップを通じて、T を知っている状態で評価してはならない。特に Step 5 の分岐の豊かさの判定、Step 6 の質問の必要性の判定において、常にプレイヤー視点（T を知らない状態）で行う。
