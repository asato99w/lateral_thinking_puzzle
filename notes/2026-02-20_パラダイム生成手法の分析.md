# パラダイム生成手法の分析

## 目的

記述素空間からパラダイム群を体系的に発見する手法を確立する。

## パラダイムの性質

パラダイムは設計者が事前に決めるものではなく、記述素空間の構造の中に潜在的に存在する。記述素間の整合・矛盾の構造として内在しており、アルゴリズムはそれを発見するプロセスである。

## 記述素展開との関係

- 記述素展開アルゴリズムが記述素空間を生成する
- 記述素展開の目的は、パラダイムが発見可能な空間を作ること
- パラダイム発見アルゴリズムは記述素空間を入力とする

## 基本的な考え方

真相を構成する事実にはプレイヤーにとっての「意外さの階層」がある。
表層的な事実から深層の核心に至る順序が存在する。

各段階で「ここまで知っている人が持つ一貫した解釈」がパラダイムになる。

### ウミガメのスープの場合

| 深度 | 理解の段階 | パラダイムの例 |
|------|-----------|-------------|
| 表層 | 問題文の字面通り | 日常的な食事体験 |
| 中層 | スープに個人的な意味がある | 過去との結びつき |
| 深層 | 極限状況での体験が背景にある | 極限体験 |
| 核心 | 人肉を食べさせられていた | 真相（人肉の発覚） |

## メインパスの存在

意外さの深度で事実を順序づけると、自然と直列のパラダイム列ができる。
これが**メインパス**に対応するパラダイム群である。

```
日常的な食事 → 過去との結びつき → 極限体験 → 真相
```

メインパスの各パラダイムは、ある段階までの事実を受け入れた上で、
残りの未知の事実を「常識的に」補完した解釈にあたる。

## メインパス以外のパラダイム

メインパスから外れた解釈の分岐も存在する。

例：
- 「料理の専門家としての確認」
- 「スープに対する恐怖・トラウマ」
- 「過去の味覚体験との比較」

これらはメインパスのどこかから分岐し、プレイヤーが横道に逸れて
最終的にメインパスへ戻ってくる構造をとる。

## パラダイム群の全体構造

メインパスを幹として分岐がぶら下がる**木構造**に近い形になる。

```
日常的な食事 ──→ 過去との結びつき ──→ 極限体験 ──→ 真相
  ├─ 専門家としての確認        ├─ トラウマ
  └─ ...                    └─ 味覚比較
```

## アルゴリズムの位置づけ

パラダイム発見の手法はアルゴリズムの一種として定義する（唯一の方法ではない）。
現時点のアルゴリズム A（深度階層型）は theory/paradigm_generation/A_深度階層型.md に記載。
