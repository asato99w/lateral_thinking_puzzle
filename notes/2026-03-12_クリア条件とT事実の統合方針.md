# クリア条件と T 事実の統合方針

## 背景

v3 アルゴリズムではクリア条件（ゲーム終了判定）が明確に設計されていない。分割命題（P1-P7）は抽象的な事実であり、T の具体的事実（熱気球、高度低下、飛び降り等）との間にギャップがある。このギャップをどう埋めるかの方針を整理する。

## クリアの定義

クリア = T の具体的事実が全てプレイヤーに観測（confirmed/derived）された状態。

## T 事実の分類

T（砂漠の男）の具体的事実:

| ID | T 事実 | 現状 |
|----|-------|------|
| T1 | 男は数人で熱気球に乗っていた | 未カバー |
| T2 | 熱気球の高度が下がり始めた | 未カバー |
| T3 | 軽量化のため荷物を全て捨てた | P7 がカバー |
| T4 | 荷物を捨てても不十分で服も脱いだ | P7 がカバー |
| T5 | 軽量化でも高度が回復しなかった | 未カバー |
| T6 | くじ引きで一人が飛び降りることになった | 未カバー |
| T7 | くじ引きはマッチの長さの比較で行った | P6+S6 がカバー |
| T8 | 男は短いマッチを引いた | 未カバー |
| T9 | 男は熱気球から飛び降りた | 未カバー |
| T10 | 男は落下により死亡した | S4 がカバー |

## 統合の基本方針

### 1. 自然に導出できるものを導出する

既存の戦術連鎖の延長として、中間命題を差し込む形で T 事実を導出する。置き換え（既存命題の具象化）は既存の棄却構造を壊すリスクがあるため採用しない。

### 2. 導出時に新たな棄却が起きないか検証する

延長命題が新たにモデルを棄却する場合、以下を確認する:
- その棄却はパストレース上、全経路で既に棄却済みのモデルに対してのみ起こるか
- 想定外の早期棄却が発生しないか
- 棄却が起きる場合、連鎖が不自然なら無理に組み込まない

### 3. 残るものはユニークな要素として扱う

自然に導出できない T 事実は、正解の核心的な驚き（ユニークな要素）である。これらは:
- 無理に連鎖で導出しない
- 全ての抽象的命題が確認された後の最終盤の探索対象として位置づける
- 「具体的な正体を問う」探索行為として自然

### 4. 強引な導出は不要

棄却を引き起こす導出、連鎖が不自然な導出は無理に組み込まない。導出できるものを事前に導出しておけば、残るものが本当にユニークな要素だと分かる。

## 砂漠の男への適用（予備分析）

### 既存カバー（導出不要）
- T3, T4 ← P7（軽量化のため裸 = 荷物投棄 + 脱衣）
- T7 ← P6 + S6（マッチくじ + 折れ = 長さ比較）
- T10 ← S4（死亡）

### 連鎖延長の候補
- T6「一人が飛び降りることに」← P6 + P5 + P4 から自然に想起可能
  - ただし棄却検証が必要（M2 はくじ引きはあるが犠牲ではない → M2 を棄却する可能性）
- T8「短いマッチを引いた」← T6 + S6 から
- T9「飛び降りた」← T6 + T8 から

### ユニークな要素（最終盤の探索対象）
- T1「熱気球」← P4 + P5 + P7 が全て揃って初めて想像可能。乗り物の具体的正体はパズルの核心的驚き
- T2「高度低下」← T1 に依存。熱気球が判明して初めて具体化
- T5「回復しなかった」← T2 に依存

## 検証ステップ（アルゴリズムに組み込むべき項目）

延長命題を追加する際の検証:
1. **到達タイミング**: 延長命題は fc の構造により、前提命題が全て確認された後にのみ到達するか
2. **棄却の安全性**: 延長命題が新たに棄却するモデルは、その時点で全パストレース経路において既に棄却済みか
3. **連鎖の自然さ**: 延長が跳躍なく想起可能か（既存のブラッシュアップ基準を適用）
4. **不合格なら除外**: 上記いずれかに不合格なら、その T 事実はユニークな要素として最終盤に回す
