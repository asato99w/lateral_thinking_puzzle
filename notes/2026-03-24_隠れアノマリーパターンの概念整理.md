# 隠れアノマリーパターンの概念整理

v6アルゴリズムの `rules/アノマリー分解.md`（未設計）に向けた概念整理。

## 背景

v6では Phase 1（構造分解）にパズルの特性に応じた規則を適用する。2要素分解は設計済み。隠れアノマリー分解は未設計。両特性は排他的でなく重なりうる（例: ウミガメのスープ）。

## 定義した概念

| 名称 | 定義 |
|---|---|
| 演繹予測 | 認知状態から演繹される、未来に対する予測 |
| 棄却前提 | 演繹予測の反証により誤りと判明する前提 |
| 主動因命題 | Sの行為・反応を了解可能にする中心的な事実 |

### 棄却前提の具体例

| パズル | 棄却前提 | 性質 |
|---|---|---|
| ウミガメのスープ | 過去に食べたのはウミガメのスープだった | 文脈獲得（特定経験から） |
| 毒キノコ | この環境に人為的な変更は加えられていない | デフォルト仮定 |
| コンタクトレンズ | 寝る前にコンタクトを外した | デフォルト仮定（ルーティン） |
| カセットテープ | その場に本人しかいなかった | デフォルト仮定 |

棄却前提には文脈獲得型とデフォルト仮定型がある。多くの場合はデフォルト仮定型。

### 主動因命題の具体例

| パズル | Sの行為 | 主動因命題 |
|---|---|---|
| ウミガメのスープ | 自殺した | 仲間の肉を食べさせられていた |
| 毒キノコ | 安堵した | 助かる見込みが立った |
| カセットテープ | （殺人と断定 — 判断であり感情ではない。主動因命題が不在に近い） | — |

## 演繹予測の具象的記述と前提命題への分解

### 方法

演繹予測を完全に具象的な命題として記述し、その認知を持つために必須の前提命題を抽出する。

### ウミガメのスープでの適用

**演繹予測（具象的命題）:**

「これから飲むウミガメのスープは、かつて『ウミガメのスープだ』と言われて飲んだものと類似した味がするだろう」

注: 男はウミガメのスープを過去に食べたことはない。遭難中に「ウミガメのスープだ」と言われて人肉を食べた。

**この認知を持つために必須の前提命題:**

1. かつて「ウミガメのスープだ」と言われて何かを飲んだことがある
2. これからウミガメのスープを飲む

この2つがなければ、この予測を持つことができない。

### 意義

演繹予測を具象的命題として記述し、その前提命題に分解することで、認知的な前提群を機械的に導出できる可能性がある。演繹予測の命題内容そのものが前提命題を内包しており、それを抽出する操作として定式化できる。

## 隠れアノマリーの連鎖構造

Sの緊張（なぜその行為をしたか）を、アノマリーから連鎖的に解消する構造。

### ウミガメのスープ

```
味が違う（アノマリー）
→ 過去に食べたのはウミガメのスープではなかった
→ 仲間の肉を食べさせられていた（主動因命題）
→ 自殺した（Sの緊張の解消）
```

### 毒キノコ

```
食用キノコがない（アノマリー）
→ 誰かが食用キノコを採った
→ この近くに人がいる
→ 助かる見込みが立った（主動因命題）
→ 安堵した（Sの緊張の解消）
```

### カセットテープ

```
テープが巻き戻されている（アノマリー）
→ 録音後に誰かがテープを巻き戻した
→ 死者が巻き戻したのではない
→ 第三者がいた
→ 殺人と断定（Sの緊張の解消）
```

## 連鎖の各ステップの認知的前提情報

### 前提の種類

| 種類 | 説明 | 例 |
|---|---|---|
| S由来 | 問題文から得られる | テープに遺書が録音されている |
| T由来 | 真相から得られる（プレイヤーが発見する必要がある） | 遭難中に仲間が死んだ |
| 一般知識 | 常識・物理法則など | 死者は物理的操作ができない |
| 自己知識 | 登場人物の自分自身に関する知識 | 裸眼ではぼやける |

### 観察: T由来の前提は連鎖の深い部分に集中する

連鎖の浅い部分（アノマリー直後）はS由来と一般知識で推論が進み、深い部分でT由来の情報が必要になるパターンが確認された。

## 主動因命題の導出構造

主動因命題は2つの流れの合流点に位置する。

```
アノマリー解消の帰結（動的・新規）──┐
                                    ├→ 主動因命題 → Sの行為の了解
T由来の背景事実（静的・既存）────────┘
```

## Sの緊張とS露出部分の区別

以下は異なる構造要素であり、混同してはならない:

| | 定義 | 毒キノコでの例 |
|---|---|---|
| Sの緊張 | Sにおける説明を要する行為・反応 | 安堵した |
| S露出部分 | アノマリーのうちSに現れている部分 | 辺りに毒キノコしかない |

## 競合モデルの方向性

### 競合モデルの対象

S全体の緊張を列挙し、それぞれに対してモデルを列挙し、隠蔽部分に対する共通抽象を導出する方向が検討された。

### Tの枠組みを前提としない

競合モデルはTを暗黙の前提としてはならない。毒キノコの場合、「食用キノコがあるべきなのにない」という枠組み自体がT由来であり、それを前提とした変奏は本当の競合ではない。

例（毒キノコ — Tの枠組みを前提としない競合）:
- この場所はもともと毒キノコしか生えない
- 男はキノコに詳しくなく、食用キノコを毒と誤認している
- 男は実験室にいる

### 判例パターンの必要性

Tの枠組みを前提としない競合モデルの生成には、Claude の能力的限界がある。判例パターン（何を変えうるかの軸と具体例のセット）を蓄積し、規則に組み込む必要がある。

## 主動因命題の鍵

### 鍵の特定方法: 概念の置き換えによる動因特性の変化

主動因命題の中で、置き換えると動因特性（Sの行為を了解可能にする力）が変化する概念が鍵である。

| パズル | 主動因命題 | 鍵 | 置き換えテスト |
|---|---|---|---|
| ウミガメのスープ | 仲間の肉を食べさせられていた | 人肉 | 犬の肉→不快だが自殺に至らない、普通の魚→動因にならない |
| 毒キノコ | 近くに人がいるので助けを求められる | 人の存在 | 熊→危険で安堵しない、川→安堵とまではいかない |
| カセットテープ | 第三者が巻き戻した | 第三者の存在 | 機械的故障→殺人とは言えない |

### 2要素分解の鍵との対応

- **2要素分解**: 鍵が2つの要素を繋ぐ（例: しゃっくりが「銃を向けた」と「ありがとうと帰った」を繋ぐ）
- **隠れアノマリー分解**: 鍵が主動因命題に動因特性を与える（例: 人肉が「食べさせられていた」を自殺に至る衝撃にする）

### 鍵の隠蔽と鍵なし版

2要素分解と同様に、主動因命題から鍵を除去した鍵なし版を構成できる。

| パズル | 主動因命題 | 鍵なし版 |
|---|---|---|
| ウミガメのスープ | 仲間の肉を食べさせられていた | 食べたものの正体が思っていたものと違った |
| 毒キノコ | 近くに人がいるので助けを求められる | 近くに誰かが来た痕跡がある |

鍵なし版は導出グラフの中間ノード（2要素分解のFPCに相当）として機能する可能性がある。

## アノマリー解消と鍵導出の距離

アノマリー解消と鍵導出の間の距離はパズルによって異なる。必ずしも距離が大きい方が良いとは限らないが、パズルの一つの特性として捉えられる。

| パズル | アノマリー解消 | 鍵導出 | 距離 |
|---|---|---|---|
| ウミガメのスープ | 過去に飲んだものはウミガメのスープではなかった | 人肉だった | 大（「ではなかった」から「何だったか」への跳躍がある） |
| 毒キノコ | 誰かが食用キノコを採った | 人がいる | 小（解消と鍵がほぼ直結） |
| カセットテープ | 誰かがテープを巻き戻した | 第三者がいた | 小 |

## 背景事実の再定義: 認知状態の前提群

### 基本的な考え方

演繹予測も鍵導出も認知の話である。背景事実とは、これらの認知状態を構成するために必須の前提群と定義できる。

### ウミガメのスープでの適用

**演繹予測の認知状態の前提群:**
1. かつて「ウミガメのスープだ」と言われて何かを飲んだことがある（T由来）
2. これからウミガメのスープを飲む（S由来）

**鍵導出の認知状態の前提群:**
1. 過去に飲んだものはウミガメのスープではなかった（アノマリー解消の帰結）
2. そのスープは遭難中に飲んだ（T由来）
3. 遭難中に仲間が死んだ（T由来）
4. 食料が尽きていた（T由来）

**背景事実** = 上記前提群のうち、アノマリー解消の帰結でもS由来でも一般知識でもないもの:
- かつて「ウミガメのスープだ」と言われて何かを飲んだ
- 遭難中に飲んだ
- 仲間が死んだ
- 食料が尽きていた

### 毒キノコでの適用

**演繹予測の前提群:**
1. 自分はキノコが自然に生える環境にいる

**鍵導出の前提群:**
1. 自分は遭難している（T由来）

**背景事実** = 「遭難している」

### 意義

背景事実を「認知状態の前提群」として定義することで、演繹予測と鍵導出の具象的記述から機械的に抽出できる。これまで漠然と「T由来の文脈情報」としていたものに導出手順を与えられる。

## ゲーム設計上の制約: アノマリー解消と鍵導出のタイミング

### 基本原則

アノマリー解消と鍵導出は、全ての背景事実が観測された後に同時に起きるべきである。背景事実の部分集合でアノマリー解消をトリガーするような設計にしない。

### 理由

背景事実の部分集合でアノマリー解消のタイミングを制御する設計は、Claude（実行者）にとって複雑すぎてゲーム性が壊れやすい。全背景事実の観測後にまとめて解消・導出する方が設計が単純で確実である（実践的理由）。

### アノマリー解消と鍵導出の距離の再解釈

先述の「距離」（ウミガメは大、毒キノコは小）はゲームの時間的順序ではなく、概念的な距離である。ゲーム上は同時に到達する。

### 解消メカニズムの隠蔽

背景事実の中には、そのままゲーム中に提示するとアノマリーの解消のされ方（解消メカニズム）を示唆してしまうものがある。

例: 「かつてウミガメのスープだと言われて何かを飲んだ」→ 味の比較 → 過去のは本物ではなかった、という解消パスが見えてしまう。

情報そのものを隠すのではなく、その情報がどのようにアノマリー解消に繋がるかのパスが見えないようにする必要がある。この隠蔽の仕組みの設計は未検討。

## 語用論的含意の検証

### 問題

命題のラベルや質問文の言語的形式が、語用論的含意（グライスの会話の含意）により、棄却前提の否定や鍵を示唆してしまうことがある。

例: 「ウミガメのスープだ**と言われて**何かを飲んだ」→ 「と言われて」「何かを」という有標な表現が、「実際にはウミガメのスープではなかった」を語用論的に含意する。本当にウミガメのスープであれば「ウミガメのスープを飲んだ」と言えば済む。

これは命題の内容ではなく言語的形式による情報漏洩であり、既存の情報漏洩検証（Phase 6 のNFラベル検証等）とは異なる種類のチェックが必要。

### 方向性

検証ステップとして語用論的含意のチェックを追加する。命題ラベル・質問文について、語用論的に棄却前提の否定や鍵を含意していないかを検証する。

### 注記

Claude（実行者）は認知レベルの情報の扱いに限界があるため、このチェックには判例パターンが必要になる可能性が高い。

## 認知状態間の包含関係

ある認知状態は別の認知状態を包含しうる。演繹予測を具象的命題として記述すると、その前提命題が内包されている。この包含関係を利用して前提命題を抽出する操作が、アノマリー分解の手順の基盤となる可能性がある。

## 未解決の課題

### 自明性の判断

前提群の抽出において「Sから自明」等の判断にClaude（実行者）の精度不足が見られる。何がSから導出可能で何がT由来かの境界判定に課題がある。

### 演繹予測の前提分解の他パズルへの適用

ウミガメのスープ・毒キノコで確認された「認知状態の具象的記述 → 前提命題の抽出」をコンタクトレンズ・カセットテープに適用し、手法の一般性を検証する必要がある。

### 認知レベルの情報の扱い

以下の点でClaude（実行者）の能力的限界が確認された:
- 登場人物の信念と事実の区別の精密な取り扱い（例: 「ウミガメのスープを食べた」と「ウミガメのスープだと言われて何かを食べた」の混同）
- 情報量の必要十分性の判定（何が独立した情報で何が導出可能かの識別精度）
- Tの枠組みを暗黙の前提としない競合モデルの生成
- 「アノマリー観測」の概念の混乱（感覚的観察、認知的認識、文脈情報の混同）
- 自明性の判断精度の不足

これらは手順の記述をより明示的・制約的にする必要があることを示唆する。
