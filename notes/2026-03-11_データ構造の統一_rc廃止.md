# データ構造の統一: recall_conditions の廃止

## 背景

turtle_soup 05 のブラッシュアップ中に、想起の自然さを評価する際に fc に含まれない命題（プレイヤーの蓄積知識）を暗黙に参照して「自然だ」と誤って主張する問題が発生した。

原因を分析すると、データ構造に recall_conditions（rc）と formation_conditions（fc）が別々に存在することが混乱を招いていた。rc は fc の重複であり（recall = fc 原則）、独立した概念ではない。にもかかわらず rc がデータ上に独立フィールドとして存在することで、fc とは別の観点（蓄積文脈など）で rc を評価するという誤りが生じた。

## 変更方針

### 1. recall_conditions フィールドの廃止

質問から `recall_conditions` フィールドを削除する。想起条件は reveals される命題の `formation_conditions` で表現される。

### 2. reveals の単一化

質問の `reveals` をリスト → 単一の命題 ID に変更する。各質問は1つの命題と1対1に対応する。

### 3. 複数事実の観測への対応

1つの質問回答から複数の事実が確認される場合、直接 reveals される命題は1つとし、残りは `entailment_conditions`（論理的導出）で表現する。

例: Q の回答で事実 A と事実 B が同時に確認される場合
- Q reveals A（直接）
- B の entailment_conditions に A を含める（A が確認されれば B も論理的に導出される）

### 変更前後の比較

```
変更前:
  質問: { recall_conditions: [[X, Y]], reveals: ["A"] }
  命題A: { formation_conditions: [[X, Y]] }
  → rc と fc の重複。不整合の余地あり。

変更後:
  質問: { reveals: "A" }
  命題A: { formation_conditions: [[X, Y]] }
  → 想起条件は fc のみで表現。重複なし。
```

## 影響範囲

- データ構造: questions の recall_conditions 削除、reveals の型変更
- eval スクリプト: check_recall_fc.py が不要化（構造的に不整合が起きない）
- エンジン: recall_conditions 参照箇所を fc 参照に変更
- 全サンプル: データ形式の更新

## 進め方

1. turtle_soup 05 で試行し、問題がなければ方針として確定
2. 確定後に他のサンプル・eval・エンジンに展開
