# 海亀パズルの具体的分析

## S（問題文）

ある男がレストランに入り、ウミガメのスープを注文した。一口飲んで店員に「これは本当にウミガメのスープか？」と尋ねた。店員は「はい、間違いなくウミガメのスープです」と答えた。男はその夜、自殺した。なぜか？

## T（真相）

男はかつて仲間と船で遭難し、食料が尽きた。仲間の一人が死に、残った者はその肉でスープを作り「ウミガメのスープだ」と偽って男に飲ませた。レストランで本物のウミガメのスープを飲み、味が違うことに気づいた男は、あのとき飲まされたものの正体を悟り、絶望して自殺した。

---

## 1. 逆算連鎖

T を「なぜ？」の連鎖で逆算する。

```
男は自殺した
  ← なぜ？ 自分が人肉を食べていたと悟ったから
    ← なぜ悟った？ 過去に飲んだ「ウミガメのスープ」が本物ではなかったと判明したから
      ← なぜ判明した？ 本物のウミガメのスープを飲んだら、記憶の味と違ったから
        ← なぜ記憶がある？ 過去に「ウミガメのスープだ」と言われてスープを飲んだから
          ← なぜ偽った？ 遭難中に仲間の肉で作ったスープを、そう偽って提供したから
            ← なぜそうなった？ 食料が尽き、仲間が死んだから
```

各ステップが「なぜ？」に対する必然的な回答であり、恣意性が少ない。

## 2. ピース

逆算連鎖の各ステップから、パズルのピースを抽出する。

| ID | ラベル | 依存 |
|----|--------|------|
| a | 食料が尽き、仲間が死んだ | なし（独立） |
| b | 仲間の肉でスープを作った | a, c |
| c | 「ウミガメのスープ」と偽って提供された | なし（独立） |
| d | 男はそれを飲んだ（味の記憶がある） | c |
| e | 本物を飲んだら味が違った | なし（独立） |
| f | 過去のものは本物ではなかったと判明した | d, e |
| g | 正体を悟り自殺した | a, b, c, d, e, f |

独立ピース（a, c, e）は前提なしで発見可能。依存ピース（b, d, f, g）は、依存先がすべて発見された後にはじめて成立する。

## 3. 探索領域

独立ピースはそれぞれ探索領域を開く。探索領域とは、そのピースを起点に深掘りできる問いの空間である。

**探索領域 a — 遭難と仲間の死**

ピース a「食料が尽き、仲間が死んだ」を起点とする領域。

- 男はどういう人物か？
- 何が起きたのか？
- 仲間はどうなったか？
- 食料はどうしたのか？

この領域内で発見されうる事実: 遭難したこと、食料が尽きたこと、仲間が死んだこと。

**探索領域 c — 偽りのラベル**

ピース c「『ウミガメのスープ』と偽って提供された」を起点とする領域。

- 過去にスープを飲まされたことがあるか？
- そのスープは何だと言われたか？
- なぜそう偽ったのか？

この領域内で発見されうる事実: 仲間からスープを飲まされたこと、「ウミガメのスープだ」と言われたこと。

**探索領域 e — 味の違い**

ピース e「本物を飲んだら味が違った」を起点とする領域。

- 男は以前にもスープを飲んだことがあるか？
- 今のスープの味に違和感があったか？
- 何と比較して違うのか？

この領域内で発見されうる事実: 以前にもスープを飲んだこと、味が違ったこと。

**探索領域の特徴**

- 各領域は独立に発見・探索できる
- 領域内で深掘りが可能（引力圏を持つ）
- T への到達には探索領域間の接続・統合が必要
- 依存ピース（b, d, f, g）は探索領域間の接続点に相当する

## 4. メカニズム

探索で得られるものには三種がある。

### 直接観測

質問によって事実を直接得る。最も基本的なメカニズム。

例:
- 「男は遭難したことがある？」→ はい → 遭難の事実を得る
- 「食料が尽きた？」→ はい → 食料枯渇の事実を得る
- 「以前にもスープを飲んだことがある？」→ はい → 過去のスープ体験の事実を得る

直接観測は探索領域内の掘り下げで主に発生する。

### リンク

複数の事実を結合して、新しい理解を生む。構成的な推論。既存の理解と矛盾しない。

例:
- 「仲間が死んだ」+「ウミガメのスープと偽った」→ スープの正体は仲間の肉だった

これは探索領域 a の事実と探索領域 c の事実が結びつくことで生まれる。どちらか一方の領域だけでは到達できない。ピース b（仲間の肉でスープを作った）はこのリンクにより発見される。

### アノマリー

事実間の矛盾を認識し、既存の理解の再解釈を迫る。変容的な推論。

例:
- 「ウミガメのスープだと言われた」+「本物と味が違った」→ 過去のスープは本物のウミガメのスープではなかった

「ウミガメのスープだ」という過去の理解が、味の違いという事実と矛盾する。この矛盾の認識が、「あれは本物ではなかった」という再解釈を強いる。ピース f（過去のものは本物ではなかったと判明した）はこのアノマリーにより発見される。

### リンクとアノマリーの違い

- **リンク**: 事実を足し合わせて新しい事実を導く。「A かつ B、ゆえに C」
- **アノマリー**: 事実間の矛盾が既存の理解を書き換える。「A と言われたが B と矛盾する、ゆえに A は偽」

どちらも複数の事実から新しい理解を導く推論だが、リンクは構成的（積み上げ）、アノマリーは変容的（書き換え）である。

### メカニズムとピースの対応

| ピース | メカニズム | 説明 |
|--------|-----------|------|
| a | 直接観測 | 探索領域 a 内の掘り下げ |
| c | 直接観測 | 探索領域 c 内の掘り下げ |
| e | 直接観測 | 探索領域 e 内の掘り下げ |
| d | 直接観測 | 探索領域 c 内の掘り下げ |
| b | リンク | 領域 a × 領域 c の結合 |
| f | アノマリー | 領域 c × 領域 e の矛盾 |
| g | リンク | 全体の統合 |

## 5. 仮説によるオープン制御

### 質問の想起

質問の集合は所与（パズルデータとして事前定義）。エンジンが判定するのは「今、どの質問がオープンか」。

質問がオープンになる過程を**想起**と呼ぶ。想起の引き金は2種類ある:

1. **事実からの直接想起**: ある事実が観測されたことで、直接的に質問が想起される
2. **仮説からの想起**: 事実の蓄積により仮説が形成され、その仮説から質問が想起される

```
事実 ──→ 想起 → 質問がオープン
事実の族 → 仮説 → 想起 → 質問がオープン
```

想起の引き金は事実と仮説の2つで尽きる。

### 仮説

仮説はピースとは全く別の概念:
- **ピース**: T を構成する真実の断片。解くべき対象
- **仮説**: プレイヤーの推論状態。探索を駆動するもの

仮説は事実集合の族（OR of AND）により形成される:
- 仮説 h に対して、それを形成する事実集合が複数ありうる
- どの経路（どの事実集合）でも h が成立する

仮説は現時点では便宜的な道具という認識だが、ドメインモデルには組み込む。

### 事実の生成

質問の回答は Yes/No どちらも対称に事実を生む。

- 「スープに毒が入っていた？」→ No → 「スープに毒は入っていない」という事実
- 「男は船乗りか？」→ Yes → 「男は船乗りである」という事実

排除質問も否定事実を生み、仮説の形成に寄与する。

仮説はあるタイミングで事実になりうる。仮説から想起された質問に回答すれば、それが新たな事実として観測され、さらに別の想起（直接的にも、別の仮説形成を介しても）に寄与する。

### サイクル

```
facts → f(facts) → hypotheses → 想起（事実 or 仮説）→ オープン質問 → 質問選択 → g(question, answer) → facts' → ...
```

- **f(facts) → hypotheses**: 事実集合から仮説集合を更新する関数
- **g(question, answer) → facts**: 質問選択と回答による事実集合を更新する関数

f と g の中身は差し替え可能で、理論の精緻化に合わせて調整できる。

### 具体例: 探索領域 a への入り方

**事実からの直接想起の例:**

S の事実 Ps-4「男は店員に『これは本当にウミガメのスープか？』と尋ねた」が観測されている。この事実から直接、「なぜ確認した？」「以前にも飲んだことがある？」といった質問が想起される。仮説を介さない。

**仮説からの想起の例:**

仮説「男は過去に遭難した」を考える。この仮説を形成しうる事実集合の族:

- {男は船乗りである, 特殊な過去がある}
- {男は船乗りである, 危機的な状況を経験した}

これらの構成要素（「男は船乗りである」「特殊な過去がある」等）は、先行する質問の回答として得られた事実。例えば:

1. 「男は船乗りか？」→ Yes → 事実「男は船乗りである」
2. 「男は危機的な状況を経験した？」→ Yes → 事実「危機的な状況を経験した」
3. この2つの事実が揃う → 仮説「男は過去に遭難した」が形成
4. この仮説から質問「遭難したことがある？」が想起される
5. 「遭難したことがある？」→ Yes → 事実「遭難した」（仮説が事実になる）
6. この新たな事実が、さらなる想起（直接的にも、別の仮説形成を介しても）に寄与する
