# 分割構造の設計問題

## 発端

turtle_soup 07 サンプル生成中に P2（「確認結果→悟り→自殺の因果」）が孤立していることが判明。どの下流命題の fc からも参照されず、可視化でも孤立ノードとして現れた。

## 問題の整理

### 問題1: 命題間の関係性の分析方法

依存関係の分析が分割構造（true_set の包含）や T の文脈に依存しすぎている。分割命題は抽象的な命題であり、本来互いに依存しにくい。依存関係の分析にはモデル集合や T を参照しない、命題の意味に基づく方法が必要。

### 問題2: 分割命題の抽象度

同一のモデル分割を達成するなら、より抽象的な命題が望ましい。例:「偽ラベル体験」→「欺瞞があった」。T の物語に沿いすぎた具体的な命題は、他の命題との不要な依存を生む。

## 関係性の分析: 概念的含意（情報の消費）

分割構造やTに依存しない、命題の概念的な意味のみに基づく分析手法。**ある命題の確認が別の命題の確認を不要にする（消費する）関係**を特定する。

### 分析対象

P1' を「欺瞞があった」（偽ラベルではなく抽象化）として:

- P1': 欺瞞があった
- P2: 確認結果→悟り→自殺の因果
- P3: 男は偽られた側だった
- P4: 深刻な状況で起きた
- P5: 生存の危機だった
- P6: 飢餓救済目的で提供された
- P7: 道徳的禁忌の食材だった

### 含意関係（A を確認すると B が不要になる）

| A → B | 理由 |
|---|---|
| P3 → P1' | 偽られた側がいる → 欺瞞があった |
| P5 → P4 | 生存の危機 → 深刻な状況 |
| P6 → P5 | 飢餓救済 → 生存の危機 |
| P6 → P3 | 飢餓から救うために提供した → 提供者がいた → 男は受けた側 |

推移的に: P6 → {P5, P4, P3, P1'} の全てを消費する。

### 独立な対（含意関係なし）

- **P2** と {P1', P3, P4, P5, P6, P7} の全て — P2 は完全に独立
- **P7** と {P1', P3, P4, P5} — P7 はほぼ独立（P6 との関係のみ要検討）
- P3 と {P4, P5}
- P1' と {P4, P5}

### 含意関係図

```
P6 ──→ P5 ──→ P4
 │
 └──→ P3 ──→ P1'

P2（独立）
P7（独立）
```

### 観察

1. **P6 は強力な消費者**: P6 を確認すれば P5, P4, P3, P1' が自動的に成立。4 命題を消費する
2. **P2 と P7 は完全に独立**: どの命題からも消費されず、どの命題も消費しない
3. **P1' は P3 に消費される**: P1' を独立の分割命題にする意義が薄い（P3 の確認で得られる）
4. **P4 は P5 に消費される**: 同様に P4 の独立性は低い

### 方針

この概念的含意関係をベースとして、命題間の依存関係と分割構造を設計する。

- 含意関係は T やモデル集合に依存せず、命題の意味のみから導出できる
- 消費される命題（P1', P4）は分割命題ではなく中間命題として扱う方が適切かもしれない
- 独立な命題（P2, P7）は並列探索の軸として扱える
- 含意のチェーン（P6→P5→P4, P6→P3→P1'）は戦術連鎖の骨格になりうる

## 今後

- 消費される命題の扱い（分割命題 vs 中間命題）を決定する
