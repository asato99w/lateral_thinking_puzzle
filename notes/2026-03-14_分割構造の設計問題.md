# 分割構造の設計問題

## 発端

turtle_soup 07 サンプル生成中に P2（「確認結果→悟り→自殺の因果」）が孤立していることが判明。どの下流命題の fc からも参照されず、可視化でも孤立ノードとして現れた。

## 直接の問題

P2 が棄却するモデル（M13, M14, M15）は全て部分緩和モデルであり、P1 または P4 によっても棄却される。P2 は固有の棄却力を持たず、探索フローに寄与していない。

## 構造的な問題: 入れ子設計

Phase 2 の division_hierarchy は分割命題を単一の入れ子（P1 ⊃ P2 ⊃ P3 ⊃ ... ⊃ P7）として設計している。これにより:

1. Phase 3 の戦術連鎖が入れ子の順序に従う線形依存チェーンになった
2. 各分割命題の fc が直前の分割命題のみを参照する単一経路になった
3. 探索が一本道になり、プレイヤーが異なる軸から探索する自由がない
4. P2 のように入れ子の一段として押し込まれたが下流への接続が作れない命題が生まれた

## 入れ子設計の誤り

division_hierarchy の「親の true_set 内で子が分ける」という構造は、各分割命題の実際の分割（全モデルに対する true/false）を正しく反映していない。分割命題は本来、モデル集合に対する独立した（または半独立な）分割軸であり、一次元の入れ子に押し込むべきものではない。

## 分割命題の本来の性質

分割命題はモデル集合の異なる軸での分割を表現する:
- 「誰が偽ったか」（被害者 vs 加害者）
- 「どんな状況か」（極限 vs 日常）
- 「何の食材か」（道徳的禁忌 vs その他）
- etc.

これらは独立した次元であり、プレイヤーはどの次元からでも探索を開始できるべき。

## 方向性（未成熟）

- Phase 2 の出力を入れ子の木構造からフラットな軸の集合に変える
- 軸間の論理的含意（P5→P4 等）は記録するが探索順序としては扱わない
- 探索の順序は Phase 3（戦術連鎖）が認知的な自然さに基づいて設計する
- Phase 2 は「何で分けるか」、Phase 3 は「どう探索するか」の分離

## 未解決の課題

- 軸間の相関（極限状況のモデルは仲間による提供が多い等）をどう扱うか
- 軸の独立性の程度をどう評価するか
- 部分緩和モデルの扱い（P2 のような「全モデル vs 部分モデル」の分割が不要になるか）
- 既存の division_hierarchy の規則・データ構造をどう改訂するか
- 具体的にどのような出力形式にするか

## 現状

turtle_soup 07 サンプルは Phase 10 まで完了し検証 PASS しているが、上記の構造的問題を内包している。アルゴリズムの改訂後に再生成が必要になる可能性がある。
