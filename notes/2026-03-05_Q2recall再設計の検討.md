# Q-2 recall_conditions 再設計の検討

## 日付
2026-03-05

## 対象
turtle_soup 03 の Q-2「男は過去に危機的な体験をしましたか？」

## 現状の問題
recall_conditions が [["D-5"]]（自殺した）のみで、この事実だけから「過去に危機的な体験があったか？」を想起するのは認知的に不自然。

## 再設計の方針

### recall_conditions 候補
- X-1: 「男は過去に船に乗っていた」（既存 D-15 相当）
- X-2: 「過去の出来事が自殺に関係している」

### X-2 を想起するための前提
- X-3: 「自殺するほどショックを受ける事実を男が知った」

### 連鎖の流れ
```
初期事実(D-5: 自殺した) → X-3(ショックを受ける事実を知った) → X-2(過去の出来事が関係)
初期事実(D-2, D-3等) → X-1(船に乗っていた)
{X-1, X-2} → Q-2 recall(危機的体験があったか)
```

## 設計上の制約
- 全ての記述素は T から導出可能であること
- T に言及のない事実（現在の職業状況、心理状態の解釈等）は記述素にできない
- 「過去の出来事が自殺の原因である」ではなく「関係している」が客観的に適切

## 分析の進展: 質問自体の問題

X-3（「自殺するほどショックを受ける事実を男が知った」）はキーピース（P-realization: 発覚と結末）周辺の記述素である。つまり Q-2 の recall を自然にするためにはキーピース方向の知識が前提として必要になる。

これは recall_conditions の修正では解決できない構造的な問題を示唆する:
- Q-2 を自然に想起するには、キーピースに近い知識が必要
- しかしキーピースの知識がある段階では、Q-2（「危機的な体験があったか」）はもはや不要（すでにその方向に進んでいる）
- つまりこの質問の存在自体に問題がある可能性

### 結論の方向性
Q-2 は工程2（調整）において**破棄を検討すべき質問**である可能性がある。recall_conditions の改善ではなく、質問そのものの必要性を再評価する。

## Q-2 破棄時の構造変更

### D-16 の扱い
- D-16 に formation_conditions [["D-6"]] を設定 → 遭難を知れば危機的体験は導出可能
- D-16 を reveals する質問は不要になる

### 到達パスの再構成
現行: Q-2 → D-16 → Q-1 → D-15 → Q-6 → D-6
変更後: (新パス) → D-15(船) → Q-6 → D-6(遭難) → D-16(導出)

### 必要な新パス: 初期事実 → D-15（船に乗っていた）
先ほどの分析の通り、初期事実からD-15への自然な到達経路を新たに設計する必要がある。
例: {D-2, D-3} → 人物像の探索(9) → D-15 方向
ただし認知的自然さの設計はLLMの限界が確認済みのため、人間の判断が必要。
