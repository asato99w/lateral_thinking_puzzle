# 2要素分解

Phase 1（構造分解）で、2要素特性を持つパズルに適用する規則。

## 適用条件

Sに繋がりのない2つの要素が並置されている。

## 最終出力

FP を根、FPC を中間ノード、共通抽象命題を葉とする導出グラフ。

## 手順

### Step 1: 要素の特定

S から、繋がりのない2つの要素（要素A・要素B）を特定する。

- 2要素間のギャップ（繋がりの不在）が緊張の主体であること
- 各要素は個別に緊張を持つ場合もある（副次的）

### Step 2: 各要素を導出するTの原子命題の特定

各要素について、それを導出するのに必要十分なTの原子命題を求める。

規則：
- 一般論的事実（常識として解き手が持つ知識）は含めない
- 物語固有の事実のみを原子命題とする
- 鍵を含む具象的な形で記述する

### Step 3: 鍵の特定

T の原子命題群から、パズルの鍵（想起前提）を特定する。
鍵は2つの要素の導出に共通して関与する核心的な事実。

### Step 4: FPC の構成

各要素について、Step 2 の原子命題群から鍵の具体的情報を省き、
1つの命題（FPC）にまとめる。

規則：
- FPC から元の原子命題群（鍵なし版）が導出可能であること
- 可能な限り具象的で意味のある命題とすること
- 導出可能性を明示的に検証すること

出力：
- FPC1: 要素Aに対応するFPC
- FPC2: 要素Bに対応するFPC

### Step 5: FP の構成

FPC1 と FPC2 から FP を構成する。

- FP は FPC1 と FPC2 の両方を演繹できる単一の命題
- FP = パズルの核心（プレイヤーの「わかった」に対応する情報）
- clear_conditions = [["FP"]]

### Step 6: 競合モデルの体系的列挙

各要素について、競合する説明モデルを体系的に列挙する。

#### 列挙の方針

要素の観測事実（Sに記述された行為・状態）に対して、
それを説明しうる具象的なストーリーを網羅的に検討する。

列挙の軸：
- **行為者の動機**: なぜその行為をしたか（善意、悪意、無意識、義務など）
- **行為の対象**: 誰に対して / 何に対してか
- **行為の文脈**: どのような状況でそれが起きたか
- **因果の方向**: 意図的か、偶発的か、反射的か

#### 規則

- 各要素について最低5つの競合モデルを列挙する
- T も競合モデルの1つとして同列に含める
- 各モデルはSの事実と整合すること
- モデルは具象的であること（抽象的な記述は不可）

### Step 7: 競合モデルの原子命題への分解

各競合モデルについて、対応する要素を導出するのに必要十分な
原子命題を特定する。

規則：
- 一般論的事実は含めない
- 物語固有の事実のみ
- Tの原子命題（Step 2）と同じ粒度で記述する

### Step 8: 共通抽象の導出

T の原子命題と各競合モデルの原子命題の間で、
共通する抽象命題を求める。

#### 生成ステップ

各T原子命題について、各競合モデルの対応する原子命題と1対1で対にし、
両者を包含する抽象命題を導出する。

規則：
- 可能な限り具象的で意味のある命題を求める
- 無理に抽象化せず、具象に近い形で留める
- 対応する原子命題がない場合は「対応なし」と記録する

### Step 9: 共通抽象の評価と間引き

Step 8 で生成された共通抽象命題を以下の基準で評価し、
基準を満たさないものを除外する。

| 基準 | 内容 |
|------|------|
| 客観性 | 主観や解釈に依存しないか |
| 意味の確定性 | 「何か」「何らかの」を含んでいないか |
| 非トートロジー性 | 情報量があるか |
| 弁別力 | 競合モデル間を区別する力があるか |

評価を通過した命題を残し、通過しなかった命題は除外する。
意味のある共通が見出せなかったT命題はTに固有であると記録する。

### Step 10: 導出グラフの構成

評価を通過した共通抽象命題を、導出ツリーの枝として配置する。

構成（具象→抽象の方向で導出）：
```
FP
├── FPC1（要素A側）
│   ├── 構成要素1（原子命題1の鍵なし版）
│   │   └── 共通抽象命題a（評価通過済み）
│   ├── 構成要素2（原子命題2の鍵なし版）
│   │   └── 共通抽象命題b（評価通過済み）
│   └── 構成要素3（原子命題3の鍵なし版）
│       └── （T固有 — 共通抽象なし）
└── FPC2（要素B側）
    ├── 構成要素1（原子命題1の鍵なし版）
    │   └── 共通抽象命題c（評価通過済み）
    └── ...
```

導出の方向：FP → FPC → 構成要素（鍵なし原子命題）→ 共通抽象命題
各ステップで情報が落ちる（具象→抽象）。

記録事項：
- FP → FPC1, FPC2 への演繹関係と理由
- 各FPC → 構成要素への演繹関係と理由
- 各構成要素 → 共通抽象命題への演繹関係と理由
- T固有の構成要素は葉ノードとして配置し、共通抽象がないことを明記する

## 適用例：バーの男

### Step 1

- 要素A：バーテンダーが男に銃を向けた
- 要素B：男は「ありがとう」と言って帰った

### Step 2

要素Aの原子命題：
1. 男はしゃっくりが止まらなかった
2. バーテンダーはしゃっくりに気づいた
3. バーテンダーは驚かせて止めようとした

要素Bの原子命題：
1. 男はしゃっくりが止まらなかった
2. 水を飲めば治ると思っていた
3. しゃっくりが止まった
4. 男はバーテンダーの意図を理解した

### Step 3

鍵：しゃっくり

### Step 4

FPC1：「バーテンダーは男の異変に気づき、その対処として銃を向けた」

導出検証：
- → 男に異変があった（原子命題1の鍵なし版）
- → バーテンダーがそれに気づいた（原子命題2の鍵なし版）
- → 銃を向けることが対処だった（原子命題3の鍵なし版）

FPC2：「男は問題への対処として水を求めていたが、バーテンダーの行為で問題が解消し、男はその意図を理解した」

導出検証：
- → 男に問題があった（原子命題1の鍵なし版）
- → 水は対処として求めた（原子命題2の鍵なし版）
- → 問題が解消した（原子命題3の鍵なし版）
- → 意図を理解した（原子命題4の鍵なし版）

### Step 5

FP：FPC1 + FPC2 から構成（具体的なラベルは確定時に設定）

### Step 6

要素Aの競合モデル：

| モデル | 概要 |
|---|---|
| 対処（T） | 男の問題に対処しようとした |
| 強盗 | 男から金を奪おうとした |
| 自衛 | 男の脅威的行動から身を守った |
| 排除 | 迷惑な男を追い出そうとした |
| 遊戯 | 冗談や悪ふざけとして |

### Step 7

要素Aの各モデルの原子命題：

| モデル | 原子命題 |
|---|---|
| 対処（T） | 1. しゃっくりが止まらなかった 2. 気づいた 3. 驚かせて止めようとした |
| 強盗 | 1. 男の金を奪おうとした |
| 自衛 | 1. 男が脅威的行動をした 2. 身を守ろうとした |
| 排除 | 1. 男が迷惑行為をしていた 2. 追い出そうとした |
| 遊戯 | 1. 冗談として銃を向けた |

### Step 8

要素AのT原子命題と各競合モデル原子命題の共通抽象（生成）：

**T原子命題1「男はしゃっくりが止まらなかった」：**

| 競合モデル | 対応する原子命題 | 共通抽象候補 |
|---|---|---|
| 自衛 | 男が脅威的行動をした | 男に外見上認識可能な状態があった |
| 排除 | 男が迷惑行為をしていた | 男に外見上認識可能な状態があった |
| 強盗 | （対応なし） | — |
| 遊戯 | （対応なし） | — |

**T原子命題2「バーテンダーはしゃっくりに気づいた」：**

| 競合モデル | 対応する原子命題 | 共通抽象候補 |
|---|---|---|
| 自衛 | 身の危険を感じた | バーテンダーは男について認識した |
| 排除 | 迷惑に気づいた | バーテンダーは男について認識した |
| 強盗 | （対応なし） | — |
| 遊戯 | （対応なし） | — |

**T原子命題3「バーテンダーは驚かせて止めようとした」：**

| 競合モデル | 対応する原子命題 | 共通抽象候補 |
|---|---|---|
| 強盗 | 脅して奪おうとした | バーテンダーは銃を向けることで目的を達成しようとした |
| 自衛 | 身を守ろうとした | バーテンダーは銃を向けることで目的を達成しようとした |
| 排除 | 追い出そうとした | バーテンダーは銃を向けることで目的を達成しようとした |
| 遊戯 | 冗談として銃を向けた | バーテンダーは銃を向けることで目的を達成しようとした |

### Step 9

評価と間引き：

| 共通抽象候補 | 客観性 | 確定性 | 非トートロジー | 弁別力 | 判定 |
|---|---|---|---|---|---|
| 男に外見上認識可能な状態があった | ○ | △（「状態」が広い） | △ | 弱（強盗・遊戯を排除するが曖昧） | **除外** |
| バーテンダーは男について認識した | ○ | ×（「何を」が不定） | △ | 弱 | **除外** |
| バーテンダーは銃を向けることで目的を達成しようとした | ○ | △（「目的」が不定） | ○（目的志向性は情報） | 中（全モデルに共通するため弁別力なし） | **除外**（弁別力なし） |

結果：要素Aについて、評価を通過する共通抽象は得られなかった。
全T原子命題はT固有と記録する。

注記：弁別力のある共通抽象が得られないことは、このパズルの
鍵（しゃっくり）が極めてユニークであることの反映。

### Step 10

導出グラフ（要素A側）：

```
FP
└── FPC1「バーテンダーは男の異変に気づき、その対処として銃を向けた」
    ├── 構成要素1「男に異変があった」（T固有）
    ├── 構成要素2「バーテンダーがそれに気づいた」（T固有）
    └── 構成要素3「銃を向けることが対処だった」（T固有）
```

要素A側は共通抽象が得られなかったため、構成要素が葉ノードとなる。

fc仮置き：
- 構成要素1.fc = [[FPC1]]
- 構成要素2.fc = [[FPC1]]
- 構成要素3.fc = [[FPC1]]
- FPC1.fc = [[FP]]

注記：要素B側も同様の手順で実施する（省略）。
