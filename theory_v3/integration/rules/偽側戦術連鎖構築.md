# 偽側戦術連鎖構築

## 概要

統合アルゴリズム v3 Phase 6 の規則。Phase 5 の棄却分析に基づき、分割階層の偽側に対する戦術連鎖を構築し、偽側の中間命題（formation_conditions 付き）を生成する。

Phase 3（真側の戦術連鎖）の偽側版。真側の連鎖が S → 分割命題への到達パスを構成するのに対し、偽側の連鎖は分岐点から棄却命題または真側への合流点に至る探索パスを構成する。

## 入力

- Phase 1〜5 の全出力
- `structure/汎用戦術.md`

## 出力

- 偽側の戦術連鎖群
- 偽側の中間命題（formation_conditions 付き）

## 出力データ構造

```json
{
  "_phase6": {
    "false_side_chains": [
      {
        "id": "FC1",
        "division_point": "P1",
        "direction": "false",
        "source": ["N1a"],
        "target": "NF1",
        "tactic": "段階的絞り込み",
        "description": "N1a →[段階的絞り込み]→ NF1",
        "fc_source": "N1a は Q1a の reveals"
      }
    ]
  },
  "propositions": [
    {
      "id": "NF1",
      "label": "偽側中間命題のラベル",
      "formation_conditions": [["N1a", "S5"]],
      "entailment_conditions": null,
      "rejection_conditions": null,
      "_truth_value": false
    }
  ]
}
```

偽側中間命題は Phase 4 で生成した `propositions` 配列に追加する。

### フィールド定義

#### false_side_chains

| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 連鎖 ID（FC* 形式） |
| division_point | string | 分岐元の分割命題 ID |
| direction | string | `"false"`（偽側） |
| source | list[string] | 連鎖の起点命題 ID リスト |
| target | string | 連鎖の到達目標命題 ID |
| tactic | string | 適用する汎用戦術名 |
| description | string | 連鎖の記述 |
| fc_source | string | 起点命題がどの質問の reveals に由来するか |

#### 偽側中間命題（propositions に追加）

Proposition 型（`models.py` 参照）。追加フィールド:
- `_truth_value`: T における真偽（エクスポート時に除去される）

## 手順

### Step 1: 偽側の到達目標の設定

各分割命題の偽側について、探索の方向と到達目標を設定する。

#### 対象

分割階層の各分割命題について、偽側（false_set）のモデル群の方向を探索する枝を設計する。

例（turtle_soup の分割階層）:
```
P1=真 {T,M1,M2,M3,M4,M8,M12}  ← 既存の連鎖あり
  P2=真 {T,M1,M4}
    ...
  P2=偽 {M2,M3,M8,M12}         ← 偽側の枝が必要
P1=偽 {M5,M6,M7,M9,M10,M11}   ← 偽側の枝が必要
```

#### 設計方針

- Phase 5 の false_side_analysis を参照し、偽側モデル群の特徴を把握する
- 到達目標の種類:
  - **棄却命題（行き止まり）**: 偽側の探索が棄却に至り、真側に収束する
  - **真側への合流点**: 偽側の探索が既存命題に合流し、別経路から同じ命題に到達する
- 各偽側枝の深さ・方向は設計判断。短ければ簡単、長く深ければ多くの探索を経験する

### Step 2: 偽側の戦術連鎖の構成

各偽側枝について、汎用戦術を適用して戦術連鎖を構成する。

- 連鎖の起点: 分岐点での排除質問の reveals（否定命題）または S 原子命題
- 連鎖の終点: 棄却命題または真側への合流点
- 各ステップは `structure/汎用戦術.md` の戦術を使用する
- 連鎖は S の情報構造から自然に生まれるものであり、T からの逆算ではない

### Step 3: 偽側中間命題の生成

連鎖の各ステップに対応する中間命題を生成する。

- 各命題に formation_conditions を設定する（連鎖の前段の reveals から定まる）
- 命題は独立性・客観性・T 忠実性を満たすこと
- 導入時検証（1 次チェック）を適用する

#### 中間命題の導入時検証（1次チェック）

##### T 文脈での真偽の明確さ

- 中間命題の真偽が T の事実から一意に定まることを確認する
- 真偽が異なる類似表現を混同しないこと

##### 意味の確定性

- 命題単体、またはその直接の前提命題と合わせて読んだとき、意味が一意に確定することを確認する
- 直接の前提命題が文脈を確立している場合、重複する情報の省略を許容する

## 規則

- 偽側の連鎖は分割階層の各レベルで構築すること
- 連鎖の起点は分岐点の排除質問の reveals または S 原子命題であること
- 各ステップは `structure/汎用戦術.md` の戦術を使用すること
- 中間命題は導入時検証（T 文脈での真偽の明確さ + 意味の確定性）を通過すること
- 枝の深さは設計判断であり、アルゴリズムは固定しない
- 連鎖は一本鎖（直列）である必要はなく、分岐・合流を持つ構造として表現できる
- 連鎖は S の情報構造から自然に生まれるものであり、T からの逆算ではない
