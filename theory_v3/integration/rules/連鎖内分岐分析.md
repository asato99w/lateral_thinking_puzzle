# 連鎖内分岐分析

## 概要

真側連鎖の各ステップにおける並列命題を分析し、モデルとの対応を確認した上で、分岐から質問を導出する。

本工程は**反復的に実施する**。Step 1〜3 の導出と検証を別々に行い、問題があれば修正→再検証のサイクルで品質を上げる。導出と検証は必ず別ファイルに記録し、同時に生成しない。

### 設計思想

- 分割命題は全て T において真。プレイヤーは常に真側の回答を受け取るため、偽側に「進む」パスは存在しない
- 棄却は真側連鎖の各ステップの「反対側」として自然に生まれる
- 分岐の設計は戦術連鎖（命題と戦術）のレベルで行う。質問は分岐から導出される

## 入力

- Phase 1〜4 の全出力

## 出力

- 連鎖内分岐のデータ（各連鎖 × 各ステップの並列命題とモデル対応）
- カバレッジ結果
- 分岐から生成された命題と質問

## 手順

### Step 1: 連鎖内分岐の分析

真側連鎖（`_phase3.tactical_chains`）の各ステップについて、並列命題とモデルとの対応を分析する。

各連鎖の各ステップに対して:

1. そのステップの真側命題と、使用された戦術を確認する
2. **その戦術を同じ入力に適用した場合に到達しうる別の結論**を並列命題として特定する
   - 段階的絞り込みなら「別のどの方向に絞り込みうるか」、動機の探索なら「別のどんな動機がありうるか」等、戦術の種類に応じて考える
   - 真偽を問わない。T において真の並列命題もありうる
3. 各並列命題に対応するモデルを特定する
   - `_phase2.model_propositions` と照合する
   - **対応モデルがないことと並列が存在しないことは別である。** 戦術から導かれる代替結論が存在するなら、モデル集合に該当がなくても並列命題として記録する
4. 各並列命題が以下の検証基準を満たすことを確認する
   - **戦術テスト**: その並列は、同じ戦術を同じ入力に適用した場合に到達しうる結論か？ → いいえなら不適切
   - **否定テスト**: その並列は、真側命題の語尾を否定しただけではないか？ → はいなら具体化が不足

判断に迷う場合は `precedents/連鎖内分岐の判例.md` の具体例と照合する。

記録形式（各連鎖について）:

| ステップ | 戦術 | 真側命題 | 並列命題 | 対応モデル |
|---|---|---|---|---|
| （命題 ID） | （戦術名） | （命題ラベル） | （並列命題ラベル） | （モデル ID リスト） |

注意:
- 連鎖の最初のステップも省略しないこと
- 並列命題が複数ある場合は行を分ける
- 対応モデルが不明確な場合は「?」を付記し、Step 2 で判断する

### Step 2: カバレッジ確認

全競合モデルが、連鎖内分岐のいずれかの並列命題に対応しているかを確認する。

1. Step 1 で特定した全並列命題の対応モデルを集約する
2. 全競合モデル（Phase 1 の models から T を除く）と照合する
3. 結果を記録する

カバレッジの結果:
- **全カバー**: 全モデルがカバーされている → Step 3 へ
- **部分カバー**: 一部モデルが未カバー → 未カバーモデルとその理由を記録し、Step 3 へ進む

未カバーモデルへの対応はこの時点では決定しない。結果を記録し、判断は後に委ねる。

### Step 3: 質問生成

Step 1 で特定した全ての並列命題から、NF 命題と QF 質問を導出する。`rules/質問生成_v3.md` の規則に従って生成すること。

1. 各並列命題について、対応する命題を定義する
   - 命題 ID: `NF*` プレフィックスを使用する
   - formation_conditions を設定する（連鎖の文脈から定まる）
   - `rules/質問生成_v3.md` の「質問・回答の導入時検証（1次チェック）」を適用する
2. 各命題に対応する質問を生成する
   - 質問 ID: `QF*` プレフィックスを使用する
   - 回答は T の事実に基づく
   - `rules/質問生成_v3.md` の reveals の厳密性・prerequisites・客観性の規則を適用する
3. 各質問に reveals を設定する
   - reveals = 対応する分岐命題
   - 「いいえ」回答の質問は reveals 先命題に negation_of を設定し、否定対象命題（H*）を作成する

## 出力データ構造

### `_phase5`

```json
{
  "_phase5": {
    "chain_branches": [
      {
        "chain_id": "C1",
        "chain_path": "S → N1a → I1 → P1",
        "branches": [
          {
            "at_step": "N1a",
            "true_side_label": "味の不味さが理由ではない",
            "parallels": [
              {
                "label": "味の不味さが理由",
                "models": ["M7"]
              }
            ]
          }
        ]
      }
    ],
    "coverage": {
      "all_models": ["M1", "M2"],
      "model_coverage": {
        "M1": [{"chain": "C3", "step": "N4a"}],
        "M2": [{"chain": "C2", "step": "N2a"}]
      },
      "uncovered": [],
      "uncovered_notes": ""
    },
    "generated": {
      "propositions": [
        {"id": "NF1a", "source_chain": "C1", "source_step": "N1a", "parallel_index": 0}
      ],
      "questions": [
        {"id": "QF1a", "reveals_proposition": "NF1a"}
      ]
    }
  }
}
```

#### chain_branches

Step 1 の出力。各連鎖 × 各ステップの分岐分析結果。

- `chain_id`: 対応する真側連鎖 ID（`_phase3.tactical_chains` の ID）
- `chain_path`: 連鎖のパスの記述
- `branches`: 各ステップの分岐。配列の各要素:
  - `at_step`: 分岐が生じるステップの命題 ID
  - `true_side_label`: 真側命題のラベル
  - `parallels`: 並列命題の配列。通常は 1 要素だが、複数の自然な分岐がある場合は複数。各要素:
    - `label`: 並列命題のラベル（この段階では ID 未付与）
    - `models`: 対応するモデル ID リスト

#### coverage

Step 2 の出力。全モデルのカバレッジ状況。

- `all_models`: 全競合モデル ID（T を除く）
- `model_coverage`: モデル ID → カバーする分岐の参照リスト。各参照は `{chain, step}` で `chain_branches` 内の位置を指す。空配列は未カバー
- `uncovered`: 未カバーモデル ID のリスト
- `uncovered_notes`: 未カバーの理由や分析メモ

#### generated

Step 3 の出力。生成した命題・質問と `chain_branches` 内の位置との対応。

- `propositions`: 各要素:
  - `id`: 命題 ID（`NF*`）
  - `source_chain`: 由来する連鎖 ID
  - `source_step`: 由来するステップの命題 ID
  - `parallel_index`: `parallels` 配列内のインデックス
- `questions`: 各要素:
  - `id`: 質問 ID（`QF*`）
  - `reveals_proposition`: reveal する命題 ID

### propositions / questions への追加

Phase 4 で生成した配列に分岐命題・分岐質問を追加する。フィールドは `Proposition` / `Question` 型（`models.py` 参照）に準拠する。

## 反復構造

本工程は導出と検証を分離し、反復的に実施する。

### 各イテレーション

1. **導出**: Step 1〜3 を実施し、`history/` に記録する
   - ファイル名: `{nn}_連鎖内分岐分析_導出_{n回目}.md`
2. **検証**: 導出結果に対して以下の検証を実施し、別ファイルに記録する
   - ファイル名: `{nn}_連鎖内分岐分析_検証_{n回目}.md`
   - 各並列命題に戦術テスト・否定テストを適用
   - カバレッジの妥当性を確認
   - 生成された質問の品質基準を確認
3. 検証で問題がなければ完了。問題があれば次のイテレーションへ

導出と検証は**必ず別ファイルに記録し、同時に生成しない**。これにより導出時の盲点を検証時に発見する機会を確保する。

## 規則

- 分岐の分析は戦術連鎖（命題と戦術）のレベルで行うこと。質問レベルで考えない
- 並列命題は真偽を問わず自然に並ぶものを探すこと
- 連鎖の全ステップを分析対象とすること（最初のステップを省略しない）
- カバーされないモデルへの対応を強制しない。結果を記録し、判断は後に委ねる
- 質問は Phase 4 と同じ品質基準を満たすこと
- 導入時検証（T 文脈での真偽の明確さ + 意味の確定性）を適用すること
