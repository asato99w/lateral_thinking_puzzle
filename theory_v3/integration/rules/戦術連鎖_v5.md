# 戦術連鎖（v5）

## 概要

統合アルゴリズム v5 Phase 4（後半）の規則。S の原子命題から分割命題への戦術連鎖を構成し、その過程で**中間命題**を導入する。連鎖の reveals → fc 関係が質問の formation_conditions を定める。

**本フェーズの意義は、適切な中間命題の導入にある。** 戦術連鎖は「どこに命題が必要か」を発見し「何が何を導出するか」を構築するための分析手法であり、出力は戦術ラベルではなく命題群とその導出関係である。

v4 版からの変更:
- 判例参照の追加（`precedents/質問文の含意の判例.md`）
- Phase 2 の仮置き fc を出発点として連鎖を設計する
- 連鎖設計の結果として fc が確定する（Step 4 を新設）

## 入力

- Phase 3 前半の出力（S の原子命題・導出命題、initial_confirmed）
- Phase 2 の出力:
  - 分割命題群（導出関係グラフを含む）
  - **独立軸**: 導出グラフで他の命題から導出されないノード
  - **根命題**: 最も抽象的なノード（S から到達可能な候補）
  - **仮置き fc**: 導出の逆方向から設定された骨格
- `structure/汎用戦術.md`

## 出力

- 戦術連鎖群
- **Phase 3 で新たに導入された中間命題群**（`_phase3.generated_intermediates` に記録）
- **確定した fc**（仮置きの修正を含む）

## 連鎖の構造

### 根命題への連鎖: S → 根命題

S の原子命題を起点とし、根命題（最も抽象的な分割命題）に至る連鎖。

- 起点: S の原子命題群（initial_confirmed）
- 目標: Phase 2 で特定された根命題
- 根命題は S の情報のみから到達可能でなければならない
- **各独立軸に対応する根命題ごとに、独立した連鎖を設計する**

### 具象方向への連鎖: 根命題 → より具象的な分割命題

根命題の確認を起点とし、導出グラフに沿ってより具象的な分割命題に至る連鎖。

- 起点: 根命題および先行する連鎖で確認された命題
- 目標: 導出グラフ上で起点の一段具象側にある分割命題
- **仮置き fc が連鎖の起点を示す**: 目標命題の仮置き fc に含まれる命題群が、連鎖の起点となる
- 仮置き fc は連鎖設計の結果として修正されうる
- **reveals → fc 関係**: 先行する連鎖の reveals が後続の連鎖の fc に含まれる

## 手順

### Step 1: 根命題への連鎖の構成

各根命題について、S の原子命題から到達する戦術連鎖を構成する。

各根命題について:
1. 関連する S 原子命題を特定する
2. 汎用戦術を適用して根命題に至るステップを構成する
3. 各ステップの出力を中間命題として導入する（量的最低基準を満たすこと）

### Step 2: 具象方向への連鎖の構成

導出グラフに沿って、根命題からより具象的な分割命題に至る連鎖を構成する。

各分割命題（根命題以外）について:
1. 仮置き fc から起点命題を確認する
2. 起点命題 + S 原子命題から、汎用戦術を適用して目標命題に至るステップを構成する
3. 各ステップの出力を中間命題として導入する（量的最低基準を満たすこと）
4. 連鎖設計の結果、仮置き fc の修正が必要であれば修正する
5. 各連鎖の起点の由来（どの先行連鎖の reveals を使用するか）を明示する

### Step 3: 中間命題の形式化

Step 1-2 で導入した中間命題を整理する:

1. 全ての中間命題に ID を付与する
2. 各中間命題の formation_conditions を戦術連鎖のステップ構造から設定する
3. `_phase3.generated_intermediates` に記録する

### Step 4: fc の確定

Phase 2 の仮置き fc を、連鎖設計の結果に基づいて確定する。

- 仮置き fc が連鎖設計と整合する場合はそのまま確定する
- 連鎖設計の結果、追加の起点命題が必要と判明した場合は fc を修正する
- 中間命題が fc に含まれるべき場合は追加する
- **fc は連鎖設計の帰結であり、fc を先に決めて連鎖をそれに合わせるのではない**

#### fc とゲーム進行の関係

fc は質問の利用可能条件を決定する。エンジンは reveals 先命題の fc が confirmed で充足されたとき、その質問をオープンにする。

確定した fc は以下を満たすこと:
- **参照先の確認可能性**: fc 内の各命題が、initial_confirmed に含まれるか、先行する質問の reveals で confirmed されること
- **質問回答サイクルとの対応**: 連鎖の各ステップの出力命題は質問で confirm され、後続ステップの命題の fc を充足する。この連鎖が質問回答サイクルとしてゲーム進行を駆動する

### Step 5: 出力の検証

- 全ての分割命題に対して連鎖が構成されていること
- **全ての連鎖が最低 1 つの中間命題を含むこと**（量的最低基準）
- 全ての連鎖が命題 ID の連鎖で構成されていること（非形式テキストが残っていないこと）
- 各連鎖の起点が initial_confirmed または先行する連鎖の reveals に含まれること
- 連鎖間に循環依存がないこと
- Phase 3 で導入された中間命題が `_phase3.generated_intermediates` に記録されていること
- **fc が確定していること（仮置きのまま残っていないこと）**

## 中間命題の導入原則

### 本フェーズの核心

戦術連鎖の分析過程で、起点と終点の間のギャップを埋める中間命題を発見し、形式的な命題として導入する。

連鎖の各ステップの出力は**命題**でなければならない。非形式的なテキスト記述（「過去の体験の状況」等）で終わらせてはならない。テキスト記述が出てきた場合、それは形式化すべき命題の候補である。

### 中間命題が必要になるパターン

- **段階的絞り込み**: 起点と分割命題の間の距離が大きい場合、中間の抽象度の命題を導入する
- **具体化パターン**: 抽象版と具体版の命題ペアが必要な場合
- **因果の中間ステップ**: 直接的な接続が跳躍を含む場合
- **矛盾の解消**: 常識的仮説の排除の結果が非自明な場合

### 量的最低基準

**各戦術ステップにつき最低 1 つの中間命題を導入すること。** 連鎖が N 個の戦術ステップを使用する場合、中間命題は最低 N 個必要である（target は中間命題に含めない）。

- **戦術ステップ数 > 中間命題数は不合格**: 複数の戦術を使用しながら中間命題が戦術数より少ない場合、一部の戦術が命題を生成しておらず実質的に機能していない。戦術ラベルを貼るだけで中間命題を導入しないステップは連鎖の設計不備である
- **0 は不合格**: いかなる理由であれ、中間命題が 0 個の連鎖は不完全であり、再検討が必要である
- **対処**: 戦術数を削減する（実質的な変換が 1 つなら戦術も 1 つ）か、不足する戦術ステップに対応する中間命題を追加する

### 導入後の適切性レビュー

中間命題を導入した後、以下の観点でレビューする:

- **T 忠実性**: T の情報から逸脱していないか
- **プレイヤー視点の自然さ**: その中間ステップはプレイヤーの思考の流れとして自然か
- **質問への展開可能性**: その中間命題から自然な質問を生成できるか

レビューで問題が見つかった場合は、中間命題を**削除するのではなく、より適切な中間命題に置き換える**。量を減らすのではなく質を上げる方向で対処する。

### 中間命題の制約

- **T 忠実性**: T の命題の抽象化・一般化、T から論理的に導出可能な中間ステップに限定
- **独立性と客観性**: 他の命題と同じ品質基準を満たすこと

### 中間命題の導入時検証（1次チェック）

中間命題を導入する際、以下の 2 点を検証する。Phase 8 の検証は 2 次チェック（安全網）であり、ここでの 1 次チェックが本来の品質保証である。

#### T 文脈での真偽の明確さ

- 中間命題の真偽が T の事実から一意に定まることを確認する
- 真偽が異なる類似表現（例:「飲んだことがある」vs「飲んだことがあると信じている」）を混同しないこと

#### 意味の確定性

- 命題単体、またはその直接の前提命題と合わせて読んだとき、意味が一意に確定することを確認する
- **直接の前提命題**: 連鎖において当該中間命題の直上に位置する命題群。formation_conditions とは異なる概念である
- **省略の可否**: 直接の前提命題が文脈を確立している場合、重複する情報の省略を許容する。確立されていない場合は完全な記載を要する

## 戦術ステップの形式

各ステップは以下を明示する:

- 入力命題集合（命題 ID）
- 適用する汎用戦術
- 出力命題（**命題 ID**。非形式テキストは不可）

```
{命題ID群} →[戦術]→ 命題ID
```

## プレイヤー視点の検証

全ての戦術連鎖に対して、以下の観点で検証する:

1. **入力からの連想は自然か**: プレイヤーがその入力情報を持っている時、その方向に考えることが自然か
2. **結論から逆算していないか**: T からの逆算ではなく、S の情報構造から自然に生まれる連鎖か
3. **戦術の適用が適切か**: why/how/what/who/when の区別が正しいか
4. **跳躍が大きすぎないか**: 中間命題の追加が必要ではないか

## 失敗パターンと対策

### 誤り: T からの逆算

- S の情報構造からではなく、T の答えから逆算して連鎖を構成した
- **対策**: 連鎖は S の原子命題から出発し、汎用戦術の自然な適用で進むこと

### 誤り: 中間命題を導入しない連鎖

- 既存命題間に戦術ラベルを貼るだけで、中間命題を導入しなかった
- 連鎖の中間結果を非形式テキストで記述し、正式な命題にしなかった
- **対策**: 連鎖の各ステップの出力は命題 ID であること。テキスト記述が出てきたら、形式化すべき命題の候補として扱う。**中間命題が 0 個の連鎖は不合格であり、再検討すること**

### 誤り: 段階的絞り込みの具体例省略

- 段階的絞り込みを戦術として使う際、中間命題の具体例を挙げずに一足飛びに書いた
- **対策**: 段階的絞り込みを使う場合、少なくとも 1 つの具体的な中間命題を正式に導入する

### 誤り: 戦術の不適切な適用

- 「なぜ」（因果の逆転）と「どうやって」（手段の探索）を混同した
- **対策**: 戦術の入出力定義を確認し、適切な戦術を選択する

### 誤り: 連鎖の起点の曖昧さ

- 連鎖の起点がどの先行連鎖の reveals に由来するか不明確
- **対策**: 各連鎖の起点の由来を明示的に記録する

### 誤り: 仮置き fc への過度な依存

- Phase 2 の仮置き fc をそのまま確定させ、連鎖設計を仮置きに合わせてしまった
- **対策**: 仮置き fc は出発点であり、連鎖設計の結果として修正されることが正常である。連鎖の自然さを優先し、fc は帰結として確定する

## 規則

- 全ての分割命題に戦術連鎖が定義されること
- 使用する戦術は汎用戦術リスト（`structure/汎用戦術.md`）に含まれること
- **各戦術ステップの出力は命題 ID であること（非形式テキストは不可）**
- **中間命題を導入しない連鎖出力は不完全であり、再検討すること**（量的最低基準: 各戦術ステップにつき最低 1 つ）
- 根命題への連鎖の起点は S 原子命題（initial_confirmed）であること
- 各連鎖の起点の由来（どの先行連鎖の reveals を使用するか）を明示すること
- 連鎖間に循環依存がないこと
- 全ての連鎖についてプレイヤー視点の検証を実施すること
- 戦術連鎖は S の情報構造から自然に生まれるものであり、T からの逆算ではないこと
- **Phase 3 で導入した中間命題は `_phase3.generated_intermediates` に記録すること**
- **中間命題は T 忠実性・独立性・客観性を満たすこと**
- 段階的絞り込みを使う場合、少なくとも 1 つの具体的な中間命題を正式に導入すること
- **fc は連鎖設計の結果として確定すること（仮置きのまま残さない）**
