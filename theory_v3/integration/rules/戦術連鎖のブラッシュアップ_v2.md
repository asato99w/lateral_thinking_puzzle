# 戦術連鎖のブラッシュアップ（v2）

## 概要

Phase 6（検証）通過後に実施するブラッシュアップ工程。2 つの目的を持つ:

1. **跳躍の解消**: 各質問の利用可能条件（reveals 先命題の fc）に跳躍がないかをレビューし、跳躍がある場合は戦術連鎖を修正して解消する
2. **fc の N*/NF* 置き換え**: fc に含まれる N*/NF*（「いいえ」で reveal される命題）を「はい」命題に置き換える

v1 版（`戦術連鎖のブラッシュアップ.md`）からの変更:
- N*/NF* 置き換え手順の追加（Step 1.5, Step 2 の拡張）

本工程は**反復的に実施する**。1回で全問題を解消する必要はなく、改善→再評価→さらに改善のサイクルで段階的に品質を上げる。

## 入力

- Phase 8 を PASS した data_src.json
- `precedents/想起条件の判例.md`

## 出力

- 跳躍の指摘一覧
- 修正後の戦術連鎖・中間命題・質問

## 更新するデータ

本工程ではデータを以下の順序で更新する。**連鎖の修正が先、fc の変更はその帰結**である。fc を直接編集して連鎖を変更しない対応は不可。

1. `_phase3.tactical_chains`: 連鎖の description を修正後の構造に書き換える
2. `propositions`: 新たな中間命題を追加する（入れ子化で生まれた命題）
3. `propositions`: 既存命題の `formation_conditions` を修正後の連鎖構造から再導出する
4. `questions`: 新たな中間命題を reveals する質問を追加する

## 手順

### Step 1: 跳躍の指摘

全質問について、利用可能条件（reveals 先命題の fc）の命題群が known のとき、その質問が**推論なしに頭に浮かぶか**を評価する。

評価基準:
- reveals される命題の fc に記載された命題群から質問へ至るまでに複雑な推論が介在する場合、跳躍と判定する
- 「1段階推論として妥当」と「自然に想起される」は別の基準。推論として成立しても、推論を経ないと想起できないなら跳躍である
- **評価は fc に記載された命題のみに基づいて行う。プレイヤーがその時点で蓄積している他の知識を暗黙に参照してはならない。** fc に含まれていない命題が想起に必要なら、跳躍と判定し Step 2 で連鎖を修正して解消する
- 判断に迷う場合は `precedents/想起条件の判例.md` の具体例と照合する

各質問について以下を記録する:
- **判定**: OK / 跳躍 / 跳躍（境界）
- **理由**: なぜ自然か、または何の推論が介在するか

### Step 1.5: fc の N*/NF* の特定と置き換え検討

全命題の fc を走査し、N*/NF*（「いいえ」で reveal される命題）が含まれているものを特定する。

各 N*/NF* について:

1. **役割の特定**: その N*/NF* が fc の中で想起の文脈として何を担っているかを特定する
2. **既存「はい」命題での代替**: 同じ想起を支える既存の「はい」命題（P*, I*）があれば、それに置き換える
3. **新規「はい」中間命題の導入**: 既存にない場合、戦術連鎖を見直し、N*/NF* が担っていた文脈を「はい」の中間ステップとして形式化する（Step 2 で連鎖を修正）
4. **連鎖の再設計**: 置き換えが困難な場合、連鎖構造自体の再設計を検討する（Phase 4 差し戻し）

### Step 1.7: prerequisites の妥当性レビュー

全質問について、質問文の言語的前提が prerequisites（または fc 経由で保証される命題）でカバーされているかをレビューする。

手順:
1. 各質問の質問文を読み、テキストが前提としている概念を列挙する
2. 各前提概念について、以下のいずれかで保証されているか確認する:
   - initial_confirmed に含まれる（S 命題）
   - prerequisites に含まれる
   - fc 経由で必ず confirmed/derived になる命題から含意される
3. 保証されていない前提がある場合、prerequisites への追加で対処する
4. prerequisites の追加で対処できない場合（前提概念に対応する命題が reveals に存在しない場合）、質問文の修正を検討する

評価基準は `precedents/質問文の含意の判例.md` に従う。各質問について以下を記録する:
- **判定**: OK / 不足 / 不足（修正済み）
- **理由**: 何が前提とされ、何で保証されているか（不足の場合は何が不足か）

### Step 2: 戦術連鎖の修正

跳躍と判定された質問、Step 1.5 で N*/NF* 置き換えが必要な箇所、または Step 1.7 で prerequisites 不足が発見された箇所について、対応する戦術連鎖を修正する。

修正の手段:
- **入れ子化**: 戦術の内部にサブ戦術を入れ子にして、自然な想起の粒度まで分解する
- **外側の調整**: 戦術自体の差し替えや、連鎖の組み替えを行う

入れ子化と外側の調整は排他的ではなく、同時に行ってもよい。跳躍の原因に応じて適切な手段を選択する。

新たな中間命題が生まれる場合、導入時検証（1次チェック）を適用する。

### Step 3: 質問の生成

Step 2 で追加・変更された中間命題に対応する質問を生成する。

- 新たな中間命題を reveals する質問を追加する
- 入れ子化により形成条件が変わった命題の fc を更新する
- 質問生成の規則（Phase 4）に従い、導入時検証を適用する

### Step 4: 再評価

修正後、以下を実施する:

1. 本イテレーションの実施記録を `history/` に書く（ファイル名: `{nn}_ブラッシュアップ_{n回目}.md`、nn は連番）。Step 1 の跳躍判定一覧、Step 2-3 の修正内容、データ更新箇所を記録する
2. Phase 6（検証）の該当項目を再実施する
3. Step 1 に戻り、修正後の質問に対して跳躍の再評価を行う

跳躍が残っている場合、fc に N*/NF* が残っている場合、または prerequisites の不足が残っている場合は Step 1 に戻り反復する。全質問が OK または許容可能な水準に達し、全 fc から N*/NF* が除去され、かつ全質問の prerequisites が妥当であるまで反復する。
