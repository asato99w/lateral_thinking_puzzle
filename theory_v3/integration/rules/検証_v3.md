# 検証（v3）

## 概要

統合アルゴリズム v3 Phase 8 の規則。大域目標の達成と各種整合性を検証する。

**本 Phase は 2 次チェック（安全網）として機能する。** 命題の品質保証は、各命題が導入される Phase（Phase 2: 分割命題、Phase 3: 中間命題、Phase 4: 質問・回答）での 1 次チェックが本来の防御線である。本 Phase では、1 次チェックで見落とされた問題を捕捉する。

v2 版（`検証.md`）からの変更:
- ピース・トリガー・clear_conditions 関連の検証を廃止
- fc 連鎖の整合性、棄却の完全性を追加
- 分割命題と棄却構造に基づく検証に置き換え
- パストレース、reveals の厳密性、prerequisites の確立可能性、1 段階推論原則は維持

## 入力

- Phase 1〜7 の全出力

## 出力

- 検証結果（PASS/FAIL）
- 各パスのトレース
- FAIL 時の差し戻し先

## 手順

### Step 1: 構造的整合性

- 全命題の ID が一意であること
- 全質問の reveals が存在する命題 ID を参照していること
- recall_conditions / formation_conditions が存在する命題 ID を参照していること
- initial_confirmed の命題が全て定義されていること

### Step 2: fc 連鎖の整合性

全質問の fc（recall_conditions）が初期確認済み命題から到達可能であることを検証する。

- fc に含まれる S 命題が initial_confirmed に含まれること
- fc に含まれる分割命題が、先行する質問の reveals に含まれること
- 循環依存がないこと（質問 A の reveals が質問 B の fc に含まれ、かつ質問 B の reveals が質問 A の fc に含まれる状態がないこと）

### Step 3: recall = fc の整合性

各質問について、recall_conditions が reveals される命題の formation_conditions と一致していることを検証する。

- reveals を持つ質問: recall_conditions = reveals[0] の formation_conditions
- reveals が空の質問: recall_conditions が連鎖の source 命題に基づいていること

不一致がある場合は FAIL。Phase 4 に差し戻す。

### Step 4: reveals の網羅性と完全性

- 全分割命題が少なくとも 1 つの質問の reveals に含まれること
- **全質問が reveals を持つこと**（reveals: [] は原則あり得ない。排除質問も否定文を中間命題として reveals に含める）
- 棄却可能な全モデルが、対応する分割命題の質問回答により棄却される経路が存在すること

### Step 5: T 整合性と命題品質（2次チェック）

以下は各 Phase の 1 次チェックで検証済みの項目を再検証する安全網である。

#### 回答の T 整合性

- 全質問の回答（はい/いいえ）が T の事実と矛盾しないこと
- 分割命題の truth_value が T の内容と整合すること

#### 全命題の T 文脈での真偽の明確さ

- 全ての分割命題・中間命題について、T の事実から真偽が一意に定まることを確認する
- 真偽が異なる類似表現の混同がないことを確認する

#### 全命題の意味の確定性

- 各命題が、その直接の前提命題と合わせて読んだとき、意味が一意に確定することを確認する
- 直接の前提命題で文脈が確立されていない場合に、省略による曖昧さがないことを確認する

### Step 6: reveals の厳密性

各質問について、reveals に含まれる命題が質問テキストと回答（Yes/No）から厳密に論理的に導かれるかを確認する。

### Step 7: prerequisites の対話上確立可能性

各質問について、prerequisites に設定された命題が以下のいずれかに含まれているかを確認する:
- initial_confirmed
- いずれかの質問の reveals

formation_conditions でのみ導出される命題が prerequisites に含まれている場合は FAIL。Phase 4 に差し戻す。

### Step 8: 導出条件の 1 段階推論原則

全命題の formation_conditions について、各 AND グループが単一の推論ステップとして妥当であるかを確認する。結論から逆算した不自然な条件がないかを検証する。

### Step 9: パストレース

複数の質問選択順でゲームをシミュレーションする。

1. confirmed = initial_confirmed から開始
2. 論理的導出（entailment_conditions 不動点）→ confirmed 拡大
3. 仮説導出（formation_conditions 1回パス）→ derived 計算
4. known = confirmed ∪ derived
5. recall_conditions が known で満たされる質問を列挙（オープン質問）
6. オープン質問から 1 つ選択 → reveals を confirmed に追加
7. 2〜6 を繰り返す

確認事項:
- **到達可能性**: いかなる選択順でも、棄却可能な全モデルが棄却される状態（T 近傍）に到達可能であること
- **行き詰まり**: オープン質問がなくなり、かつ分割命題が未確認の状態が発生しないこと
- **想起の合理性**: 各質問の開放が唐突な飛躍を含まないこと

最低 3 パスを異なる入口から検証する。

### Step 10: 棄却の完全性

- 探索パスの終了時に、棄却可能な全モデルが棄却されていること
- 残存するモデルが棄却不可能モデル（Phase 5 で特定済み）のみであること

## 検証条件

全ステップが PASS であること。

## FAIL 時の差し戻し

| 検証項目 | FAIL 時の差し戻し先 |
|---|---|
| 構造的整合性 | 該当フェーズ |
| fc 連鎖の整合性 | Phase 3, 4, 6, または 7 |
| recall = fc の整合性 | Phase 4 または Phase 7 |
| reveals の網羅性 | Phase 4 または Phase 7 |
| T 整合性と命題品質 | Phase 2, 3, 4, 6, または 7 |
| reveals の厳密性 | Phase 4 または Phase 7 |
| prerequisites の確立可能性 | Phase 4 または Phase 7 |
| 1 段階推論原則 | Phase 3, 4, 6, または 7 |
| パストレース（到達不可） | Phase 3, 4, 6, または 7 |
| パストレース（行き詰まり） | Phase 3, 4, 6, または 7 |
| 棄却の完全性 | Phase 2 または Phase 5 |

## 規則

- 全検証項目を順に実施すること
- FAIL の場合、差し戻し先を明示して修正を行うこと
- パストレースは最低 3 パスを異なる入口から検証すること
- 棄却不可能モデルの存在は FAIL ではない（Phase 5 で許容済みの場合）
