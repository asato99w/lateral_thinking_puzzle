# 冗長命題の間引き

## 概要

工程3（棄却側の構築）完了後、質問分類の前に実施する後処理。導出関係上で概念的距離が不十分な命題ペアを特定し、冗長な側を間引いて質問数を削減する。

### 設計思想

Phase 4 の量的最低基準は「戦術ステップごとに中間命題を導入する」ことを要求するが、結果として中間命題と target が同義反復になるケースがある。これは戦術連鎖の設計段階では避けがたい（概念的距離が十分かは事後的にしか判断できない）。本 Phase で事後的に間引くことで、量的基準の保証と冗長性の排除を両立する。

## 入力

- Phase 1〜8 の全出力（全質問・命題が確定した状態）

## 出力

- 間引き対象の一覧と判断根拠
- 修正後の propositions / questions
- 修正後の fc

## 手順

### Step 1: 冗長ペアの特定

全ての導出関係（Y.fc に X が含まれる = X は Y の導出元）について、X と Y の概念的距離を評価する。

**判定基準**:
- **ラベルの類似度**: X と Y のラベルが同義・言い換えであるか
- **情報の差分**: Y が X に対して追加する情報が実質的に存在するか
- **質問の区別可能性**: X と Y に対応する質問がプレイヤーにとって異なる問いに感じられるか

**冗長ペアの典型パターン**:
- 否定↔肯定の言い換え（「不利益ではなかった」↔「利益をもたらした」）
- 修飾語の追加のみ（「察知して治した」↔「その場で察知して治そうとした」）
- 別連鎖での意味的重複（異なる連鎖で同一の否定情報を持つ I* と N*）

**冗長でないペアの特徴**:
- 抽象度が明確に異なる（「何かに困っていた」→「身体的・心理的な不調を抱えていた」）
- 新しい軸の情報を追加する（「不調がある」→「水で緩和できる不調」）
- プレイヤーの思考の段階として区別可能（段階的絞り込みが機能している）

### Step 2: 間引き判断

各冗長ペアについて、どちらを間引くか判断する。

**デフォルト: 導出元を間引く**

導出元（fc で参照される側、上流の命題）を間引き、導出先（下流の命題）を残す。

理由:
- 導出先は Phase 2-3 で先に設計された構造的命題（分割命題・中間命題）であり、model分割データや導出グラフ上の位置を持つ
- 導出元は Phase 4 で橋渡しとして追加された中間命題であり、冗長 = 橋渡しの不成功を意味する
- 導出先を残す方が fc の付け替えが単純（導出先の fc を 1 段飛ばしにするだけ）

**例外: 導出元のラベルが明らかに優れている場合**

導出元を残し、導出先を間引く。この場合:
- 導出先の model分割データを導出元に移転する
- 導出先を参照する他の命題の fc を導出元に差し替える
- 判断根拠を明示すること

### Step 3: データ修正

間引く命題 X について、以下の修正を順に実施する。

#### 3a: fc の付け替え

X を fc 内で参照する全命題 Y について、Y.fc 内の X を **X 自身の fc の内容で置換**する。

例: X.fc = [[A, B]], Y.fc = [[X]] → Y.fc = [[A, B]]
例: X.fc = [[A]], Y.fc = [[X, C]] → Y.fc = [[A, C]]

#### 3b: prerequisites の付け替え

X を `prerequisites` で参照する全質問について、X を **X の fc 内容に含まれる非 S 命題で置換**する。X の fc が S 命題のみで構成される場合は、X を prerequisites から単純に除去する（S 命題は initial_confirmed に含まれるため prerequisites に不要）。

例: X.fc = [[P1, S5]], Q.prerequisites = [X] → Q.prerequisites = [P1]
例: X.fc = [[S10]], Q.prerequisites = [X] → Q.prerequisites = []

#### 3c: 命題の削除

`propositions` から X を削除する。X に `negation_of` がある場合、対応する命題の `negation_of` も null にする。

#### 3d: 孤立した negation_of 相手の削除

X の negation_of 相手（H* 等）が存在し、X の削除により孤立する場合（その H* を reveal する質問も fc で参照する命題もない場合）は、H* も削除する。

#### 3e: 質問の削除

X を `reveals` に含む質問を `questions` から削除する。削除した質問の reveals に含まれる他の命題がある場合は、別の質問で reveals されることを確認する。

#### 3f: 連鎖データの更新

`_phase4.tactical_chains` の該当連鎖から:
- `intermediates` から X を除去
- `tactic` のステップ記述を更新（戦術数を削減）
- `generated_intermediates` から X を除去

#### 3g: 分岐データの更新

`_phase8.chain_branches` から X のステップの分岐エントリを除去する。その分岐から生成された NF/QF がある場合:
- 対応する NF 命題を `propositions` から削除
- 対応する QF 質問を `questions` から削除
- 対応する H* 命題を `propositions` から削除
- `_phase8.generated` から除去

#### 3h: メタデータの更新

- `_phase5.chain_mapping` から削除した質問を除去
- その他の Phase メタデータを整合させる

### Step 4: 再検証

`run_all.py` で整合性を確認する。FAIL の場合は修正する。

## 規則

- 間引きは冗長ペアに限る。概念的距離が十分なペアは間引かない
- **デフォルトで導出元を間引く。例外は判断根拠を明示する**
- 間引き後に連鎖の量的最低基準（戦術ステップ数 ≤ 中間命題数）を満たさなくてもよい。間引きフェーズは量的基準より優先する
- 間引きにより消滅したステップの分岐（NF/QF）も合わせて削除する
- 再検証で FAIL の場合は修正する
- 間引き結果を `_phase10` に記録する
