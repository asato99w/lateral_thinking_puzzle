# 逆算連鎖

## 概要

統合アルゴリズム v3 Phase 1 の規則。T を「なぜ？」の連鎖で分解し、T の命題を抽出する。

## 入力

- T: 真相

## 出力

- T の直接命題群（T テキストの因果分解）
- T の導出命題群（因果充足性のために必要な、T から論理的に含意される命題）

出自情報は `_phase1` メタデータとして記録する（`t_descriptors`, `t_derived_descriptors`）。命題の ID は統一的な命名（D-*）を使用する。

## 手順

### Step 1: T の逆算

T の結論から「なぜ？」を繰り返し、因果の連鎖を最深部（前提）まで辿る。各ステップが T の直接命題の候補となる。

### Step 2: T の直接命題の抽出

逆算連鎖の各ステップを T の直接命題として列挙する（`_source: "T"`）。

### Step 3: 因果充足性の検証と T 導出命題の抽出

Step 1 の各逆算ステップ「A ← B（B が A の理由）」について、因果の充足性を検証する。

**手順:**

1. B の命題群だけで A が必然的に説明されるかを検証する（充足性テスト）
2. 不十分な場合、欠落している前提条件 C を特定する
3. C が以下の3条件を全て満たすとき、T 導出命題として抽出する（`_source: "T-derived"`）

**T 導出命題の認定条件:**

| 条件 | 内容 | 検証方法 |
|------|------|----------|
| 含意テスト | T の直接命題群の組み合わせから論理的に導かれる | C が T の直接命題群のどの組み合わせから帰結するかを明示する |
| 必要性テスト | C がなければ当該因果リンクが成立しない | C を否定したとき因果が破綻することを示す |
| 内部性テスト | T の世界内の事実であり、外部知識・価値判断・程度評価を含まない | C が客観的に検証可能な命題であることを確認する |

**出力形式:** 各 T 導出命題について、導出元の直接命題 ID と3条件の検証結果を記録する。

## 規則

### T の直接命題に対する規則

- 各命題は他の命題との論理演算を前提としない独立した単位とする
- 各命題は客観的に検証可能な命題であること。主観的な判断（「深刻な」「強い」等の程度表現）や評価的表現を含まない
- 行為の目的・解釈を命題に混入させない（例: 「捨てて軽量化した」→「捨てた」）
- 程度の評価（「不十分だった」）や必要性の判断（「必要があった」）は客観的事実に置き換える
- 逆算連鎖の各ステップは「なぜ？」に対する必然的な回答であること
- T の直接命題は T のテキストに含まれる情報の分解であり、解釈・推測による情報の追加を行わない

### T の導出命題に対する規則

- T の直接命題と同じ品質基準（客観性・独立性・検証可能性）を満たすこと
- 認定条件（含意・必要性・内部性）の3つ全てを満たすこと
- 導出の根拠（どの直接命題群から帰結するか）を明示すること
- 推測や蓋然性の判断（「〜の可能性がある」）は T 導出命題として認められない。T の因果構造から必然的に導かれるもののみを対象とする

### 独立性における暗黙の依存パターン

命題は自己完結的でなければならない。以下のパターンに注意する:

- **時系列への依存**: 「下がり続けた」「後も下がった」は過去からの継続・先行状態を前提としており独立性に反する
- **対処**: 依存先の内容を文中に含めて自己完結させる（例: 「高度低下への対処が行われた後も高度が下がっていた」）。短くしようとして依存を省くのではなく、複合文にしてでも独立性を保つ
- **独立であれば複合的な文でよい**: 「荷物を全て捨て服を脱いでも熱気球の高度低下は止まらなかった」は他の命題と内容が重複するが、自己完結しているので独立性を満たす
