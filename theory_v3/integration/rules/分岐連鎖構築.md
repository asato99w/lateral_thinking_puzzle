# 分岐連鎖構築

## 概要

統合アルゴリズム v3 Phase 6 の規則。Phase 3〜4 で構築した T 方向の連鎖に分岐構造を追加する。

Phase 5（棄却分析）の結果を前提に、偽側の枝の構築・内部分岐・合流の設計を扱う。

## 入力

- Phase 1〜4 の全出力
- Phase 5（棄却分析）の出力

## 出力

- 偽側の枝（新たな連鎖・質問）
- 既存連鎖の内部分岐
- 合流による fc の OR 構造（代替導出経路）

## 手順

### Step 1: 偽側の枝の構築

分割階層の各レベルで、偽側の枝を構築する。

#### 対象

分割階層の各分割命題について、偽側（false_set）のモデル群の方向を探索する枝を設計する。

例（turtle_soup 05 の分割階層）:
```
P1=真 {T,M1,M2,M3,M4,M8,M12}  ← 既存の連鎖あり
  P2=真 {T,M1,M4}
    ...
  P2=偽 {M2,M3,M8,M12}         ← 偽側の枝が必要
P1=偽 {M5,M6,M7,M9,M10,M11}   ← 偽側の枝が必要
```

#### 枝の設計方針

- 各枝は分岐点（分割命題を問う質問の位置）から伸びる
- 枝は偽側の方向を探索し、行き止まり（棄却）に至って真側に収束する構造とする
- 偽側の内部を詳細に分割する必要は必ずしもない。棄却命題への到達経路があれば十分
- 枝の長さ・深さは難易度パラメータ。短ければすぐに T 方向に収束し簡単になり、長く深ければ多くの探索を経験する

#### 枝の構成要素

- 偽側の方向を示す中間命題（排除質問で導入される命題の否定形等）
- 偽側の探索で到達する棄却命題
- 棄却から真側への収束（既存の連鎖への合流）

### Step 2: 既存連鎖の内部分岐

Phase 3〜4 で構築した既存の連鎖の各ステップで、反対の回答に進む枝を設計する。

- 既存のノードから分岐するため、新たな連鎖の構築は不要
- 各質問の「いいえ」回答の場合の展開を検討する
- 浅い設計で十分。全ステップの完全な分岐設計は不要

### Step 3: 合流の検討

Step 1, 2 の設計過程で、異なる枝が同じ命題に合流する構造を意識する。

- 合流が生じれば、その命題の fc に自然に複数の AND グループ（代替導出経路）が生まれる
- 代替導出経路があれば、プレイヤーが異なる質問順序で探索しても同じ命題に到達でき、探索の自由度が上がる
- 合流の強制は不要。自然に生じる場合に取り入れる

合流の判例は `precedents/戦術連鎖の構造の判例.md` を参照。

## 規則

- 連鎖は一本鎖（直列）である必要はなく、分岐・合流を持つ構造として表現できる
- 偽側の枝は分割階層の各レベルで設計すること
- 枝の深さは設計判断であり、アルゴリズムは固定しない
- 既存連鎖の内部分岐は浅く設計すること
- 合流は自然に生じる場合に取り入れること。強制しない
