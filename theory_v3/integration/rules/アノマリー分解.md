# アノマリー分解

Phase 1（構造分解）で、隠れアノマリー特性を持つパズルに適用する規則。

## 適用条件

登場人物のパラダイムが破れ、推論連鎖で真相に至る構造がある。

## 最終出力

FP を根、共通抽象命題を葉とする導出グラフ。背景事実と認知状態は同列のノードとして並び、S 変数を葉ノードに付与する。

## 概念定義

| 名称 | 定義 |
|---|---|
| 演繹予測 | 認知状態から演繹される、未来に対する予測 |
| 棄却前提 | 演繹予測の反証により誤りと判明する前提 |
| 主動因命題 | Sの行為・反応を了解可能にする中心的な事実 |
| 鍵 | 主動因命題の中で、置き換えると動因特性が変化する概念 |
| 背景事実 | 演繹予測および鍵導出の認知状態を構成するために必須のT由来前提群（鍵を除く） |
| 導出元集合族 | 主動因命題を導出する命題集合の族。各要素集合の全命題が揃えば導出される（必要条件）。族としてはいずれかの集合が充足すれば十分（OR関係） |
| 導出元部分集合 | 導出元集合族の要素集合の部分集合。全体が揃わなくても導出に部分的に寄与する |

## 手順

### Step 1: 演繹予測の具象的記述

演繹予測を完全に具象的な命題として記述する。未来に対する予測の形式とする。

規則：
- 登場人物の信念と事実を混同しない（例: 「ウミガメのスープを食べた」ではなく「ウミガメのスープだと言われて飲んだ」）
- 予測の対象を具体的に特定する

### Step 2: 演繹予測の前提命題の抽出

Step 1 の具象的命題から、その認知を持つために必須の前提命題を抽出する。

規則：
- 命題に内包された前提を抽出する操作として行う
- 認知的フレーミング（「記憶がある」等）は命題に付加しない。具象的な事実として記述する
- 各前提がS由来かT由来かを記録する

### Step 3: 主動因命題の特定

Sの行為・反応を了解可能にする中心的な事実を特定する。

### Step 4: Tの原子命題分解と主動因命題の導出元集合の特定

Tを原子命題に分解し、主動因命題の導出元集合を特定する。

規則：
- 各命題は自己完結的・意味確定的な形式で記述する（文脈依存の表現「それ」「その」等は使わない。主語・対象・相手を明示する）
- 一般知識は含めない。物語固有の事実のみ
- 鍵を含む具象的な形で記述する
- 主動因命題の導出元集合（主動因命題を導出する命題の集合）を特定する
- アノマリー解消の帰結が導出元に含まれる場合、そのように記録する
- 各命題の前提（意味の成立に必要な命題）を特定する。前提関係は時系列や因果関係ではなく、プレイヤーにとってその命題が意味を持つかどうかで判断する
- 各命題の導出元部分集合を特定する（導出の十分条件ではないが寄与するもの）

### Step 5: 鍵の特定

Step 4 の前提命題群の中で、置き換えると動因特性（Sの行為を了解可能にする力）が変化する概念を鍵として特定する。

### Step 6: 背景事実の収集

Step 4 のT原子命題から鍵（Step 5）を除いたものを背景事実として収集する。各背景事実について前提と導出元部分集合を記録する。

### Step 7: アノマリー解消の認知的過程の列挙

アノマリーから解消に至る登場人物の認知状態の推移を具象的事実として列挙する。

規則：
- 一般知識は推論の道具であり、推論が行われたこと自体はT由来の具象的事実である。「一般知識で完結」という判断を安易に行わない
- 各ステップが外部から観察可能な行動か、内的な認知体験かを区別する。Sに含まれる行動（例: 「本当にウミガメのスープかと尋ねた」）と、T由来の内的体験（例: 「以前に飲んだ時と味が違うと感じた」）は認知的に異なる
- 各ステップについて以下を記録する:
  - 使用される一般知識（推論の道具）
  - プレイヤーにとっての認知的前提（この情報が意味を持つために何を知っている必要があるか = fc根拠）
- 粒度は細かすぎるとくどい。ゲーム体験として適切な粒度の判断が求められる
- 認知状態は背景事実と同列のノードとして扱う

### Step 8: 語用論的含意の検証と再構成

背景事実および認知的過程の各ステップについて、言語的形式が棄却前提の否定や鍵を語用論的に含意していないか検証する。

規則：
- 語用論的含意（グライスの会話の含意）に注意する。有標な表現（「と言われて」「何かを」等）が暗黙の否定を含意しうる
- 単独での含意だけでなく、他の命題と組み合わさった時の含意にも注意する
- 含意がある場合、他の背景事実から想起される具象命題の形に再構成する

### Step 9: Sの緊張の列挙と競合モデルの列挙

#### Sの緊張の列挙

Sを読み、プレイヤーが感じる「なぜ？」「どういうこと？」を生む緊張を特定する。

緊張構造とは、Sの記述の中で以下のいずれかに該当する箇所:
- **因果の欠落**: 行動や結果の理由が示されていない
- **期待との不一致**: 通常の予測に反する展開がある
- **情報の不足**: 状況を理解するために必要な情報が隠されている

各緊張について以下を記述する:
- 緊張の内容（Sのどの部分が、何に対する緊張を生むか）
- 常識的説明（M0）の予測: その状況に対して常識的に予測される展開
- M0 では説明不能な点

#### 緊張の分解

各緊張に対して、以下の3基準を順に適用し、独立した細粒度の緊張に分解する。

**基準1: 程度の分離**
緊張の中に「程度・深刻さ」自体が独立した驚きを生む要素がある場合、メカニズムの緊張と程度の緊張を分離する。

**基準2: 前提の抽出**
ある行動が成立するために必要な前提条件が非自明である場合、その前提自体を独立した緊張として抽出する。

**基準3: 独立検証**
分解した各緊張について、異なるモデルによって独立に緩和可能であることを確認する。両方とも想像できない場合、分解は冗長であり実施しない。

規則：
- 分解はSの記述から読み取れる情報に基づくこと。Tの知識から逆算して分解を導かない
- 独立検証（基準3）を通過しない分解は実施しない
- Sの緊張とアノマリーのS露出部分を混同しない。Sの緊張は説明を要する行為・反応（例: 安堵した、自殺した）であり、S露出部分はアノマリーのうちSに現れている観察事実（例: 毒キノコしかない）である

#### 競合モデルの列挙

各緊張に対して、Tの枠組みを前提としない競合モデルを列挙する。

規則：
- Tの枠組みを暗黙の前提としない。T方向の解釈の変奏ではなく、根本的に異なる解釈を含める
- T も競合モデルの1つとして同列に含める
- 各モデルはSの事実と整合すること
- モデルは具象的であること
- 各緊張について最低5つの競合モデルを列挙する

### Step 10: 共通抽象の導出と評価

背景事実・認知状態と競合モデルの間で共通抽象を導出する。

規則：
- 可能な限り具象的で意味のある命題を求める
- 無理に抽象化せず、具象に近い形で留める
- 対応する命題がない場合は「対応なし」と記録する
- 以下の基準で評価し、基準を満たさないものを除外する:

| 基準 | 内容 |
|------|------|
| 客観性 | 主観や解釈に依存しないか |
| 意味の確定性 | 「何か」「何らかの」を含んでいないか |
| 非トートロジー性 | 情報量があるか |
| 弁別力 | 競合モデル間を区別する力があるか |

共通抽象は導出グラフの葉に位置する。

### Step 11: fc関係の定義

背景事実および認知状態の間の想起関係をfcとして定義する。

規則：
- プレイヤーの認知的前提（「この情報が意味を持つために何を知っている必要があるか」）がfcの根拠となる
- 背景事実間、認知状態間、および背景事実と認知状態の間の全てを対象とする

### Step 12: 導出グラフの構成

FPを根、共通抽象を葉とする導出グラフを構成する。

構成：
```
FP
└── アノマリー解消
    ├── 背景事実 → 共通抽象命題（葉）
    ├── 認知状態 → 共通抽象命題（葉）
    ├── 背景事実
    └── ...
```

導出の方向：FP → アノマリー解消 → 背景事実/認知状態 → 共通抽象命題
各ステップで情報が落ちる（具象→抽象）。

各葉ノード（共通抽象命題、またはT固有の背景事実/認知状態）に S 変数を付与する。
S 変数は Phase 2（戦術連鎖）で、その葉ノードに至る連鎖の末端の S 命題集合に代入される。

記録事項：
- FP → アノマリー解消への関係と理由
- 主動因命題の導出元集合
- 各背景事実/認知状態の前提と導出元部分集合
- アノマリー解消 → 各背景事実/認知状態への関係と理由
- 各背景事実/認知状態 → 共通抽象命題への関係と理由
- T固有のノードは葉として配置し、共通抽象がないことを明記する
- 各葉ノードの S 変数

## ゲーム設計上の制約

- アノマリー解消と鍵導出は、全ての背景事実が観測された後に同時に起きる設計とする
- 背景事実の部分集合でアノマリー解消をトリガーしない（実践的理由: 設計が単純で確実）

## 実行時の注意

### 一般知識と認知状態の推移の区別

一般知識（「同じ料理は同じ味がする」等）は推論の道具である。登場人物が実際にその推論を行ったことはT由来の具象的事実であり、モデル化の対象。「一般知識で完結」という判断を安易に行わないこと。

以下の問いを挟む：
- この推論を登場人物が実際に行ったか → 行ったならT由来の具象的事実
- この情報がプレイヤーにとって意味を持つための前提は何か → fcの根拠
- この粒度はゲーム体験として適切か → 細かすぎないか、飛びすぎていないか

### 認知的フレーミングの排除

命題に「記憶がある」「気づいた」等の認知的フレーミングを不要に付加しない。具象的な事実で記述する。ただし認知的イベントそのものが内容である場合（「味が違うと感じた」等）は認知的記述が必要。

### 語用論的含意への感度

Claude（実行者）は語用論的含意の検出に限界がある。有標な表現パターン（「と言われて」「何かを」等の修飾）に特に注意し、判例パターンを参照すること。

## 適用例：ウミガメのスープ

### Step 1

「これから飲むウミガメのスープは、かつて『ウミガメのスープだ』と言われて飲んだものと類似した味がするだろう」

注: 男はウミガメのスープを過去に食べたことはない。遭難中に「ウミガメのスープだ」と言われて人肉を食べた。

### Step 2

1. かつて「ウミガメのスープだ」と言われて何かを飲んだことがある（T由来）
2. これからウミガメのスープを飲む（S由来）

### Step 3

Sの行為: 自殺した
主動因命題: 「遭難中に死んだ仲間の肉で作ったスープを飲まされていたと悟った」

### Step 4

前提命題:
1. 過去に飲んだものはウミガメのスープではなかった（アノマリー解消の帰結）
2. そのスープは遭難中に飲んだ（T由来）
3. 遭難中に仲間が死んだ（T由来）
4. 食料が尽きていた（T由来）
5. 残った仲間がスープを作って飲ませた（T由来）
6. そのスープは死んだ仲間の肉で作られていた（T由来 — 鍵を含む）

### Step 5

鍵: 人肉（仲間の肉）
- 「仲間の肉」→ 自殺に至る衝撃
- 「犬の肉」→ 不快だが自殺に至らない
- 「普通の魚」→ 動因にならない

### Step 6

背景事実（鍵を除く）:
- 男は遭難した
- 遭難中に食料が尽きた
- 遭難中に仲間が死んだ
- 残った仲間がスープを作った
- 仲間はそれをウミガメのスープだと言った
- 男はそのスープを飲んだ

### Step 7

アノマリー解消の認知的過程:
```
以前に飲んだ時と味が違うと感じた（内的認知体験 — T由来）
→ 店員に確認し、今のは本物のウミガメのスープだと知った（外部行動 — S由来）
→ 過去に飲んだものはウミガメのスープではなかったと悟った（解消）
```

一般知識（道具）: 同じ料理は類似した味がする

認知的前提:
- 「味が違うと感じた」← 男がスープを飲んだ（背景事実） + S（今回のスープ）
- 「ウミガメのスープではなかった」← 味が違うと感じた + 今のは本物だと確認

「味が違うと感じた」は背景事実と同列のノードとして扱う。

### Step 8

- 「男は遭難した」: 含意なし
- 「食料が尽きた」: 含意なし
- 「仲間が死んだ」: 含意なし
- 「仲間がスープを作った」: 含意なし
- 「ウミガメのスープだと言った」: 単独では含意弱いが、「仲間が死んだ」「食料が尽きた」との組み合わせで含意が強まる可能性。要注意
- 「以前に飲んだ時と味が違うと感じた」: 含意なし

### Step 9

Sの緊張の特定:
- α: 男はウミガメのスープを一口飲んで「本当にウミガメのスープか」と確認した（期待との不一致 — 通常、注文した料理を疑わない）
- β: 男はその夜、自殺した（因果の欠落 — スープを飲んだことと自殺の因果が見えない）

緊張の分解:
- β → β1: 確認行為から自殺に至るメカニズム（因果の仕組み） + β2: それが自殺に至るほど深刻であること（程度の分離）
- 独立検証: β1のみ緩和（確認で不快な事実を知ったが深刻ではない）とβ2のみ緩和（深刻だが確認と無関係）は想像可能 → 分解妥当

競合モデル（緊張β「なぜ自殺したか」に対して）:
- 不治の病を宣告されていた
- 大切な人を失った悲しみ
- 借金で追い詰められていた
- 過去の食事の正体を悟った（→ T方向）
- スープに毒が入っていると思い込んだ

### Step 10

| 背景事実/認知状態 | 競合モデルの対応事実 | 共通抽象 |
|---|---|---|
| 遭難した | 不治の病 | 男は過去に深刻な苦境を経験した |
| 仲間が死んだ | 大切な人を失った | 男は人の死に関わる経験をした |
| 正体を悟った | 借金の全容を知った | 男は耐えがたい事実を知った |

### Step 11

- 食料が尽きた ← {遭難した}
- 仲間が死んだ ← {遭難した}
- 仲間がスープを作った ← {仲間が死んだ, 食料が尽きた}
- ウミガメのスープだと言った ← {仲間がスープを作った}
- スープを飲んだ ← {ウミガメのスープだと言った}
- 味が違うと感じた ← {スープを飲んだ} + S
- アノマリー解消 ← {味が違うと感じた} + S（店員の確認）

### Step 12

```
FP（仲間の肉を食べさせられていたと悟った）
└── アノマリー解消「過去に飲んだものはウミガメのスープではなかった」
    ├── 味が違うと感じた → 共通抽象（葉）
    ├── 遭難した → 共通抽象「男は過去に深刻な苦境を経験した」（葉）
    ├── 仲間が死んだ → 共通抽象「男は人の死に関わる経験をした」（葉）
    ├── 食料が尽きた
    ├── 仲間がスープを作った
    ├── ウミガメのスープだと言った
    └── スープを飲んだ
```

各葉ノードに S 変数を付与する:
- 共通抽象/T固有ノード ── {s_1}, {s_2}, ...

S 変数は Phase 2（戦術連鎖）で具体的な S 命題集合に代入される。
