# 戦術連鎖（v3）

## 概要

統合アルゴリズム v3 Phase 3（後半）の規則。S の原子命題から分割命題への戦術連鎖を構成し、その過程で**中間命題**を導入する。連鎖の reveals → fc 関係が質問の formation_conditions を定める。

**本フェーズの意義は、適切な中間命題の導入にある。** 戦術連鎖は「どこに命題が必要か」を発見し「何が何を導出するか」を構築するための分析手法であり、出力は戦術ラベルではなく命題群とその導出関係である。

v2 版（`戦術連鎖.md`）からの変更:
- ピース・トリガーへの連鎖を廃止。S → 分割命題への連鎖に置き換え
- 上位連鎖（S→上位分割命題）と下位連鎖（上位→下位分割命題）の 2 段構造
- reveals → fc の関係が連鎖間の依存を定義する

## 入力

- Phase 3 前半の出力（S の原子命題・導出命題、initial_confirmed）
- Phase 2 の出力（分割命題群、分割の階層構造）
- `structure/汎用戦術.md`

## 出力

- 戦術連鎖群（上位連鎖 + 下位連鎖）
- **Phase 3 で新たに導入された中間命題群**（`_phase3.generated_intermediates` に記録）
- 各連鎖の fc source（前段の reveals との依存関係）

## 連鎖の 2 段構造

### 上位連鎖: S → 上位分割命題

S の原子命題を起点とし、上位の分割命題（棄却力の大きいもの）に至る連鎖。

- 起点: S の原子命題群（initial_confirmed）
- 目標: 上位分割命題
- 上位分割命題は S の情報のみから到達可能でなければならない

### 下位連鎖: 上位分割命題 → 下位分割命題

上位分割命題の確認を起点とし、下位の分割命題に至る連鎖。

- 起点: 上位連鎖で reveal された分割命題 + S の原子命題
- 目標: 下位分割命題
- **reveals → fc 関係**: 前段の連鎖の reveals が次の連鎖の fc に含まれる

## 手順

### Step 1: 上位連鎖の構成

棄却力の大きい分割命題への到達パスを構成する。

各上位分割命題について:
1. 関連する S 原子命題を特定する
2. 汎用戦術を適用して分割命題に至るステップを構成する
3. 各ステップの出力を中間命題として導入する（量的最低基準を満たすこと）

### Step 2: 下位連鎖の構成

上位連鎖の reveals を活用して、下位分割命題への到達パスを構成する。

各下位分割命題について:
1. fc source を明示する（どの上位連鎖の reveals を使用するか）
2. 上位分割命題 + S 原子命題から、汎用戦術を適用して下位分割命題に至るステップを構成する
3. 各ステップの出力を中間命題として導入する（量的最低基準を満たすこと）
4. 連鎖間の依存関係が明確であること

### Step 3: 中間命題の形式化

Step 1-2 で導入した中間命題を整理する:

1. 全ての中間命題に ID を付与する
2. 各中間命題の formation_conditions を戦術連鎖のステップ構造から設定する
3. `_phase3.generated_intermediates` に記録する

### Step 4: 出力の検証

- 全ての分割命題に対して連鎖が構成されていること
- **全ての連鎖が最低 1 つの中間命題を含むこと**（量的最低基準）
- 全ての連鎖が命題 ID の連鎖で構成されていること（非形式テキストが残っていないこと）
- 各連鎖の起点が initial_confirmed または先行する連鎖の reveals に含まれること
- 連鎖間に循環依存がないこと
- Phase 3 で導入された中間命題が `_phase3.generated_intermediates` に記録されていること

## 中間命題の導入原則

### 本フェーズの核心

戦術連鎖の分析過程で、起点と終点の間のギャップを埋める中間命題を発見し、形式的な命題として導入する。

連鎖の各ステップの出力は**命題**でなければならない。非形式的なテキスト記述（「過去の体験の状況」等）で終わらせてはならない。テキスト記述が出てきた場合、それは形式化すべき命題の候補である。

### 中間命題が必要になるパターン

- **段階的絞り込み**: 起点と分割命題の間の距離が大きい場合、中間の抽象度の命題を導入する
- **具体化パターン**: 抽象版と具体版の命題ペアが必要な場合
- **因果の中間ステップ**: 直接的な接続が跳躍を含む場合
- **矛盾の解消**: 常識的仮説の排除の結果が非自明な場合

### 量的最低基準

各戦術連鎖は、**最低 1 つの中間命題を導入しなければならない**。

- **1 つは最低評価である**: 1 つの中間命題は「基準をかろうじて満たしている」状態であり、望ましい水準ではない。連鎖の複雑さに応じて 2 つ以上を目指すべきである
- **0 は不合格**: いかなる理由であれ、中間命題が 0 個の連鎖は不完全であり、再検討が必要である

### 導入後の適切性レビュー

中間命題を導入した後、以下の観点でレビューする:

- **T 忠実性**: T の情報から逸脱していないか
- **プレイヤー視点の自然さ**: その中間ステップはプレイヤーの思考の流れとして自然か
- **質問への展開可能性**: その中間命題から自然な質問を生成できるか

レビューで問題が見つかった場合は、中間命題を**削除するのではなく、より適切な中間命題に置き換える**。量を減らすのではなく質を上げる方向で対処する。

### 中間命題の制約

- **T 忠実性**: T の命題の抽象化・一般化、T から論理的に導出可能な中間ステップに限定
- **独立性と客観性**: 他の命題と同じ品質基準を満たすこと

### 中間命題の導入時検証（1次チェック）

中間命題を導入する際、以下の 2 点を検証する。Phase 8 の検証は 2 次チェック（安全網）であり、ここでの 1 次チェックが本来の品質保証である。

#### T 文脈での真偽の明確さ

- 中間命題の真偽が T の事実から一意に定まることを確認する
- 真偽が異なる類似表現（例:「飲んだことがある」vs「飲んだことがあると信じている」）を混同しないこと

#### 意味の確定性

- 命題単体、またはその直接の前提命題と合わせて読んだとき、意味が一意に確定することを確認する
- **直接の前提命題**: 連鎖において当該中間命題の直上に位置する命題群。formation_conditions とは異なる概念である
- **省略の可否**: 直接の前提命題が文脈を確立している場合、重複する情報の省略を許容する。確立されていない場合は完全な記載を要する

## 戦術ステップの形式

各ステップは以下を明示する:

- 入力命題集合（命題 ID）
- 適用する汎用戦術
- 出力命題（**命題 ID**。非形式テキストは不可）

```
{命題ID群} →[戦術]→ 命題ID
```

## プレイヤー視点の検証

全ての戦術連鎖に対して、以下の観点で検証する:

1. **入力からの連想は自然か**: プレイヤーがその入力情報を持っている時、その方向に考えることが自然か
2. **結論から逆算していないか**: T からの逆算ではなく、S の情報構造から自然に生まれる連鎖か
3. **戦術の適用が適切か**: why/how/what/who/when の区別が正しいか
4. **跳躍が大きすぎないか**: 中間命題の追加が必要ではないか

## 失敗パターンと対策

### 誤り: T からの逆算

- S の情報構造からではなく、T の答えから逆算して連鎖を構成した
- **対策**: 連鎖は S の原子命題から出発し、汎用戦術の自然な適用で進むこと

### 誤り: 中間命題を導入しない連鎖

- 既存命題間に戦術ラベルを貼るだけで、中間命題を導入しなかった
- 連鎖の中間結果を非形式テキストで記述し、正式な命題にしなかった
- **対策**: 連鎖の各ステップの出力は命題 ID であること。テキスト記述が出てきたら、形式化すべき命題の候補として扱う。**中間命題が 0 個の連鎖は不合格であり、再検討すること**

### 誤り: 段階的絞り込みの具体例省略

- 段階的絞り込みを戦術として使う際、中間命題の具体例を挙げずに一足飛びに書いた
- **対策**: 段階的絞り込みを使う場合、少なくとも 1 つの具体的な中間命題を正式に導入する

### 誤り: 戦術の不適切な適用

- 「なぜ」（因果の逆転）と「どうやって」（手段の探索）を混同した
- **対策**: 戦術の入出力定義を確認し、適切な戦術を選択する

### 誤り: fc source の曖昧さ

- 下位連鎖の起点がどの上位連鎖の reveals に由来するか不明確
- **対策**: 各下位連鎖の fc source を明示的に記録する

## 規則

- 全ての分割命題に戦術連鎖が定義されること
- 使用する戦術は汎用戦術リスト（`structure/汎用戦術.md`）に含まれること
- **各戦術ステップの出力は命題 ID であること（非形式テキストは不可）**
- **中間命題を導入しない連鎖出力は不完全であり、再検討すること**（量的最低基準: 各連鎖に最低 1 つ）
- 上位連鎖の起点は S 原子命題（initial_confirmed）であること
- 下位連鎖の fc source（前段の reveals との依存関係）を明示すること
- 連鎖間に循環依存がないこと
- 全ての連鎖についてプレイヤー視点の検証を実施すること
- 戦術連鎖は S の情報構造から自然に生まれるものであり、T からの逆算ではないこと
- **Phase 3 で導入した中間命題は `_phase3.generated_intermediates` に記録すること**
- **中間命題は T 忠実性・独立性・客観性を満たすこと**
- 段階的絞り込みを使う場合、少なくとも 1 つの具体的な中間命題を正式に導入すること
