# 棄却条件（v3）

## 概要

統合アルゴリズム v3 Phase 3 の規則。Phase 2 の分割命題に基づき、各競合モデルの棄却条件を整理する。

v2 版（`棄却条件.md`）からの変更:
- 入力が葉命題・パターンから分割命題に変更
- パターン内命題と棄却命題の重複分析を廃止（パターン概念の廃止に伴い不要）
- 棄却力の評価と棄却不可能モデルの扱いを追加
- 棄却の進行順序の整理を追加

## 入力

- Phase 2 の出力（分割命題群、分割の階層構造）

## 出力

- 各モデルの棄却条件（偽となる分割命題）
- 棄却力の評価（各分割命題の棄却対象数）
- 棄却不可能モデル群とその根拠
- 棄却の進行順序

## 手順

### Step 1: 各モデルの偽命題の特定

全競合モデルに対して、分割命題のうち偽となるものを特定する。

- 分割命題は T において全て真である（Phase 2 で保証済み）
- 各モデルについて、どの分割命題が偽であるかを判定する
- 判定は分割命題の意味とモデルの具象命題を照合して行う

### Step 2: 最小棄却条件の決定

各モデルについて、最も少ない命題の確認で棄却できる条件を決定する。

- 複数の分割命題が偽であるモデルの場合、最も効率的な棄却経路を特定する
- 単一の分割命題で棄却可能なモデルはその命題が最小棄却条件
- 複数の命題の組み合わせが必要なモデルは AND 条件として記録する

### Step 3: 棄却力の評価

各分割命題について、それが確認された場合に棄却されるモデル数を評価する。

- 棄却力の大きい命題は探索の早い段階で確認すべき候補
- 棄却力の順位付けは Phase 5（質問生成）の設計指針となる

### Step 4: 棄却不可能モデルの特定

分割命題の全てが真であるモデル（T と同じ真偽パターンを持つモデル）を特定する。

- これらのモデルは共通命題では T と区別できない
- パズルの探索としての重要度を評価する（通常、T 近傍のモデルであり重要度は低い）
- Phase 2 の「T の抽象的位置づけ」の記録と整合させる

### Step 5: 棄却の進行順序の整理

分割命題の確認順序に応じて、モデル集合がどう縮小するかを整理する。

- 各段階で: 確認される命題、棄却されるモデル、残存するモデルを記録する
- この進行順序は Phase 5（質問生成）の棄却進行表の基礎となる

## 規則

- 棄却条件は分割命題の真偽に基づくこと（分割命題が確認されれば、偽であるモデルは棄却される）
- 全競合モデルに対して棄却条件を定義すること
- 棄却不可能モデルの存在を明示的に記録し、その根拠（T との共通構造）を説明すること
- 棄却力の評価を行い、探索設計への指針とすること
