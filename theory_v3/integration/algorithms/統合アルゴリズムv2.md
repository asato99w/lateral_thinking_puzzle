# 統合アルゴリズム v2

## 概要

S の緊張構造分析を起点に、仮説的要素（H）の抽象的な葉命題・競合モデル・パターン・棄却条件を段階的に構築し、探索用と T 導出用の二種の戦術連鎖を経て質問を生成するアルゴリズム。

## 前版からの主要な変更

- **起点の変更**: T の逆算連鎖 → S の緊張構造分析。演繹的要素と仮説的要素（H）を分離する
- **逆算連鎖・構造設計の廃止**: T の逆算分解から始めてピース・トリガーを設計するアプローチを廃止。緊張構造 → H 要素 → 抽象の葉 → パターン → 棄却条件の流れに置き換える
- **葉命題の抽象性保証**: H の葉命題は最も抽象化された命題であること。T の具象をそのまま葉に置かない。複数の競合モデルが共有できる座標軸として設計する
- **命題の真偽検証の義務化**: 全命題の T に照らした真偽判定を最優先の要件とする。客観性や完全性はその後の問題
- **競合モデル・パターン・棄却条件の統合**: 旧工程3（仕掛けと分類）の認知的断層分析・対立仮説構成をコア生成に統合
- **棄却条件**: 全競合モデルに対する OR of AND の定義。戦術連鎖後の中間命題に基づく追加棄却条件
- **二種の戦術連鎖**: 探索用（S→H、H 間関係）と T 導出用（棄却、パターン→T）の区別
- **ゲームフェーズの設計**: 探索フェーズ（パターン到達まで）と T 導出フェーズの区別が質問設計を制約する
- **命題一覧の逐次管理**: 独立したフェーズではなく、各フェーズで命題導入の都度更新

## 維持する原則

- **二段階導出**: entailment_conditions（confirmed→confirmed 不動点）と formation_conditions（confirmed→derived 1回パス）
- **recall_conditions = reveals される命題の formation_conditions**: 質問側の独自の跳躍を排除
- **仮説連鎖の廃止**: 各 formation_condition は confirmed のみ参照。derived を参照しない
- **1 段階推論の原則**: 各導出条件は単一の推論ステップを表現する
- **命題の独立性・客観性**: 全命題は自己完結的で客観的に検証可能
- **戦術ステップの出力は命題**: 非形式テキストは不可
- **T 忠実性**: 生成する命題は T の情報から逸脱しない

## 入力

- S: 問題文
- T: 真相

## 出力

- data.json: パズルデータ

## 大域目標

**複数の質問選択順において合理的なパスで T に到達できること**

合理的なパスとは、各質問のオープンが想起（確認済み命題または導出命題による）により説明でき、唐突な飛躍がないパスを指す。

## ゲームフェーズ

アルゴリズムが設計するゲームの流れは 2 つのフェーズから成る。ゲーム上に明示的な境界は存在せず、提示される質問の性質の変化として現れる。

- **探索フェーズ**: パターンを構成する命題が全て観測されるまで。複数のモデルが並立しうる情報空間を広げる
- **T 導出フェーズ**: パターンから T に至る過程。競合モデルの棄却と T への収束を含む

## 命題一覧の管理

命題一覧は独立したフェーズではなく、各フェーズで命題が導入される都度更新する。

各命題について以下を管理する:
- ID、ラベル、出自（導入フェーズ）
- T に照らした真偽（Phase 2 以降で設定。以降のフェーズで命題導入時に随時判定）

**真偽判定の最優先性**: 命題を導入する際、T に照らした真偽の判定が最も基本的な要件である。特にラベルと実体が乖離する命題に注意する（例: 「ウミガメのスープを飲んだ」は男の認識であり、T の事実として偽である場合がある）。

## Phase 構成

### Phase 1: 緊張構造分析

規則: `rules/緊張構造分析.md`（新規）

S に含まれる緊張構造を特定し、T がそれをどのように緩和するかを分析する。演繹的要素と仮説的要素を分離する。

**入力**: S, T

**出力**:
- S の緊張構造の記述
- T の演繹連鎖（緊張を解消する因果の骨格）
- T の命題群
- 仮説的要素 H1〜Hn の特定

### Phase 2: 抽象の葉

規則: `rules/抽象の葉.md`（新規）

各仮説的要素に対して、抽象的な葉命題を設計する。

**入力**: Phase 1 の出力

**出力**:
- 各 H の葉命題群
- 各葉命題の T に照らした真偽

### Phase 3: パターンと代表モデル

規則: `rules/パターンと代表モデル.md`（新規）

競合モデルを体系的に列挙し、H 葉互換パターンを定義し、代表モデルを選定する。

**入力**: Phase 1, 2 の出力

**出力**:
- 全競合モデルの列挙
- 各モデルの H 葉互換パターン
- 代表モデルの選定と根拠
- パターン境界

### Phase 4: 棄却条件

規則: `rules/棄却条件.md`（新規）

全競合モデルに対して棄却条件（OR of AND）を定義する。

**入力**: Phase 2, 3 の出力

**出力**:
- 各モデルの棄却条件
- パターン内命題と棄却命題の重複分析

### Phase 5: S 細分化

S を最小単位の命題に分解する。

**入力**: S

**出力**:
- S の細分化命題群

### Phase 6: 戦術連鎖

規則: `rules/戦術連鎖.md`（既存を使用。探索用/T 導出用の区別をアルゴリズム側で制御）

戦術連鎖の構成を通じて中間命題を導入し、全命題の導出条件を構築する。探索用と T 導出用を区別して設計する。

**入力**: Phase 1〜5 の全出力、`structure/汎用戦術.md`

**出力**:
- 中間命題群（各命題の真偽判定を含む）
- 各命題の導出条件（formation_conditions、確定版）
- 戦術連鎖の記録

#### 探索用の戦術連鎖

パターン到達までの情報空間を構築する:

**(a) H 間・中間命題集合間の関係性構築のための連鎖**

**(b) 細分化 S から H の抽象へ至る連鎖**

**(c) 中小から具象へ至る縦の連鎖**（必要に応じて）

**(d) 連想的に繋がる横の概念の連鎖**（必要に応じて）

#### T 導出用の戦術連鎖

パターン到達後に T に至る経路を構築する:

**(e) 棄却用中間命題からの連鎖**

**(f) パターンから T への連鎖**

#### Phase 6 固有の規則

既存の `rules/戦術連鎖.md` の規則（中間命題導入原則、量的最低基準、プレイヤー視点検証、失敗パターン、保留判断等）に加え、以下を適用する:

- 全中間命題の T に照らした真偽を判定すること
- **T アンカリングの防止**: 探索用の連鎖は、T からの逆算ではなく S の情報構造から自然に生まれるものであること
- **S の entailment を reveals に置かない**: S から論理的に導出できるものは質問で新たに確認する対象ではない

### Phase 7: 棄却条件の更新

Phase 6 で導入された中間命題を含めて、棄却条件を追加・更新する。

**入力**: Phase 4, 6 の出力

**出力**:
- 更新された棄却条件

### Phase 8: 質問生成

規則: `rules/質問生成.md`（既存を使用。以下の規則を追加適用）

戦術連鎖の各ステップを質問に展開する。

**入力**: Phase 6, 7 の出力

**出力**:
- 質問群
- clear_conditions の確定版

#### Phase 8 固有の規則

既存の `rules/質問生成.md` の規則（recall=fc、reveals 厳密性、prerequisites、客観性等）に加え、以下を適用する:

- **回答の T 整合性**: 質問に対する回答が T の事実と矛盾しないこと
- **探索/T 導出の区別**: T 導出用連鎖由来の質問は探索フェーズでの提示を抑制する
- **パターン内命題と棄却命題の重複**: あるパターン内命題が別モデルの棄却条件に含まれる場合の取り扱いに注意する

### Phase 9: 検証

規則: `rules/検証.md`（既存を使用。以下の検証項目を追加）

大域目標の達成を検証する。

**入力**: Phase 8 の出力

#### Phase 9 で追加する検証項目

既存の `rules/検証.md` の検証項目に加え、以下を検証する:

- **命題の真偽一貫性**: 全命題の T に照らした真偽が正しいこと
- **葉の抽象性**: 葉命題が最も抽象化された命題であること
- **回答の T 整合性**: 全質問の回答が T の事実と矛盾しないこと
- **探索フェーズの質問制御**: 棄却質問が探索フェーズで不適切に提示されないこと

## フェーズ間の依存

```
Phase 1（緊張構造分析）
  → Phase 2（抽象の葉）
    → Phase 3（パターンと代表モデル）
      → Phase 4（棄却条件）
        → Phase 5（S 細分化）
          → Phase 6（戦術連鎖）← 探索用 + T 導出用
            → Phase 7（棄却条件の更新）
              → Phase 8（質問生成）
                → Phase 9（検証）
```
