# 統合アルゴリズム

## 概要

T からの逆算分解を起点に、3 構成要素（ピース・トリガー・戦術連鎖）を段階的に構築し、全記述素の導出条件を設定した上で質問を生成するアルゴリズム。工程1（コア生成）、工程2（調整）、工程3（仕掛けと分類）の 3 工程から成る。

## v2 からの主要な変更

- **仮説連鎖の廃止**: 多段の導出条件を意図的に設計する構造（仮説連鎖）を廃止。導出条件は各記述素に対して個別に設定する
- **導出条件の統一**: 全記述素に対して導出条件を設定する。各導出条件は 1 段階の推論を表現する
- **戦術連鎖の拡張**: 中間記述素の導入に加え、全記述素の導出条件の構築も担う
- **想起条件の統一**: 質問の想起条件 = reveals される記述素の導出条件（質問側の跳躍を排除）
- **構成要素の整理**: 4 構成要素（ピース・トリガー・仮説連鎖・戦術連鎖）から 3 構成要素（ピース・トリガー・戦術連鎖）へ

## 入力

- S: 問題文
- T: 真相

## 出力

- data.json: パズルデータ（記述素・ピース・トリガー・戦術連鎖・質問の全定義）

## 大域目標

**複数の質問選択順において合理的なパスで T に到達できること**

合理的なパスとは、各質問のオープンが想起（確認済み記述素または導出記述素による）により説明でき、唐突な飛躍がないパスを指す。

## 工程構成

```
工程1: コア生成（Phase 1〜5）
  → 構造的な到達性を優先し、パズルデータの骨格を構築する
工程2: 調整（Phase 6〜8）
  → プレイヤー視点の認知的自然さを検査し、骨格を調整する
工程3: 仕掛けと分類（Phase 9〜15）
  → 認知的断層を分析し、対立仮説と棄却メカニズムを構成し、質問を分類する
```

各工程は前の工程の出力を入力とする。後の工程は前の工程の出力に対して追記のみ行い、既存の構造を変更しない。

## S/T 分離原則

Phase 1〜2 は **T の構造のみ**を扱う。S が登場するのは Phase 3（戦術連鎖）以降である。

- Phase 1: T の因果構造を分解する
- Phase 2: T の記述素からピース群を構成し、トリガー族・依存関係・導出条件を定義する
- Phase 3: ここで初めて S の記述素集合を起点とした戦術連鎖を構成する

| 工程 | Phase | 規則 | ファイル |

## 工程1: コア生成

### Phase 1: 逆算連鎖

規則「逆算連鎖」を適用し、T から因果の逆算分解を構成する。S と T の記述素を統一的な ID（D-*）で抽出する。

### Phase 2: 構造設計

規則「構造設計」を適用し、ピース群を構成する。各ピースのトリガー族と依存関係を定義し、全記述素の導出条件（formation_conditions）を設定する。

### Phase 3: 戦術連鎖

規則「戦術連鎖」を適用し、S の記述素を抽出し、戦術連鎖の構成を通じて中間記述素を導入する。導入した中間記述素の導出条件を設定し、Phase 2 で暫定設定した導出条件を確定する。

### Phase 4: 質問生成

規則「質問生成」を適用し、戦術連鎖の各戦術ステップを質問に展開する。各質問の想起条件は reveals される記述素の導出条件から導出する。

### Phase 5: 検証

規則「検証」を適用し、大域目標の達成を検証する。FAIL の場合は原因フェーズに差し戻す。

### フェーズ間の依存

```
Phase 1（逆算連鎖）
  → Phase 2（構造設計・導出条件設定）
    → Phase 3（戦術連鎖・導出条件構築）← ここで S が登場
      → Phase 4（質問生成）
        → Phase 5（検証）
```

Phase 1〜2 が T 側の骨格を構成し、Phase 3 で S との接続を設計しつつ導出条件を確定し、Phase 4 で質問を生成し、Phase 5 で検証する。

## 工程2: 調整

v2 の工程2 と同様の構成。検査項目から仮説連鎖関連を除外し、導出条件の個別妥当性の検査に置き換える。エージェント分離の原則は v2 と同一。

## 工程3: 仕掛けと分類

v2 の工程3 と同様の構成。仮説連鎖・末端記述素への参照を除外する。認知的断層分析、対立仮説構成、質問分類の基本構造は v2 と同一。

## サンプルの構成

```
app/poc_v3/samples/{puzzle}/
└── {nn}_{YYYY-MM-DD}/
    ├── data.json
    ├── data_src.json
    └── history/
        ├── 01_逆算連鎖.md
        ├── 02_構造設計.md
        ├── ...
```

- `nn` は連番（同一パズルの反復）、`YYYY-MM-DD` は実施日
- data_src.json はアルゴリズムの各ステップに従って段階的に構築される中間データ
- data.json は data_src.json から `_` プレフィックスのメタ情報を除去したもの
- history/ には各ステップの実施記録を追記する
