# サンプル運用ルール

## ディレクトリ命名

```
samples/NNN_タイトル/
```

- NNN: 3 桁連番（001, 002, ...）
- タイトル: 問題の通称

## 反復ディレクトリ

```
samples/NNN_タイトル/
  01_YYYYMMDD_内容/
  02_YYYYMMDD_内容/    ← 改善・再適用のたびに連番で追加
  03_YYYYMMDD_内容/
```

- 反復のたびに新しい連番ディレクトリを作成する
- YYYYMMDD: 反復の作成開始日
- サブディレクトリ名は `NN_YYYYMMDD_内容` の形式（例: `02_20260226_統合アルゴリズム適用`）

## ファイル生成の原則

- ファイルはアルゴリズムの適用過程で順次生成される。固定のファイルリストは定めない
- 生成済みファイルは変更しない。修正が必要な場合は新しいファイルを追加する（オープンクローズド）
- 各ファイルの冒頭に使用した規則・アルゴリズム名を明記する

## 調律履歴

Phase B（データ調律）の変更記録は反復ディレクトリ内の `history/` に保存する。

```
samples/NNN_タイトル/
  NN_YYYYMMDD_内容/
    01_入力.md        <- Phase A のサンプルファイル群
    ...
    NN_形式化.md
    history/           <- Phase B の調律記録
      0001_YYYYMMDD.md
      0002_YYYYMMDD.md
```
