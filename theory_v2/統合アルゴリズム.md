# 統合アルゴリズム

## 概要

T からの逆算連鎖を起点に、4 構成要素（ピース・トリガー・仮説連鎖・戦略連鎖）を段階的に構築し、その骨格に基づいて質問を生成するアルゴリズム。

## 入力

- S: 問題文
- T: 真相

## 出力

- data.json: パズルデータ（事実・ピース・トリガー・仮説連鎖・戦略連鎖・質問の全定義）

## 大域目標

**複数の質問選択順において合理的なパスで T に到達できること**

合理的なパスとは、各質問のオープンが想起（事実または仮説による）により説明でき、唐突な飛躍がないパスを指す。

## 手順

### Phase 1: 逆算連鎖

T を「なぜ？」の連鎖で逆算し、因果の連鎖を構成する。

**入力**: T
**出力**: 逆算連鎖

各ステップが「なぜ？」に対する必然的な回答であること。

### Phase 2: ピース抽出

逆算連鎖からピースを抽出し、依存関係を特定する。

**入力**: 逆算連鎖
**出力**: ピースの一覧（ID、ラベル、依存先リスト）

- 依存先が空のものが独立ピース
- 他のピースの発見が前提のものが依存ピース

### Phase 3: トリガー定義

各ピースに対して、オープン条件（トリガー）を事実と仮説の族として定義する。

**入力**: ピース一覧
**出力**: 各ピースのトリガー（事実/仮説の族: OR of AND）

### Phase 4: 仮説連鎖の展開

トリガーに含まれる仮説について、形成条件（事実集合の族）を再帰的に展開する。S の事実以外は仮説として扱い、S の事実に到達するか展開深さに達した時点で末端とする。

**入力**: トリガー定義
**出力**: 仮説連鎖の木（各仮説の形成条件、深さ最大 5）

### Phase 5: 戦略連鎖の構成

仮説連鎖の末端要素に対して汎用戦略を適用し、S の事実へと辿る鎖を構成する。

**入力**: 仮説連鎖の末端要素、汎用戦略リスト
**出力**: 末端から S までの戦略連鎖

仮説連鎖の空間と S の空間は基本的に直接つながっていない。戦略がその間を橋渡しする。

### Phase 6: 質問生成

Phase 1〜5 で構築された骨格に基づき、質問の集合を生成する。

**入力**: 骨格（ピース・トリガー・仮説連鎖・戦略連鎖）、S の事実
**出力**: 質問一覧

各質問に以下を定義する:

- テキスト（質問文）
- 回答テキスト（Yes/No それぞれ）
- 回答効果（Yes で得られる事実、No で得られる事実）
- 想起条件（事実または仮説の族）
- メカニズムラベル（直接観測/リンク/アノマリー）

### Phase 7: 検証

大域目標の達成を検証する。

#### 7-1: 到達パス検証

複数の質問選択順で T に到達できることを確認する。行き詰まり（オープン質問が尽きる状態）がないこと。

#### 7-2: 想起の合理性検証

各経路において、質問のオープンが想起条件により合理的に説明できることを確認する。唐突な飛躍がないこと。

## フェーズ間の依存

```
Phase 1（逆算連鎖）
  → Phase 2（ピース抽出）
    → Phase 3（トリガー定義）
      → Phase 4（仮説連鎖の展開）
        → Phase 5（戦略連鎖の構成）
          → Phase 6（質問生成）
            → Phase 7（検証）
```

Phase 1〜5 がパズルの骨格を構成し、Phase 6 で質問を肉付けし、Phase 7 で検証する。

## サンプルの構成

```
samples_v2/{puzzle}/
├── data.json          # パズルデータ本体（直接編集）
└── history/           # 各ステップの実施記録（追記専用）
    ├── 01_逆算連鎖.md
    ├── 02_ピース抽出.md
    ├── ...
```

- data.json はアルゴリズムの各ステップに従って段階的に構築・編集される
- history/ には各ステップで何をしたかの概要を記録する
