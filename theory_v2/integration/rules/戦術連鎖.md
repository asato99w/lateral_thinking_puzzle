# 戦術連鎖

## 概要

統合アルゴリズム Phase 4 の規則。汎用戦術を順次適用して記述素間の到達経路を構成する。

このフェーズで初めて S が登場する。戦術連鎖は 2 種類ある:

1. **末端記述素への戦術連鎖**: S の記述素集合を起点として末端記述素に到達するパス
2. **ピース内部の戦術連鎖**: トリガーを起点としてピース内の記述素に到達するパス

## 入力

- 末端記述素群（Phase 3 の出力）
- 各ピースのトリガー族と構成記述素（Phase 2 の出力）
- 起点となる記述素集合（アルゴリズムが指定）
- 汎用戦術リスト（`structure/汎用戦術.md`）

## 出力

- 各末端記述素に対する戦術連鎖（記述素集合 + 汎用戦術の配列）
- 各ピースに対するピース内部の戦術連鎖（記述素集合 + 汎用戦術の配列）

## 戦術ステップの形式

各ステップは以下を明示する:

- 入力記述素集合
- 適用する汎用戦術
- 出力記述素集合

```
記述素集合 →[戦術]→ 記述素集合
```

## 手順

### Step 1: 末端記述素への戦術連鎖

末端記述素に対して、起点記述素集合からその記述素に至るまでの汎用戦術の列を定義する。

```
記述素集合 →[戦術1]→ 記述素集合 →[戦術2]→ ...→ 末端記述素
```

起点はピースの種類により異なる:
- **独立ピース**: S の記述素集合
- **依存ピース**: S の記述素集合 + 依存先ピースの記述素集合（推移的）

記述素自体が起点の記述素集合に含まれる場合は、直接同定（`structure/汎用戦術.md`）を適用する。

### Step 2: ピース内部の戦術連鎖

各ピースに対して、トリガーを起点としてピース内の記述素に到達する汎用戦術の列を定義する。

```
トリガー(+既知記述素) →[戦術1]→ 記述素 →[戦術2]→ 記述素 →...
```

- 前ステップの出力は次ステップの入力に利用可能（順序依存の連鎖）
- 全ての構成記述素がいずれかのステップの出力に含まれること

## 質問生成との紐付け

戦術連鎖の各戦術ステップは、Phase 5（質問生成）で 1 つ以上の具体的な質問に展開される（1:多の対応）。この対応関係を明確に保つために:

- 各戦術ステップは、その戦術が「何を入力として」「何を導くか」を明示する
- 戦術の入力記述素集合は、対応する質問の recall_conditions の基礎となる
- 戦術の出力記述素集合は、対応する質問の reveals の基礎となる

## 規則

- 全ての末端記述素に戦術連鎖が定義されること
- 全てのピースにピース内部の戦術連鎖が定義されること
- 使用する戦術は汎用戦術リストに含まれること
- 各戦術ステップの入力・戦術・出力を明示すること
- ピース内部の戦術連鎖で、全構成記述素がいずれかのステップの出力に含まれること
