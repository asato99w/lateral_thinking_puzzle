# 戦術連鎖

## 概要

統合アルゴリズム Phase 4 の規則。仮説連鎖の末端仮説に対して、ある記述素集合から汎用戦術を順次適用してその仮説に到達するパスを構成する。

このフェーズで初めて S が登場する。S の記述素集合を起点として、戦術の適用により末端仮説への到達経路を設計する。

## 入力

- 仮説連鎖の末端仮説群（Phase 3 の出力）
- 起点となる記述素集合（アルゴリズムが指定）
- 汎用戦術リスト（`structure/汎用戦術.md`）

## 出力

- 各末端仮説に対する戦術連鎖（記述素集合 + 汎用戦術の配列）

## 個別処理

入力: 単一の末端仮説

1. 起点となる記述素集合から仮説に至るまでの汎用戦術の列を定義する

```
記述素集合 →[戦術1]→ 記述素集合 →[戦術2]→ ...→ 記述素
```

各ステップの入出力には事実（Ps-*, Pt-*）も仮説（H-*）も含まれうる。記述素統一モデルでは両者の区別はデータ定義上のカテゴリではなく、ゲーム中の確認状態の違いに過ぎない。

仮説自体が起点の記述素集合に含まれる場合は、直接同定（`structure/汎用戦術.md`）を適用する。

## 手順

仮説連鎖の全末端仮説に対して、上記の個別処理を反復適用する。

## 質問生成との紐付け

戦術連鎖の各戦術ステップは、Phase 5（質問生成）で 1 つ以上の具体的な質問に展開される（1:多の対応）。この対応関係を明確に保つために:

- 各戦術ステップは、その戦術が「何を入力として」「何を導くか」を明示する
- 戦術の入力記述素集合は、対応する質問の recall_conditions の基礎となる
- 戦術の出力記述素集合は、対応する質問の reveals の基礎となる

## 規則

- 全ての末端仮説に戦術連鎖が定義されること
- 使用する戦術は汎用戦術リストに含まれること
- 各戦術ステップの入力と出力を明示すること
