# 統合アルゴリズム

## 概要

T からの逆算連鎖を起点に、4 構成要素（ピース・トリガー・仮説連鎖・戦術連鎖）を段階的に構築し、その骨格に基づいて質問を生成するアルゴリズム。

## 入力

- S: 問題文
- T: 真相

## 出力

- data.json: パズルデータ（事実・ピース・トリガー・仮説連鎖・戦術連鎖・質問の全定義）

## 大域目標

**複数の質問選択順において合理的なパスで T に到達できること**

合理的なパスとは、各質問のオープンが想起（事実または仮説による）により説明でき、唐突な飛躍がないパスを指す。

## 使用規則

| Phase | 規則 | ファイル |
|-------|------|----------|
| 1 | 逆算連鎖 | `rules/逆算連鎖.md` |
| 2 | ピース抽出 | `rules/ピース抽出.md` |
| 3 | トリガー定義 | `rules/トリガー定義.md` |
| 4 | 仮説連鎖 | `rules/仮説連鎖.md` |
| 5 | 戦術連鎖 | `rules/戦術連鎖.md` |
| 6 | 質問生成 | `rules/質問生成.md` |
| 7 | 検証 | `rules/検証.md` |

## 手順

### Phase 1: 逆算連鎖

規則「逆算連鎖」を適用し、T から因果の逆算連鎖を構成する。

### Phase 2: ピース抽出

規則「ピース抽出」を適用し、逆算連鎖からピースと依存関係を抽出する。

### Phase 3: トリガー定義

規則「トリガー定義」を適用し、各ピースの想起条件（トリガー）を設計する。

### Phase 4: 仮説連鎖

規則「仮説連鎖」を適用し、トリガー中の仮説を再帰的に展開して形成条件を確定する。

### Phase 5: 戦術連鎖

規則「戦術連鎖」を適用し、仮説連鎖の末端仮説に戦術連鎖を構成する。起点となる事実集合はピースの種類により異なる:

- **独立ピース**: S の事実集合
- **依存ピース**: S の事実集合 + 依存先ピースの事実集合（推移的）

### Phase 6: 質問生成

規則「質問生成」を適用し、骨格に基づいて質問群を生成する。

### Phase 7: 検証

規則「検証」を適用し、大域目標の達成を検証する。FAIL の場合は原因フェーズに差し戻す。

## フェーズ間の依存

```
Phase 1（逆算連鎖）
  → Phase 2（ピース抽出）
    → Phase 3（トリガー定義）
      → Phase 4（仮説連鎖の展開）
        → Phase 5（戦術連鎖の構成）
          → Phase 6（質問生成）
            → Phase 7（検証）
```

Phase 1〜5 がパズルの骨格を構成し、Phase 6 で質問を肉付けし、Phase 7 で検証する。

## サンプルの構成

```
app/poc_v2/samples/{puzzle}/
└── {nn}_{YYYY-MM-DD}/
    ├── data.json
    └── history/
        ├── 01_逆算連鎖.md
        ├── 02_ピース抽出.md
        ├── ...
```

- `nn` は連番（同一パズルの反復）、`YYYY-MM-DD` は実施日
- data.json はアルゴリズムの各ステップに従って段階的に構築される
- history/ には各ステップの実施記録を追記する
