# 統合アルゴリズム

## 概要

T からの逆算連鎖を起点に、4 構成要素（ピース・トリガー・仮説連鎖・戦術連鎖）を段階的に構築し、その骨格に基づいて質問を生成するアルゴリズム。

## 入力

- S: 問題文
- T: 真相

## 出力

- data.json: パズルデータ（記述素・ピース・トリガー・仮説連鎖・戦術連鎖・質問の全定義）

## 大域目標

**複数の質問選択順において合理的なパスで T に到達できること**

合理的なパスとは、各質問のオープンが想起（確認済み記述素または導出記述素による）により説明でき、唐突な飛躍がないパスを指す。

## S/T 分離原則

Phase 1〜3 は **T の構造のみ**を扱う。S が登場するのは Phase 4（戦術連鎖）以降である。

- Phase 1: T の因果構造を分解する（S は記述素の抽出元としてのみ扱う）
- Phase 2: T の記述素からピースを構成し、トリガー族と依存関係を定義する
- Phase 3: トリガー記述素の横展開とピースメンバーの導出条件を設定する
- Phase 4: ここで初めて S の記述素集合を起点とした戦術連鎖を構成する

## 使用規則

| Phase | 規則 | ファイル |
|-------|------|----------|
| 1 | 逆算連鎖 | `rules/逆算連鎖.md` |
| 2 | ピース構成 | `rules/ピース構成.md` |
| 3 | 仮説連鎖（導出条件設定） | `rules/仮説連鎖.md` |
| 4 | 戦術連鎖 | `rules/戦術連鎖.md` |
| 5 | 質問生成 | `rules/質問生成.md` |
| 6 | 検証 | `rules/検証.md` |

## 手順

### Phase 1: 逆算連鎖

規則「逆算連鎖」を適用し、T から因果の逆算連鎖を構成する。S と T の記述素を統一的な ID（D-*）で抽出する。

### Phase 2: ピース構成

規則「ピース構成」を適用し、キーピースの特定を起点にピース群を構成する。各ピースのトリガー族と依存関係を同時に定義する。必要に応じて新しい記述素を生成する。

### Phase 3: 仮説連鎖（導出条件設定）

規則「仮説連鎖（導出条件設定）」を適用し、トリガー記述素の横展開とピースメンバーの導出条件を設定する。

### Phase 4: 戦術連鎖

規則「戦術連鎖」を適用し、2 種類の戦術連鎖を構成する:

1. **末端記述素への戦術連鎖**: 起点記述素集合から末端記述素に至るパス。起点はピースの種類により異なる:
   - **独立ピース**: S の記述素集合
   - **依存ピース**: S の記述素集合 + 依存先ピースの記述素集合（推移的）
2. **ピース内部の戦術連鎖**: トリガーを起点としてピース内の記述素に至るパス

### Phase 5: 質問生成

規則「質問生成」を適用し、戦術連鎖の各戦術ステップを 1 つ以上の具体的な質問に展開する。各質問は対応する戦術ステップとの紐付けを維持する。

### Phase 6: 検証

規則「検証」を適用し、大域目標の達成を検証する。FAIL の場合は原因フェーズに差し戻す。

## フェーズ間の依存

```
Phase 1（逆算連鎖）
  → Phase 2（ピース構成）
    → Phase 3（仮説連鎖・導出条件設定）
      → Phase 4（戦術連鎖の構成）← ここで S が登場
        → Phase 5（質問生成）
          → Phase 6（検証）
```

Phase 1〜3 が T 側の骨格を構成し、Phase 4 で S との接続を設計し、Phase 5 で質問を生成し、Phase 6 で検証する。

## サンプルの構成

```
app/poc_v2/samples/{puzzle}/
└── {nn}_{YYYY-MM-DD}/
    ├── data.json
    ├── data_src.json
    └── history/
        ├── 01_逆算連鎖.md
        ├── 02_ピース構成.md
        ├── ...
```

- `nn` は連番（同一パズルの反復）、`YYYY-MM-DD` は実施日
- data_src.json はアルゴリズムの各ステップに従って段階的に構築される中間データ
- data.json は data_src.json から `_` プレフィックスのメタ情報を除去したもの
- history/ には各ステップの実施記録を追記する
