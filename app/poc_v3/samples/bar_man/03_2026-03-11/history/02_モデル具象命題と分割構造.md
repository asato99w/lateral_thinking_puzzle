# Phase 2: モデル具象命題と分割構造

## 各モデルの具象命題

### T: しゃっくり
1. 男はしゃっくりが止まらなかった
2. 男は水を飲めばしゃっくりが治ると考えた
3. 男はしゃっくりを止めるためにバーに水を求めに来た
4. バーテンダーは男のしゃっくりに気づいた
5. バーテンダーは驚かせてしゃっくりを止めようとした
6. バーテンダーは驚かせる手段として銃を向けた
7. 男は銃を向けられて驚いた
8. しゃっくりが止まった
9. 男は水が不要になった
10. 男はしゃっくりを止めてくれたことに感謝した

## モデル横断の共通命題

### 男の状態

- 「男は何らかの症状・状態に困っていた」→ {T, M1, M2, M3, M4, M7, M8}
- 「水はその症状への対処法だった」→ {T, M1, M2, M3, M7, M8}
- 「水は口実・暗号だった」→ {M4, M5, M6}

### バーテンダーの認識と行動

- 「バーテンダーは男の状態を（その場で）察知した」→ {T, M1, M2, M3, M4, M7, M8}
- 「バーテンダーと男の間に事前の取決めがあった」→ {M5, M6, M9, M10}
- 「バーテンダーは男を脅威と判断した」→ {M11, M13, M14}
- 「バーテンダーは男を助けるために銃を向けた」→ {T, M1, M2, M3, M4, M7, M8}

### 解消メカニズム

- 「銃を向けられた衝撃が男の問題を解消した」→ {T, M1, M2, M3, M4, M7, M8, M12}
- 「問題が解消したので水が不要になった」→ {T, M1, M2, M3, M7, M8}

## 観察: T の抽象的位置づけ

T は「しゃっくり」という具象モデルだが、パズルの解答としての本質は「困りごとがあり、バーテンダーが察知して驚かせて解消した」という抽象レベルにある。{T, M1, M3, M7} 内での分割（しゃっくりか喉詰まりか等）はパズルとしてほぼ無意味。

このことは、モデル集合の分割において意味のある境界がどこにあるかを示している。分割の階層において、ある段階から先は「T の具象を当てるクイズ」になり、ウミガメのスープとしての探索の本質から外れる。
