# Phase 3: 導出構造と最終命題

## Step 1: T から分割命題への演繹構造

### 直接演繹
- T → P1: 「困っていた」→ 困りごと ✓
- T → P2: 「止めてやろうと」→ 助ける意図 ✓
- T → P3: 「立ち寄った」「気づき」→ 事前関係なし ✓
- T → P4: 驚きでしゃっくりが止まった → 驚きで異常解消 ✓
- T → P5: しゃっくり → 不随意の身体的反応 ✓
- T → P6: 「水を飲めば治ると思い」→ 水を飲みたい意図 ✓

### 命題間の演繹関係
- P5 → P1: 「不随意の身体的反応の困りごと」→「何らかの困りごと」（困りごとの具体的内容を落とす）
  - モデル集合: P5 true {G1} ⊆ P1 true {G1,G2,G3,M12} ✓
- P4 → P1: 「異常な状態が解消された」→「異常な状態=困りごとがあった」（解消の事実から存在を導く）
  - モデル集合: P4 true {G1,G2,G3} ⊆ P1 true {G1,G2,G3,M12} ✓

### 冗長辺の除去
- T → P1 は T → P5 → P1 および T → P4 → P1 に包含される → 除去

### 結果
T 直接ノード: {P2, P3, P4, P5, P6}
非直接ノード: {P1}（P5 と P4 から演繹）

## Step 2-3: 共通具象元の発見と検証

### 候補 1: P5 と P6 の共通具象元

T の事実: 「男はしゃっくりが止まらなくて困っていた。水を飲めば治ると思いバーに立ち寄った」

この事実は P5（不随意の身体反応）と P6（水を飲みたい意図）の両方の情報を含む。

**IP1: 男は不随意の身体的反応に困り、水で治せると考えてバーに来た**
- IP1 → P5: 水の意図を落とす → 「不随意の身体的反応の困りごと」
- IP1 → P6: 困りごとの内容を落とす → 「水を飲みたいという意図」
- 演繹的分岐: P5 と P6 を同時に演繹 ✓
- true_models: {G1}（G1 のみが身体反応 + 水の意図を持つ）
- false_models: {G2,G3,G4,M10,M11,M12,M13,M14}
- モデル非自明分割: {G1} vs {rest} — 非自明 ✓（P5 と同一分割だが分岐があるので許容）
- T 真偽: 真 ✓
- 意味確定: 明確 ✓

### 候補 2: P2 と P4 の共通具象元

T の事実: 「バーテンダーはしゃっくりに気づき、驚かせて止めてやろうと銃を向けた。しゃっくりが止まった」

この事実は P2（助ける意図）と P4（驚きで解消）の両方の情報を含む。

**IP2: バーテンダーは男の異常を察知し、驚かせて治そうとした結果、男の異常が即座に解消された**
- IP2 → P2: 結果を落とす → 「助ける意図で銃を向けた」
- IP2 → P4: 意図・察知を落とす → 「驚きで異常が即座に解消された」
- 演繹的分岐: P2 と P4 を同時に演繹 ✓
- true_models: {G1,G2,G3}（察知 + 治療意図 + 解消の全てを満たすモデル。M11 は察知ではなく事前知識、M12 は解消が即座の驚きではない）
- false_models: {G4,M10,M11,M12,M13,M14}
- モデル非自明分割: {G1,G2,G3} vs {rest} — 非自明 ✓（P4 と同一分割だが分岐があるので許容）
- T 真偽: 真 ✓
- 意味確定: 明確 ✓

## Step 4: 再帰的適用

IP1 と IP2 を追加した後の構造:

```
T → IP1 → P5 → P1
T → IP1 → P6
T → IP2 → P2
T → IP2 → P4 → P1
T → P3
```

T 直接ノード（更新後）: {IP1, IP2, P3}

IP1 と IP2 の間、または IP1/IP2 と P3 の間に、さらなる共通具象元を探索:
- IP1 (男の状態・意図) と IP2 (バーテンダーの察知・行動・結果) は異なるアクター・異なる因果要素を記述。共通の具象元は T 自体に近づくため、有意な中間は見出せない。
- P3 (事前関係なし) は独立した否定的事実で、IP1/IP2 とは直交する。

→ 新たな共通具象元なし。再帰終了。

## Step 5: 抽象化による発見

T 直接ノード: {IP1, IP2, P3}

これらの間に抽象化を通じた共通構造がないか検討:

1. IP1「男の身体反応 + 水の意図」と IP2「バーテンダーの察知 + 治療 + 解消」
   - 抽象: IP1 → 「男には解消すべき問題があった」、IP2 → 「他者が問題を解消した」
   - 共通抽象: 「問題の発生と他者による解消」
   - 具象元候補: 「男の身体反応をバーテンダーが察知して驚かせて治した」
   - これは FP そのものに近い。FP で処理するため、ここでは中間命題にしない。

2. IP1 と P3:
   - 抽象: 「男が問題を持ってバーに来た」+ 「初対面の出会い」→ 「偶然の来店」
   - これは IP1 と P3 の和であり、意味のある共通具象元ではない。

3. IP2 と P3:
   - 抽象: 「察知して治した」+「事前関係なし」→ 「初対面で即座に察知して助けた」
   - 具象元候補: 「初対面のバーテンダーが男の異常を即座に察知した」
   - true_models: {G1,G2,G3}（偶発的出会い + 察知 = G1,G2,G3。M12 は察知だが P4 の即座の効果がない）
   - これは IP2 ∩ P3 の model set = {G1,G2,G3} ∩ {G1,G2,G3,M12,M13,M14} = {G1,G2,G3}。IP2 と同一。
   - IP2 と同一分割であり、かつ P3 への分岐を生まない（「事前関係なし」は具象元に含まれるが、IP2 とは別の情報を落とすことで P3 は得られない — 「初対面の」を落とすと「バーテンダーが察知して治した」= IP2 にしかならない）。
   - → 有意な中間命題にならない。

→ 新たな命題なし。

## Step 6: 最終命題 (FP)

T 直接ノード: {IP1, IP2, P3}

FP: **「男は不随意の身体的反応を止めたくて水を求めてバーに入ったが、初対面のバーテンダーがそれに気づき、驚かせることで反応を止めた」**

FP → IP1: バーテンダーの情報を落とす → 「男は不随意の身体的反応に困り、水で治せると考えてバーに来た」 ✓
FP → IP2: 男の初期意図を落とす → 「バーテンダーが異常を察知し驚かせて治した結果、解消された」 ✓
FP → P3: 「初対面の」→ 「事前の関係なし」 ✓

FP true_models: {G1}（G1 のみが全要素を満たす）

設計基準チェック:
- パズルの核心: しゃっくりを止めようとした男がバーテンダーに驚かされて治った ✓
- 全 T 直接ノードを演繹: IP1, IP2, P3 全て演繹可能 ✓
- 独立性・客観性: 自己完結的・客観的な事実 ✓
- 1 つの命題: 単一文 ✓

## Step 10: 棄却不可能モデルの確認

最も具象的なノード FP の true_models = {G1} = {T, M1, M2, M3}

G1 メンバー内の差異:
- T: しゃっくり
- M1: 過呼吸
- M2: 喉の痙攣
- M3: 嘔吐感

差異は「不随意の身体的反応」の具体的種類のみ。緩和メカニズム（驚愕による反射中断）は同一。パズルの探索としてこの具象レベルの差異を問う必要はなく、棄却不可能モデル群として適切。

unrejectable_models: G1 = {T, M1, M2, M3}
根拠: 全分割命題を共有し、具象的差異（身体反応の種類）のみ。
