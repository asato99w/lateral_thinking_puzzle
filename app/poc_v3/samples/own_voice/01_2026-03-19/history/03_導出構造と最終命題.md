# Phase 3: 導出構造と最終命題

## Step 1: T → 分割命題の演繹構造

T → 各分割命題:
- T → P1: ストーカーは外部の人物 → 恐怖の原因は外部
- T → P2: ストーカーは意図的に女を狙っている → 意図的に害を与える第三者が存在
- T → P3: 電話を通じて声が聞こえた → 通信デバイスを介して聞こえた
- T → P4: ストーカーがベッドの下にいた → 同じ部屋にいた
- T → P5: 電話は技術的手段 → 技術的手段を通じて聞こえた

命題間の演繹関係:
- P2 → P1: 「意図的に害を与える第三者が存在」→「恐怖の原因は外部」（第三者は外部の存在。「意図的に害を〜」を落とす）
- P4 → P1: 「脅威の存在が同じ部屋にいた」→「恐怖の原因は外部」（同室の存在は外部的。「同じ部屋」を落とす）
- P3 → P5: 「通信デバイスを介して」→「技術的手段を通じて」（通信デバイスは技術的手段の下位概念。「通信」を落とす）

冗長辺の除去:
- T → P1: T → P2 → P1 および T → P4 → P1 に包含 → 除去
- T → P5: T → P3 → P5 に包含 → 除去

整理後: T → P2, T → P3, T → P4

## Step 2: 共通具象元の発見

### P2 と P5 の共通具象元

T の事実: 「ストーカーが電話（技術的手段）を使って女のベッドの下から電話をかけた」

この事実は P2（悪意ある第三者）と P5（技術的手段）の両方に跨る。

候補: P6 = 「悪意ある第三者が技術的手段を用いて女の声を利用した」
- P6 → P2: 「技術的手段を用いて女の声を利用した」を落とす → 「悪意ある第三者が存在する」✓
- P6 → P5: 「悪意ある第三者が」を落とす → 「技術的手段を通じて女の声が聞こえた」✓

## Step 5: 抽象化による発見

T 直接ノード: P7, P4

P7: 「悪意ある第三者が通信デバイスを通じて女の声を利用した」{T, M6}
P4: 「脅威の原因となる存在が女と同じ部屋にいた」{T, M5}

抽象化の試み:
- P7 → 「外部の脅威が通信に関連」
- P4 → 「外部の脅威が物理的に近接」
- 共通抽象: 「脅威の顕在化」→ これは P1（恐怖の原因は外部）と同等。既に存在。

P7 と P4 は本質的に独立した軸（通信 vs 物理的近接）であり、その結合が T を一意に特定する。新たな共通具象元は見つからない。

## Step 6: 最終命題（FP）の定義

T 直接ノード: P7, P4

FP: 「女を狙うストーカーが同じ部屋に潜んでおり、女が電話に出て声を発した時、電話越しにリアルタイムで自分の声が反射して聞こえた」

FP → P7: ストーカー（悪意ある第三者）+ 電話（通信デバイス）+ 声の利用 → 「同じ部屋」の情報を落とす → P7 ✓
FP → P4: ストーカーが同じ部屋に潜んでいた → ストーカーの特性を落とす → 「脅威の存在が同じ部屋にいた」→ P4 ✓

FP true_models: {T}（最終命題に非自明分割は不要）

## Step 7: FP のクリア適切性検証

FP が確認された場合、演繹で得られる命題群: P7, P6, P3, P2, P1, P5, P4 + initial_confirmed

- α1（声が通常でない経路で聞こえた）: FP に「電話越しにリアルタイムで反射」→ 通信デバイスを介した非通常経路 ✓
- α2（自分の声であることが恐怖の鍵）: FP に「自分の声が反射して聞こえた」→ 自分の声が聞こえたことで電話の相手が至近距離にいると判明 ✓
- β（声→恐怖の因果）: FP に「ストーカーが同じ部屋に潜んでいた」→ 差し迫った身体的脅威 ✓

全緊張解消。clear_conditions = [["FP"]]
