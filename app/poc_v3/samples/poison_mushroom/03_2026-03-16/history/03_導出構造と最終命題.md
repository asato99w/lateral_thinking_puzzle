# Phase 3: 導出構造と最終命題

## Step 1: T から分割命題への演繹構造

T → P1(生存): 遭難中で助かる見込みの安堵 → 生存に関わる安堵
T → P2(キノコ活動): キノコ狩り中 → キノコに関連する活動
T → P3(不在理由): 食用不在→人が採った → 不在理由を推論
T → P4(遭難): 山で遭難 → 遭難
T → P5(物質無関係): 推論が安堵の理由 → 物質的性質由来ではない

命題間の演繹:
- P4 → P1: 遭難 → 安堵は生存に関わる ✓ (遭難すれば生存が脅かされる)
- 他の命題間に演繹関係なし

冗長辺の除去: T → P1 は T → P4 → P1 に包含 → 除去

## Step 2-3: 共通具象元と中間命題

### P6: "男は遭難しており、安堵の理由は食用キノコがないという事実から導いた推論だった"

T の事実: 遭難中 + 食用不在からの推論が安堵の根拠。この事実は P4 と P5 に跨る。
- → P4: "遭難" の情報を残し推論部分を落とす → 遭難していた ✓
- → P5: "食用不在からの推論が安堵理由" → 物質的性質由来ではない ✓

二重要件:
- 演繹的分岐: P4, P5 → 2分岐 ✓
- モデル非自明分割: true_models = {T, M14} (T は遭難+食用不在推論、M14 は遭難+食用不在で動かない) → 非自明 ✓

導入時検証:
- T 文脈: 遭難中 + 食用不在からの推論 → 真 ✓
- M14 文脈: 遭難中 + 食用不在から「動かずに待てる」推論 → 真 ✓
- 意味確定性: 明確 ✓

### P7: "男はキノコに関連する活動中に、食用キノコがない理由について推論した"

T の事実: キノコ狩り中に食用不在の理由(人が採った)を推論。P2 と P3 に跨る。
- → P2: "食用不在の理由の推論" を落とす → キノコ活動のためにいた ✓
- → P3: "キノコ活動" を落とす → 食用不在理由を推論した ✓

二重要件:
- 演繹的分岐: P2, P3 → 2分岐 ✓
- モデル非自明分割: true_models = {T, M4} → 非自明 ✓

導入時検証:
- T 文脈: キノコ狩り + 食用不在理由推論 → 真 ✓
- M4 文脈: 栽培偵察(キノコ活動) + 栽培失敗(不在理由)推論 → 真 ✓
- 意味確定性: 明確 ✓

## Step 4: 再帰的適用

更新後のグラフ:
```
T → P7 → P2
         → P3
T → P6 → P4 → P1
         → P5
```

T 直接ノード: P7, P6

P7 と P6 の共通具象元を探索:
- "キノコ狩り中に遭難し、食用不在の理由を推論して安堵" → true_models = {T} only → FAIL
- 抽象化しても {T} 以外にならない → 共通具象元なし

## Step 5: 抽象化による発見

T 直接ノード: P7, P6

P7 を抽象化: "何らかの活動中に、周囲の状況から推論した"
P6 を抽象化: "困難な状況で、不在の事実から推論して安堵した"

共通構造: "状況の読み取り（推論）が安堵に至った" — これは P7, P6 のいずれも演繹しない（情報不足）→ 新たな中間命題は見つからない

## Step 6: FP の定義

T 直接ノード: P7, P6

FP: "男はキノコ狩り中に山で遭難しており、食用キノコが残っていないのは人が採ったからだと推論し、人が近くにいると悟って安堵した"

FP → P7: "キノコ狩り中" + "食用不在理由を推論" → ✓
FP → P6: "遭難" + "食用不在からの推論が安堵の理由" → ✓

FP true_models: {T} (T 固有の推論)

設計基準:
- パズルの核心: "食用不在 → 人が採った → 人がいる → 助かる" ✓
- 全 T 直接ノードを演繹: P7, P6 → ✓
- 独立性・客観性: ✓
- 1命題に統合: ✓

## Step 7: クリア適切性検証

FP 確認時に S の全緊張が解消されるか:

- α1 (毒キノコの存在がポジティブ): FP から「食用は人が採り、毒キノコが残っている」→ 毒キノコの存在は選択的採集の残余の証拠 → ✓ 解消
- α2 (食用不在がポジティブ): FP から「食用不在 → 人が採った → 人がいる」→ 食用不在が人の存在の間接証拠 → ✓ 解消
- β (男の困難): FP から「キノコ狩り中に遭難」→ 困難が判明 → ✓ 解消

全緊張解消 → clear_conditions = [["FP"]]

## Step 9: fc の仮置き

導出グラフの逆方向（アブダクション）:
- P4.fc = [[P1]] (P1 が知られていれば P4 を想起)
- P7.fc = [[P2, P3]] (P2+P3 → P7 を想起)
- P6.fc = [[P4, P5]] (P4+P5 → P6 を想起)
- FP.fc = [[P7, P6]] (P7+P6 → FP を想起)

根命題 (P1, P2, P3, P5) の fc は Phase 4 で S 命題から設定する。

## Step 10: 棄却不可能モデルの確認

全分割命題 P1-P7 の真偽パターンで T が一意に特定される。
P1∧P2 でも T が一意に特定可能（他に両方真のモデルがない）。

棄却不可能モデル: なし（全競合モデルが少なくとも 1 つの分割命題で T と異なる）
