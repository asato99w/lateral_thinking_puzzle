# Phase 2: 構造設計

## 設計方針: シングルピース・複数パス

### 01 イテレーションの問題点

01 では P-DOOR（支持）→ P-CAPTIVE（キー）の2ピース構成としたが、以下の問題が発生した:

1. **情報隔離の意味的破綻**: P-DOOR の D-7（外の世界を知らなかった）が監禁とほぼ等価であり、支持ピースの探索3問目で核心に近接
2. **連鎖の降順構造**: P-DOOR 内部が D-7（核心に近い）→ D-2（周辺）と、核心側から周辺へ向かう非自然な順序
3. **概念空間の狭さ**: 「外を知らない」と「監禁」の間に意味的緩衝地帯がなく、2ピース分割の前提が成立しない

### シングルピースの利点

- D-7 を entailment に回すことで早期開示を回避可能
- 連鎖順序を自由に設計でき、観察→推論→核心の昇順を実現
- 複数パスにより、プレイヤーに探索の選択肢を提供

## Step 1: 核心の特定

核心情報 = 女性が監禁されていたという事実（D-1）

核心に至る導出関係の整理:

```
D-1（監禁されていた）
 ↑ Path A: 物理的観察ルート
 │  D-2（ドアは外に通じていた）← D-3（外の世界が見えた）← D-8（知らない世界）
 │  「外に出るドアを開けるなと言われていた → 閉じ込められていた」
 │
 ↑ Path B: 心理的観察ルート
 │  D-9（状況の異常さを悟った）← D-7（外の世界を知らなかった）← D-8（知らない世界）
 │  「異常な状況に置かれていた → 監禁されていた」
 │
 ↓ 核心からの帰結
    D-6（監禁を知らなかった）→ D-4（気づいた）→ D-5（通報した）
```

## Step 2: シングルピース P-TRUTH の設計

### P-TRUTH: 監禁の真相（キーピース、唯一のピース）

members:
- D-1: 女性は地下室に監禁されていた（核心）
- D-2: 「開けてはいけないドア」とは監禁部屋から外に出るドアのことだった
- D-3: ドアを開けたら普通の外の世界が見えた
- D-4: 女性は自分が監禁されていたことに気づいた（entailment）
- D-5: 女性は自分の監禁を通報するために警察に行った（entailment）
- D-6: 女性は自分が監禁されていることを知らなかった
- D-7: 女性は地下室の外の世界を知らなかった
- D-8: ドアの向こうには女性が知らない世界があった（Phase 2 生成）
- D-9: 女性は自分の置かれた状況が異常だと悟った（Phase 2 生成）

## Step 3: トリガーの設計

trigger: [[D-8]]

D-8（ドアの向こうには女性が知らない世界があった）は S の情報から導出可能であり、ピースへの入口として機能する。D-8 確認後、Path A と Path B の両方が開放される。

## Step 4: 複数パスの設計

### Path A: 物理的観察ルート
D-8 → D-3（外の世界が見えた）→ D-2（ドアは外に通じていた）→ **D-1**

ドアの物理的な性質を追跡し、「外に出るドアを開けるなと言われていた」という事実から監禁を導く。

### Path B: 心理的観察ルート
D-8 → D-7（外の世界を知らなかった）→ D-9（状況の異常さを悟った）→ **D-1**

女性の認知状態を追跡し、「異常な状況に置かれていた」という事実から監禁を導く。

### 収束
D-1 確認後: D-1 → D-6 → entailment（D-3 + D-6 → D-4 → D-5）

注: D-4 の entailment には D-3 が必要。Path B 経由で D-1 に到達した場合、D-3 は別途確認が必要（D-8 確認済みなので Q-3 は利用可能）。
