# Phase 2+3: ピース構成（ピース抽出 + トリガー定義）

## ピース構成

### P-past（過去の遭難）— 独立ピース
- 構成事実: Pt-1（遭難）, Pt-2（食料枯渇）, Pt-3（仲間の死）
- depends_on: []
- トリガー: 「男の過去に重大な体験がある」

### P-deception（偽装）— 独立ピース
- 構成事実: Pt-4（スープを飲ませた）, Pt-5（ウミガメと偽った）
- depends_on: []
- トリガー: 「男は何かを騙された経験がある」

### P-realization（気づきと絶望）— 依存ピース
- 構成事実: Pt-6（正体は仲間の肉）, Pt-7（味が違った）, Pt-8（正体を悟った）, Pt-9（絶望して自殺）
- depends_on: [P-past, P-deception]
- トリガー: 「危機的状況で、ウミガメのスープと偽って何かを飲まされた」

## 依存構造

```
P-past（独立）    P-deception（独立）
       \                /
        → P-realization ←
```

## 設計メモ

- Pt-6（正体は仲間の肉）は偽装行為ではなく発覚・気づきの文脈 → P-realization に配置
- トリガーは T 側の構造から定義。S との接続は Phase 5（戦術連鎖）で扱う
