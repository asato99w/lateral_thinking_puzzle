# Phase 6: 質問生成

## エンジン互換性の課題と解決

### 問題
末端仮説（`formation_conditions = []`）はエンジンの `h.id in state.observed_facts` で導出される。
つまり仮説IDと一致する事実が `observed_facts` に存在する必要がある。
しかし facts 配列には Ps-*/Pt-* のみで H-* が存在せず、末端仮説は導出不能だった。

### 解決: 仮説事実（H-* facts）の導入
- 末端仮説に対応する事実を facts 配列に追加（例: `{"id": "H-sailor", "label": "男は船乗りだった"}`）
- 質問の reveals で H-* 事実を公開 → `observed_facts` に追加 → 仮説導出が起動
- H-* はピースの facts にも含め、スコープチェックを通過させる

### eval の修正
`check_integrity.py` の `check_hypothesis_refs` と `check_question_refs` を修正。
H-* が facts と hypotheses の両方に存在する場合を許可（事実のみの参照はエラーのまま）。

## ピース構成の変更

仮説事実をピースに含める必要があるため、ピース構成を更新:

| ピース | 元の facts | 追加した H-* facts | 理由 |
|--------|-----------|-------------------|------|
| P-past | Pt-1, Pt-2, Pt-3 | H-sailor, H-shipwreck, H-lost-someone, H-food-shortage | P-past の質問の recall に必要 |
| P-deception | Pt-4, Pt-5 | H-fed-soup, H-labeled-turtle | P-deception の質問の recall に必要 |
| P-realization | Pt-6, Pt-7, Pt-8, Pt-9 | なし | recall は H-forced-soup のみ（trigger、非事実） |

P-realization の H-forced-soup 導出可能性:
- P-realization スコープ = S + 自身 + P-past + P-deception
- H-food-shortage（P-past経由）, H-fed-soup, H-labeled-turtle（P-deception経由）が全て導出可能
- → H-forced-soup の条件 [H-food-shortage, H-fed-soup, H-labeled-turtle] が成立

## Q-8 の複数 recall 経路

Q-8 の recall_conditions は `[[H-lost-someone], [H-food-shortage]]`（OR）。
