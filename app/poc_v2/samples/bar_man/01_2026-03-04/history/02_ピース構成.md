# Phase 2: ピース構成

## Step 1: キーピースの特定

T の核心: バーテンダーが銃を向けたのはしゃっくりを驚かせて止めるためだった。これがパズルの転換点。

**P-resolution**（キーピース）: バーテンダーの真意と結末
- members: [D-9, D-10, D-11, D-12, D-13]
- clear_conditions: [["D-10", "D-13"]]
  - D-10: 銃を向けたのは驚かせる手段だった（銃の目的）
  - D-13: 「ありがとう」はしゃっくりを止めてくれたことへの感謝（感謝の意味）

## Step 2: 支持ピースの設計

**P-hiccup**（支持ピース）: 男のしゃっくり
- members: [D-6, D-7, D-8]
- D-6: 男はしゃっくりが止まらなくて困っていた
- D-7: 水を飲めばしゃっくりが治ると思った
- D-8: バーテンダーは男のしゃっくりに気づいた

情報隔離の確認: P-hiccup の members はしゃっくりの存在と水の動機のみ。銃の目的（核心情報）は含まない。

## Step 3: 新規記述素の生成とトリガー族の設計

### Phase 2 生成記述素

| ID | label | 目的 |
|----|-------|------|
| D-14 | 男が水を求めたのは喉の渇きのためではない | P-hiccup のフック（水の動機への疑問） |
| D-15 | 男の様子に通常と異なる点があった | P-hiccup のフック（男の異変への注目） |

### トリガー族

**P-hiccup**: triggers = [["D-14"], ["D-15"]]
- D-14: 水の動機が喉の渇きでないと分かれば → なぜ水を？ → しゃっくり
- D-15: 男に異変があったと分かれば → 何が？ → しゃっくり
- 全メンバー接続: D-14 → D-7（水の理由）→ D-6（しゃっくり）→ D-8（気づき）

**P-resolution**: triggers = [["D-8"]]
- D-8: バーテンダーがしゃっくりに気づいていた → 何をした？ → D-9, D-10
- 全メンバー接続: D-8 → D-9（止めようとした）→ D-10（銃が手段）→ D-11（止まった）→ D-12（水不要）, D-13（感謝）

## Step 4: 依存関係の導出

- P-resolution: トリガーに D-8（P-hiccup の member）→ **依存ピース**, depends_on: ["P-hiccup"]
