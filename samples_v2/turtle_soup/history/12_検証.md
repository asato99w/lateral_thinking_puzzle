# 12 検証

## 実施内容

複数の質問選択順で T に到達できることを確認した。

### パス A: 排除先行

```
q_h01 → q_h02 → q_h03 → q_e01(Pt-7) → q_e02(Pt-8, ピースe)
→ q_a01(Pt-1) → q_a02(Pt-2) → q_a03(Pt-3, ピースa) → q_c01(Pt-4)
→ q_c02(Pt-5, ピースc,d) → q_c03(Pt-9, ピースf) → q_b01(Pt-6, ピースb)
→ q_g01(Pt-10, ピースg) ✅
```

13 問。排除質問で常識的仮説を消去した後、スープ→過去→接続の順に進行。

### パス B: 過去先行

```
q_a01(Pt-1) → q_a02(Pt-2) → q_a03(Pt-3, ピースa)
→ q_c01(Pt-4) → q_c02(Pt-5, ピースc,d)
→ q_e01(Pt-7) → q_e02(Pt-8, ピースe)
→ q_c03(Pt-9, ピースf) → q_b01(Pt-6, ピースb) → q_g01(Pt-10, ピースg) ✅
```

10 問。排除質問を省略し、過去の探索から開始。

### パス C: スープ先行

```
q_e01(Pt-7) → q_e02(Pt-8, ピースe)
→ q_a01(Pt-1) → q_a02(Pt-2) → q_a03(Pt-3, ピースa)
→ q_c01(Pt-4) → q_c02(Pt-5, ピースc,d)
→ q_c03(Pt-9, ピースf) → q_b01(Pt-6, ピースb) → q_g01(Pt-10, ピースg) ✅
```

10 問。スープの味の探索から開始し、過去に遡る。

### 検証結果

- 3 パスいずれも全ピース発見、T に到達 ✅
- 行き詰まり（オープン質問が尽きる状態）なし ✅
- 各質問のオープンは想起条件により説明可能 ✅

### 備考

現在の仮説定義では全仮説が初期状態で形成されるため、仮説駆動の質問（q_e01, q_a01）が初手から開いている。これにより質問選択の自由度が高い反面、仮説形成による段階的な開放という効果は現れていない。仮説の形成条件の調整は今後の課題。
