# 質問と記述素の対応関係

## 質問と記述素は別物である

- **記述素**: 理論側の概念。記述素空間の構造を規定する。
- **質問**: アプリ側の概念。プレイヤーが記述素空間を探索するためのインターフェース。具体的な文言や提示順序はプレイヤー体験の設計に基づいて調整される。

このドキュメントでは、両者の対応関係の構造を理論的に定義する。

## 対応関係の定義

質問 q に対して、回答ごとに O または R に追加される記述素を定義する。

- **ans(q, はい)** = {(d₁, 1), (d₂, 1), ...} — O に追加される記述素と確定値
- **ans(q, いいえ)** = {(d₁, 0), (d₂, 0), ...} — O に追加される記述素と確定値
- **ans(q, 関係ない)** = {d₁, d₂, ...} — R に追加される記述素

一つの質問が複数の記述素に影響しうる。また「はい」と「いいえ」で影響する記述素の集合が異なりうる。

### 例

質問:「男は過去に遭難した経験がありますか？」

- ans(q, はい) = {(Ft-C06「遭難した」, 1), (Ft-C03「異常な状況だった」, 1)}
- ans(q, いいえ) = {(Ft-C06「遭難した」, 0)}
- ans(q, 関係ない) = {Ft-C06}

「はい」の場合、遭難したこと（直接）に加え、異常な状況だったこと（含意）も確定する。「いいえ」の場合は遭難の否定のみ。

### H への対称的な影響

「はい」と「いいえ」はいずれも H の積極的な更新である。

- **ans(q, はい)**: H(d) を ≈ 1 に更新する。パラダイムの D⁺ に対応する記述素を確定する。
- **ans(q, いいえ)**: H(d) を ≈ 0 に更新する。パラダイムの D⁻ に対応する記述素を確定する。

「いいえ」の回答は単なる情報の不在ではなく、プレイヤーの暫定理論に否定的事実を積極的に組み込む。この否定が他の可能性を排除し、パラダイムの輪郭を明確にする。例えば「男は料理の専門家ですか？→いいえ」は、P4（専門家仮説）を排除し、H の構造を変化させる。

## 対応の種類

### 直接対応

質問が記述素の真偽をそのまま確認する。

- 質問「男は一人でしたか？」→ Fs-A03

### 含意対応

質問の回答から論理的に導かれる記述素にも影響する。

- 質問「男は過去に遭難しましたか？」→ はい → Ft-C06（直接）+ Ft-C03（遭難は異常な状況であるため含意的に確定）

含意対応に含める記述素の範囲は、回答から自明に導かれるものに限定する。プレイヤーの推論に委ねるべきものは含めない。

## 記述素の到達経路

全ての記述素が質問で直接確認されるわけではない。記述素には到達経路による区別がある。

### 質問到達

何らかの質問の回答により O または R に直接追加される記述素。基本的な事実に関する記述素がこれに該当する。

### 推論到達

質問では直接確認されず、O の蓄積からプレイヤーが内的に推論で到達する記述素。以下のような記述素が該当しやすい:

- 合成により生成された記述素（例:「今回が本物なら過去は偽物」）
- 異常性検出により生成された記述素（例:「確認行為は異常」）
- 緊張関係抽出により生成された記述素（例:「本物確認と自殺の緊張」）

推論到達の記述素は、プレイヤーの内的なパラダイムシフトの材料となる。これらを質問で直接開示してしまうと、プレイヤーの「発見する」体験が損なわれる。

## 設計上の制約

- 全ての記述素に質問が対応する必要はない
- 一つの記述素に複数の質問から到達可能でも構わない
- 質問到達と推論到達の配分は、プレイヤー体験の設計に属する判断であり、理論が一意に決定するものではない
