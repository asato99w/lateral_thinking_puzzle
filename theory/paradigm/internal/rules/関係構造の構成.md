# 関係構造の構成

パラダイムの予測関数と想起可能集合が決定された後、R(P) を構成するルール。

## 前提

R(P) ⊆ Conceivable(P) × Conceivable(P) × W であり、(d₁, d₂, w) は「P の説明において d₁ が d₂ の理由・原因・前提である」ことを表す。

## Rule 1: P_pred(d)=1 の記述素間の関係

P_pred(d)=1 の記述素間の因果・含意関係。P の肯定的説明の中の論理構造。

例: P1 において Fs-01（食事目的で来店） → Fs-05（特定の目的）は、P1 の説明の中で前者が後者を含意する。

## Rule 2: P_pred(d)=1 から P_pred(d)=0 への関係

P_pred(d)=1 の事実が成り立つことが、P_pred(d)=0 の事実が成り立たないことの根拠となる関係。

例: P1 において Fs-01（食事目的で来店） → Ft-03（命の危険にさらされた）=0 は、日常の食事目的が極限体験を排除する関係。

この関係は同化において P_pred(d)=0 の記述素の H を 0 方向に押す効果を持つ。

## Rule 3: 共有想起記述素への接続

固有想起記述素から共有想起記述素（隣接パラダイムの Conceivable との交差部分）への関係。段階的開示の連鎖を可能にするために重要。

- 固有想起記述素 → 共有想起記述素 の関係が存在すること
- 同化により共有想起記述素の H が ε 圏内に到達できる w であること

## Rule 4: 強さの設定

w の値は以下の基準で設定する:

- **強い関係（w > 0.6）**: 1ホップの同化で open(H) の ε 圏内に到達する。確認すれば次が自然に問われる関係。
- **中程度の関係（0.3 < w ≤ 0.6）**: 単独では ε 圏内に到達しないが、複数ソースの合流で到達しうる。複数の裏付けが揃って見えてくる関係。
- **弱い関係（w ≤ 0.3）**: 直接的な同化効果は小さい。背景的な関連性。

## Rule 5: ゲート辺

浅い層の記述素から深い層の記述素への縦方向の接続。Q(P) の多層構造において、層間の遷移を制御する。

R(P) の形式定義（Conceivable × Conceivable × W）は Rule 1〜4 と同一であり、ゲート辺は構成上の役割による区別である。

- **機能**: 浅い層の記述素 d_src を観測し同化が起きると、ゲート辺 (d_src, d_tgt, w) を通じて H(d_tgt) が P の予測値方向に動く。H(d_tgt) が閾値（ε 圏内）に到達すると、d_tgt を参照する深い層の質問がオープンする。
- **横辺との区別**: 横辺は同一層内の因果・含意関係（Rule 1〜3）であり、層内での H の広がりを担う。ゲート辺は層間の縦方向の接続であり、次の層への進行を担う。
- **強さの基準**: ゲート辺は原則として強い関係（w > 0.6）とする。ゲートの開放は質問構造設計で意図された到達経路の一部であり、確実に開放される必要がある。
- **設計の方向**: ゲート辺は質問構造設計（アノマリーからの逆算）の結果として必要性が判明し、R(P) に追加される。前方向の積み上げ（Rule 1〜3）とは設計方向が異なる。
