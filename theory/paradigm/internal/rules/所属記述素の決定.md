# 予測と想起可能性の決定

パラダイム発見アルゴリズムによって見出されたパラダイムに対して、その予測関数 P_pred と想起可能集合 Conceivable(P) を決定するルール。

## 前提

パラダイムは「ある段階までの事実を受け入れた上で、残りの未知の事実を常識的に補完した一貫した解釈」である。この一貫性は、肯定（何が真か）だけでなく否定（何が偽か）も含む。

## Rule 1: P_pred の決定

P_pred(P) は、パラダイム P の世界観から論理的に導出される予測の全体。想起可能性とは独立に決定される。

### 1a. P_pred(d) = 1 の判定

P の解釈のもとで真と予測される記述素。

**判定基準**: P のストーリーを構成する要素、および P の説明から自然に導かれる帰結。

### 1b. P_pred(d) = 0 の判定

P の解釈のもとで偽と予測される記述素。

**判定基準**: P の解釈から**論理的に**偽が導出される記述素。P_pred(d)=1 の記述素を根拠として、P の解釈のもとで偽が導かれるものに限る。

具体的には以下のいずれかに該当する記述素:

- **前提矛盾**: P の解釈の前提と論理的に矛盾する記述素
- **排他的説明**: P が同じ現象について異なる説明を持つ記述素
- **範囲否定**: P の解釈のスコープ内にある記述素のうち、P が積極的に「ない」と予測するもの

P_pred(d)=0 には、想起可能性に関係なく、P の世界観から論理的に偽が導出されるすべての記述素が含まれる。例: P1（日常の食事）のもとで Ft-29（人肉を食べた）は想起されないが、日常の食事という世界観から P_pred(Ft-29)=0 は論理的に導出される。

### 1c. 不定の判定

P_pred にキーが存在しない記述素は P にとって不定とする。

**判定基準**: P の解釈が予測を持たない事実。P の世界観から真も偽も論理的に導出できない。

## Rule 2: Conceivable(P) の決定

Conceivable(P) ⊆ keys(P_pred(P)) は、P の概念体系において自然に想起される記述素の集合。

### 判定基準

P の枠組みにいるプレイヤーにとって、自然に思い浮かぶ記述素。P の解釈を形成する中で積極的に参照される概念に対応する。

### Conceivable に含まれるもの

- P の中核的な説明を構成する記述素
- P の説明から直接派生する関連概念に対応する記述素
- P の枠組みで自然に想起される否定（「Xではない」と積極的に思う記述素）

### Conceivable に含まれないもの

- P の世界観が論理的に予測を持つが、P の枠組みでは概念として想起されない記述素
- P にとって「想定外」の概念に対応する記述素

### P_pred との関係

Conceivable(P) ⊆ keys(P_pred(P)) が成立する（想起可能なものは必ず予測を持つ）。逆は成立しない: P_pred に予測があっても Conceivable に含まれない記述素が存在しうる。

例: P1（日常の食事）のもとで「人肉を食べた」は P_pred(d)=0 と論理的に予測されるが、日常の食事の枠組みではその概念が想起されないため Conceivable(P1) には含まれない。一方、「命の危険にさらされた」は日常の食事と矛盾する概念として積極的に否定される（想起される否定）ため Conceivable(P1) に含まれる。

## Rule 3: Conceivable の粒度

Conceivable(P) は P_pred の全キーではなく、P の概念的可視範囲に限定する。P_pred は論理的に広いが、Conceivable は認知的に自然な範囲に制限される。この差分が、パラダイム間の質問ブロックの源泉となる。

## Rule 4: P_pred の運用上の表現

### 理論上の P_pred と運用上の表現

P_pred は理論上、P の世界観から論理的に導出される予測の全体である（Rule 1）。しかし、Conceivable 外の予測（想起されないが論理的には偽と導出される記述素）は膨大になりうる。

運用上、Conceivable 外の予測は以下の条件のもとで**不定として仮置き**してよい:

- エンジンの計算結果に影響しないこと

### エンジンへの影響分析

Conceivable 外の P_pred がエンジンの各計算に与える影響:

| 関数 | P_pred の参照スコープ | Conceivable 外の影響 |
|------|---------------------|---------------------|
| tension(O, P) | Conceivable(P) のみ | なし |
| select_shift_target: attention | Conceivable(P') に明示的に制限 | なし |
| select_shift_target: resolve | Conceivable(P') に制限（Rule 4a） | なし |
| _assimilate_descriptor | R(P) 経由、Conceivable 内に閉じる | なし |
| _assimilate_from_paradigm | O ∩ Conceivable(P) を対象 | なし |
| init_questions | Conceivable ブロック通過後の記述素のみ参照 | なし |
| open_questions | 同上 | なし |

### Rule 4a: resolve の Conceivable 制限

select_shift_target の resolve（P_current のアノマリーのうち P' が正しく予測する数）は、**P' の Conceivable 内に限定する**。

根拠: attention が Conceivable(P') に制限されているのと対称的に、resolve も P' が概念として想起する範囲内でのみ評価する。P' が想起しない記述素について「正しく予測する」と判定するのは、想起可能性に基づくゲームモデルと整合しない。

この制限により、P_pred を Conceivable スコープで表現しても（Conceivable 外を不定として仮置きしても）エンジンの全計算結果が保存される。

### サンプル・データにおける表現

サンプルおよびデータ（JSON）では、P_pred を Conceivable スコープで表現する（D⁺ ∪ D⁻ = Conceivable の記述素のみを列挙する）。Conceivable 外の予測は不定として省略する。
