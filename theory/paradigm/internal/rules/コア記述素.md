# コア記述素

パラダイム P のコア記述素 Core(P) は、P に至って初めて記述可能になる概念に対応する記述素の集合である。

## 定義

Core(P) ⊆ D(P)

Core(P) の各記述素 d は、P の深度より浅いパラダイムの概念体系では記述すること自体ができない概念に対応する。

## anomaly との区別

パラダイム P の D(P) 内の記述素には、以下の二種類がある。

### anomaly 記述素

前段のパラダイムの概念体系で**記述可能だが説明できない**記述素。前段のパラダイムの D(P) にも含まれており、そのパラダイムは予測を持つが、観測と矛盾する。anomaly として緊張を蓄積し、パラダイムシフトを駆動する役割を持つ。

### コア記述素

前段のパラダイムの概念体系で**記述すらできない**記述素。前段のパラダイムの D(P) に含まれない（不定である）か、あるいはその概念が前段のパラダイムの解釈の枠組みから見て自然に想起されないものである。

## 質問オープンへの制約

質問 q が参照する記述素（ans(q, はい)、ans(q, いいえ)、ans(q, 関係ない) のいずれかに含まれる記述素）のうち、いずれかが Core(P_k) に属し、かつ depth(P_k) > depth(P_current) である場合、q は現在のパラダイム P_current のもとではオープンしない。

この制約は prerequisites や H 値条件よりも先に適用される。コア制約を満たさない質問は、他の条件にかかわらずオープンしない。

## 判定基準

記述素 d がパラダイム P のコアであるかの判定:

1. d ∈ D(P) であること
2. d の内容を質問テキストに含めることが、depth(P) 未満の段階のプレイヤーにとって核心的筋書きを直接暴露すること
3. depth(P) より浅いパラダイムの概念体系から、d に対応する概念が自然に想起されないこと

基準 2 と 3 はいずれも判断を伴う。anomaly（「説明できないが記述はできる」→ コアではない）との区別を意識して判定する。

## 粒度

Core(P) は D(P) 全体ではなく、P の核心概念に限定する。パラダイムの中核的な説明を構成する少数の記述素のみがコアとなる。周辺的な記述素はコアに含めない。

## D(P) 所属との関係

コア記述素は、それが属するパラダイム P の D(P) に含まれるが、P より浅いパラダイム P' の D(P') には含まれない。浅いパラダイムにとってコア記述素は不定（想定外）である。

これは所属記述素の決定 Rule 2 の帰結である。D⁻ には P の解釈から否定が積極的に導出される記述素のみが含まれるが、コア記述素は浅いパラダイムの概念体系では想起すらされないため、否定の導出が成り立たない。したがってコア記述素は浅いパラダイムの D⁺ にも D⁻ にも属さず、不定となる。

この性質により、コア記述素を参照する質問は浅いパラダイムのもとで H 値条件を満たさない（H がデフォルト値 0.5 のまま）。質問オープンへの制約（上述）はこれに加えてのセーフガードとして機能する。

## 備考

- コアはクーンの科学革命における通約不可能性に着想を得ている。異なるパラダイム間で概念の可視性が異なるという性質を、ゲームの段階的情報開示の制御に応用したものである。ただし、クーンの通約不可能性が「概念の意味が変わる」ことを含むのに対し、本フレームワークのコアは「概念の可視性」の問題であり、対応の範囲は限定的である。
- コアが空のパラダイム（Core(P) = ∅）も許容される。表層のパラダイムは一般に概念的にアクセス可能な記述素のみで構成されるため、コアを持たないことが自然である。
