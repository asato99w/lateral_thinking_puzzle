# /property-analysis — 特性分析

引数: `$ARGUMENTS`（分析対象の名前）
- 例: `/property-analysis コンタクトレンズ`
- 例: `/property-analysis 腕時計`

## 目的

日常の対象について、パズル設計に活用できる前提群と演繹構造を体系的に導出する。

## 背景理論（分析14より）

パラダイムの主要機能は**前提群の提供**である。
前提群から演繹が行われ、予測が導出される。
観察が予測を反証した時、前提群のいずれかが誤りだったことになり、
**どの前提が誤りで、なぜか**がパズルの真相になりうる。

## 実行手順

### Step 1: テンプレート展開

対象について以下の6項目を展開する。網羅的に、具体的に記述すること。

1. **状態**: どのような状態を取りうるか。二値的か連続的か。取りうる状態を全て列挙する
2. **ルーティン**: 状態遷移の時間的パターン。日常の中でどう使われるか。典型的な使用サイクルを記述する
3. **帰結**: 各状態から観察可能な結果は何か。状態ごとに、何が知覚・観測できるかを記述する
4. **遷移条件**: 状態が変わるには何が必要か。自然に起きるものと意図的行為が必要なものを区別する
5. **推論**: 予想外の状態が観察された場合、何が推論できるか。各状態の異常と、そこから導かれる推論を記述する
6. **関係性**: 人・身体・環境とどう関わるか。所有、使用、社会的意味、他の物との関連を記述する

### Step 2: 本質的機能の特定

テンプレート展開の結果から、対象の**本質的機能**（その対象がその対象である所以の機能）を特定する。
本質的でない特性も記録する（ミスリードの素材として活用可能なため）。

### Step 3: 演繹構造の導出

本質的機能を中心に、以下の形式で演繹構造を導出する。
反証可能性が明確で強いものを優先すること。

```
演繹N（原理名）：
- 前提群：[前提1] ＋ [前提2] ＋ ...
- 演繹：[予測]
- 反証：[予測の否定が観察される状況] → [覆される前提] → [推論]
```

演繹構造を生む原理（分析13より）：
- **保存・持続**: 存在するものは作用なしに存在し続ける
- **因果**: 行為は痕跡を残す
- **共起**: 自然に共存するものがある
- **同一性**: 同じものは同じ性質を持つ
- **能力**: 特定の効果には特定の能力が必要
- **時間変化**: 時間経過に伴い予測可能な変化がある

各原理を対象に適用し、成立する演繹を全て抽出する。

### Step 4: 所感

- どの演繹構造が最も豊かか（連鎖が長い、分岐が多い）
- 既存パズルとの構造的類似はあるか
- パズル設計の種として有望なものはどれか
- 対象の全体的な「前提群の豊かさ」の評価

## 出力

分析結果を `notes/` に以下のファイル名で保存する：
`YYYY-MM-DD_特性分析_{対象名}.md`
